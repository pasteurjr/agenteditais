# Caso de Uso: UC-033

## Nome
Gerar peticao de impugnacao

## Objetivo
Criar peticao formal de impugnacao ou esclarecimento a partir de inconsistencias identificadas na validacao legal (UC-032), utilizando templates pre-configurados e geracao de texto via IA com formato juridico adequado.

## Escopo
Modulo Impugnacao — ImpugnacaoPage.tsx (aba Peticoes — botao Nova Peticao)

## Atores
- Usuario autenticado (Analista Juridico / Comercial)
- Sistema backend (Flask)
- Motor de IA (DeepSeek) para geracao de texto
- Templates de peticao (CRUD recurso-templates)

## Gatilho
Usuario clica no botao "Nova Peticao" (icone Plus) na aba Peticoes da ImpugnacaoPage, abrindo modal de criacao.

## Pre-condicoes
1. Usuario esta autenticado com token JWT valido
2. Backend Flask ativo na porta 5007
3. Ao menos um edital salvo no sistema com validacao legal executada (UC-032)
4. Templates de peticao cadastrados via crudList("recurso-templates")
5. Inconsistencias identificadas na aba Validacao Legal (lista nao vazia)
6. Empresa cadastrada com dados juridicos completos (razao social, CNPJ, endereco)
7. DeepSeek acessivel para geracao de texto da peticao

## Pos-condicoes
1. Peticao criada e salva via crudCreate("recursos") com status "rascunho"
2. Conteudo da peticao disponivel para edicao posterior na lista de peticoes
3. Peticao associada ao edital selecionado (edital_id)
4. Tipo da peticao registrado: "esclarecimento" ou "impugnacao"
5. Template utilizado registrado (template_id)
6. Peticao visivel na DataTable da aba Peticoes com status, tipo e data
7. Conteudo em formato markdown pronto para exportacao

## Regras de negocio
1. **RN-PET-001:** Peticao possui dois tipos: "esclarecimento" (duvida) e "impugnacao" (contestacao formal)
2. **RN-PET-002:** Templates carregados de crudList("recurso-templates"), filtrados por ativo !== false
3. **RN-PET-003:** Ao selecionar template, conteudo_md pre-preenche o campo de conteudo da peticao
4. **RN-PET-004:** Peticao criada com status inicial "rascunho"; pode evoluir para "revisao" e "enviada"
5. **RN-PET-005:** Formato formal obrigatorio: Cabecalho, Dos Fatos, Do Direito, Do Pedido, Encerramento
6. **RN-PET-006:** Geracao via IA usa createSession + sendMessage com prompt contendo inconsistencias mapeadas
7. **RN-PET-007:** Peticao persistida via crudCreate("recursos") com campos: edital_id, tipo, template_id, texto_minuta, status

## Componentes de UI envolvidos
1. ActionButton "Nova Peticao" (icone Plus) que abre modal showNovaPeticaoModal
2. Modal de criacao com campos: SelectInput edital, SelectInput tipo (esclarecimento/impugnacao), SelectInput template
3. TextArea para conteudo da peticao (peticaoConteudo) com pre-preenchimento do template
4. ActionButton "Salvar" (icone Save) com estado loading (peticoesSaving)
5. DataTable na aba Peticoes listando peticoes salvas com colunas: edital, tipo, status, data_criacao
6. Badges de status: rascunho (neutral), revisao (warning), enviada (success)
7. Icones de acao por linha: Eye (visualizar), Edit3 (editar), Trash2 (excluir), Download (exportar)

## Endpoints ou servicos envolvidos
1. **crudList("recurso-templates")** — carrega templates de peticao ativos
2. **crudCreate("recursos")** — salva nova peticao com dados do formulario
3. **crudList("recursos")** — lista peticoes existentes para DataTable
4. **crudUpdate("recursos", id)** — atualiza peticao (status, conteudo)
5. **crudDelete("recursos", id)** — exclui peticao
6. **createSession("gerar-peticao")** — cria sessao de chat para geracao IA
7. **sendMessage(sid, prompt)** — envia prompt com inconsistencias para IA gerar texto

## Dados manipulados
1. **Peticao:** id, edital_id, edital_numero, tipo ("esclarecimento"|"impugnacao"), template_id, status, conteudo, data_criacao
2. **TemplateOption:** id, nome, conteudo_md — template pre-configurado
3. **peticaoTipo:** "esclarecimento" | "impugnacao" — tipo selecionado pelo usuario
4. **peticaoTemplateId:** string — ID do template escolhido
5. **peticaoEditalId:** string — ID do edital vinculado
6. **peticaoConteudo:** string — texto editavel da peticao em markdown
7. **peticoesSaving:** boolean — estado de loading durante salvamento

## Fluxo principal
1. Usuario acessa aba Peticoes na ImpugnacaoPage e clica no botao "Nova Peticao" (icone Plus) na ActionBar
2. Modal abre com formulario: SelectInput de edital (numero - orgao), SelectInput de tipo (esclarecimento/impugnacao), SelectInput de template
3. Usuario seleciona edital vinculado, tipo da peticao e template desejado; sistema pre-preenche TextArea com conteudo_md do template
4. Usuario edita o conteudo da peticao, ajustando trechos especificos para o caso concreto do edital selecionado
5. Usuario clica em "Salvar"; frontend chama crudCreate("recursos") com edital_id, tipo, template_id, texto_minuta e status "rascunho"
6. Backend persiste peticao no banco de dados e retorna ID gerado; frontend fecha modal e recarrega lista de peticoes
7. Nova peticao aparece na DataTable da aba Peticoes com status "rascunho", permitindo edicao, visualizacao e exportacao posterior

## Fluxos alternativos
1. **FA-01 — Gerar com IA:** Usuario clica "Gerar Peticao" na aba Validacao Legal; sistema cria sessao, envia inconsistencias e preenche peticaoSugerida
2. **FA-02 — Editar peticao existente:** Usuario clica icone Edit3 em peticao da lista; modal reabre com dados pre-preenchidos para edicao
3. **FA-03 — Alterar status:** Usuario altera status de "rascunho" para "revisao" ou "enviada" via acao na DataTable
4. **FA-04 — Sem template:** Nenhum template cadastrado; TextArea inicia vazio e usuario redige peticao do zero
5. **FA-05 — Exportar peticao:** Usuario clica Download para exportar peticao em formato DOCX ou PDF
6. **FA-06 — Duplicar peticao:** Usuario cria nova peticao copiando conteudo de peticao existente como base
7. **FA-07 — Visualizar preview:** Usuario clica Eye para visualizar peticao renderizada em markdown antes de enviar

## Fluxos de excecao
1. **FE-01 — Nenhum edital selecionado:** Botao Salvar desabilitado se peticaoEditalId estiver vazio
2. **FE-02 — Erro ao salvar:** crudCreate retorna erro; frontend exibe mensagem de falha e mantem modal aberto
3. **FE-03 — Template nao carregado:** crudList("recurso-templates") falha; setTemplates([]) graciosamente, usuario digita manualmente
4. **FE-04 — Conteudo vazio:** Validacao impede salvamento de peticao sem conteudo no TextArea
5. **FE-05 — Token expirado:** Requisicao retorna 401; frontend redireciona para login
6. **FE-06 — Erro de rede:** fetch lanca excecao; console.error registra e usuario recebe feedback visual
7. **FE-07 — Template corrompido:** conteudo_md do template vazio ou invalido; TextArea inicia vazio com aviso
