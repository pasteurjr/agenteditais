# Caso de Uso: UC-049

## Nome
Registrar entrega com NF

## Objetivo
Permitir ao usuario registrar entregas de contratos com numero de nota fiscal (NF) e empenho, controlando quantidade, valor, data prevista e data realizada, com calculo automatico de valor total e status de entrega.

## Escopo
Modulo Contratos — ProducaoPage.tsx > Aba Entregas

## Atores
- Usuario autenticado (Gestor de Contratos)
- Sistema backend (Flask)
- CRUD generico (crudCreate contrato-entregas)
- Modulo de cronograma (atualizacao automatica)

## Gatilho
Usuario seleciona um contrato na aba Contratos, navega para aba "Entregas" e clica no botao "Nova Entrega".

## Pre-condicoes
1. Usuario esta autenticado no sistema
2. Ao menos um contrato cadastrado e selecionado (UC-048)
3. Backend Flask ativo com CRUD /api/crud/contrato-entregas
4. API crudCreate importada para contrato-entregas
5. Contrato selecionado possui status "vigente"
6. ProducaoPage acessivel no menu do sistema
7. Dados da entrega disponiveis (descricao, quantidade, valor, data prevista)

## Pos-condicoes
1. Entrega persistida no banco vinculada ao contrato_id selecionado
2. Status da entrega setado como "pendente" automaticamente
3. Valor total calculado: quantidade * valor_unitario
4. DataTable de entregas atualizada com novo registro
5. Cronograma de entregas (UC-050) atualizado com nova data prevista
6. Dashboard Contratado x Realizado (UC-054) reflete nova entrega
7. Modal de registro fechado e formulario resetado

## Regras de negocio
1. **RN-ENT-001:** Status inicial = "pendente" (setado automaticamente)
2. **RN-ENT-002:** 3 status possiveis: pendente, entregue, atrasado
3. **RN-ENT-003:** valor_total = quantidade * valor_unitario (calculado no frontend)
4. **RN-ENT-004:** Entrega vinculada a contrato via contrato_id (foreign key)
5. **RN-ENT-005:** Nota fiscal (nota_fiscal) e numero de empenho (numero_empenho) sao opcionais no cadastro
6. **RN-ENT-006:** Entrega sem data_realizada e data_prevista < hoje e considerada "atrasada"
7. **RN-ENT-007:** crudCreate("contrato-entregas") utiliza CRUD generico (UC-058)

## Componentes de UI envolvidos
1. TabPanel aba "Entregas" na ProducaoPage (requer contrato selecionado)
2. ActionButton "Nova Entrega" com icone Plus para abrir modal
3. Modal de registro com FormFields: descricao, quantidade, valor_unitario, data_prevista, nota_fiscal, numero_empenho
4. DataTable de entregas com colunas: Descricao, Quantidade, Valor Unit., Valor Total, Data Prevista, NF, Status
5. Badge de status: pendente (amarelo), entregue (verde), atrasado (vermelho)
6. Formatacao monetaria R$ para valores
7. Mensagem "Selecione um contrato" quando nenhum contrato selecionado

## Endpoints ou servicos envolvidos
1. **crudCreate("contrato-entregas")** — cria entrega vinculada ao contrato
2. **crudList("contrato-entregas", { filters: { contrato_id } })** — lista entregas do contrato
3. **CRUD /api/crud/contrato-entregas** — CRUD completo para entregas
4. **GET /api/contratos/{id}/cronograma** — cronograma atualizado apos nova entrega
5. **GET /api/dashboard/contratado-realizado** — dashboard que incorpora entregas
6. **PUT /api/crud/contrato-entregas/{id}** — atualizar entrega (marcar como entregue)
7. **Auth middleware** — validacao de token JWT

## Dados manipulados
1. **EntregaAPI:** id, contrato_id, descricao, quantidade, valor_unitario, valor_total, data_prevista, data_realizada, nota_fiscal, numero_empenho, status
2. **Form state:** descricao, quantidade (string), valor_unitario (string), data_prevista, nota_fiscal, numero_empenho
3. **Payload crudCreate:** { contrato_id, ...form, quantidade: parseFloat(), valor_unitario: parseFloat(), valor_total: calculado, status: "pendente" }
4. **Calculo:** valor_total = (parseFloat(quantidade) || 0) * (parseFloat(valor_unitario) || 0)
5. **Badge cores:** pendente=#fef3c7/#92400e, entregue=#dcfce7/#166534, atrasado=#fee2e2/#991b1b
6. **NF e empenho:** strings opcionais para rastreabilidade fiscal
7. **Datas:** data_prevista (planejada), data_realizada (efetiva, preenchida ao confirmar entrega)

## Fluxo principal
1. Usuario seleciona contrato na aba "Contratos" e sistema carrega entregas via crudList com filtro contrato_id
2. Usuario navega para aba "Entregas" e visualiza DataTable com entregas existentes do contrato selecionado
3. Usuario clica em "Nova Entrega" (icone Plus) que abre Modal de registro
4. Usuario preenche campos: descricao, quantidade, valor_unitario, data_prevista, nota_fiscal e numero_empenho
5. Usuario clica em "Salvar" e frontend calcula valor_total = quantidade * valor_unitario e chama crudCreate("contrato-entregas")
6. Backend persiste entrega com contrato_id e status="pendente" via CRUD generico
7. Frontend fecha Modal, reseta formulario e chama selectContrato() que recarrega entregas, cronograma, aditivos e designacoes

## Fluxos alternativos
1. **FA-01 — Sem contrato selecionado:** Aba Entregas exibe mensagem "Selecione um contrato na aba Contratos" sem acoes
2. **FA-02 — Marcar como entregue:** Usuario atualiza entrega preenchendo data_realizada e NF; status muda para "entregue"
3. **FA-03 — Entrega sem NF:** Nota fiscal nao informada no cadastro; pode ser adicionada depois via edicao
4. **FA-04 — Entrega atrasada:** data_prevista < hoje e status "pendente"; badge mostra "atrasado" em vermelho
5. **FA-05 — Multiplas entregas:** Usuario registra varias entregas para mesmo contrato; cada uma com data prevista propria
6. **FA-06 — Editar entrega:** Usuario usa CRUD update para corrigir dados; DataTable atualiza
7. **FA-07 — Excluir entrega:** Usuario remove entrega incorreta via CRUD delete; lista atualiza

## Fluxos de excecao
1. **FE-01 — Erro no crudCreate:** API retorna erro; try/catch loga no console
2. **FE-02 — Token expirado:** Requisicao retorna 401; sistema redireciona para LoginPage
3. **FE-03 — Valores invalidos:** parseFloat retorna NaN; valores salvos como 0 (default)
4. **FE-04 — Contrato inexistente:** contrato_id invalido; backend retorna erro de foreign key
5. **FE-05 — Data prevista no passado:** Sistema aceita mas marca como potencialmente atrasada
6. **FE-06 — Erro de rede:** fetch falha; try/catch captura e loga
7. **FE-07 — Contrato encerrado:** Tentativa de registrar entrega em contrato nao vigente; backend pode rejeitar
