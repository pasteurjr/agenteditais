# Caso de Uso: UC-027

## Nome
Editar proposta no editor rico

## Objetivo
Permitir ao usuario editar o texto da proposta tecnica em um editor rico com suporte a Markdown, utilizando barra de ferramentas com formatacao (negrito, italico, titulos, listas, tabelas), preview em tempo real e controle de versoes via PropostaLog.

## Escopo
Modulo Proposta — PropostaPage.tsx

## Atores
- Usuario autenticado (Analista Comercial / Redator de Proposta)
- Sistema backend (Flask)
- Banco de dados

## Gatilho
Usuario clica em "Editar" em uma proposta existente na DataTable da PropostaPage, ou acessa a area de edicao apos geracao de texto por IA (UC-026).

## Pre-condicoes
1. Proposta criada com texto_tecnico preenchido (UC-026 executado) ou proposta em rascunho
2. Proposta em status "rascunho" ou "revisao" (status "aprovada" ou "enviada" bloqueiam edicao)
3. Backend Flask ativo e banco de dados acessivel
4. PropostaPage carregada com proposta selecionada
5. Componente MarkdownToolbar renderizado com botoes de formatacao
6. TextArea com ref (textareaRef) disponivel para manipulacao de texto
7. Usuario com permissao para editar propostas

## Pos-condicoes
1. Texto da proposta atualizado com formatacao Markdown aplicada
2. Alteracao persistida via PUT /api/propostas/{id} no campo texto_tecnico
3. PropostaLog atualizado com registro da edicao (acao, usuario, timestamp)
4. Preview em ReactMarkdown atualizado em tempo real com texto editado
5. Status da proposta mantido ou alterado conforme fluxo (rascunho -> revisao)
6. Historico de alteracoes disponivel para auditoria via PropostaLog
7. Proposta pronta para validacao ANVISA (UC-028) e auditoria documental (UC-029)

## Regras de negocio
1. **RN-EDT-001:** Editor suporta Markdown completo: negrito, italico, titulos H1/H2, listas, tabelas
2. **RN-EDT-002:** Preview em tempo real renderiza Markdown via ReactMarkdown com remarkGfm
3. **RN-EDT-003:** Edicao so e permitida em status "rascunho" ou "revisao"
4. **RN-EDT-004:** Cada salvamento registra entrada no PropostaLog com diff do texto alterado
5. **RN-EDT-005:** Toggle entre modo edicao (TextArea) e modo preview (ReactMarkdown) via ToggleLeft/ToggleRight
6. **RN-EDT-006:** Barra de ferramentas insere formatacao na posicao do cursor preservando selecao
7. **RN-EDT-007:** Texto salvo automaticamente a cada 30 segundos (autosave) para evitar perda

## Componentes de UI envolvidos
1. MarkdownToolbar com botoes: Bold, Italic, Heading1, Heading2, List, Table
2. TextArea com ref (textareaRef) para edicao do texto em Markdown puro
3. ReactMarkdown com remarkGfm para preview em tempo real do texto formatado
4. Toggle edicao/preview (ToggleLeft/ToggleRight) para alternar entre modos
5. Botao "Salvar" (ActionButton com icone FileEdit) que persiste texto editado
6. StatusBadge exibindo status atual da proposta (rascunho/revisao/aprovada/enviada)
7. Botao "Alterar Status" com SelectInput para transicao de status (rascunho -> revisao -> aprovada)

## Endpoints ou servicos envolvidos
1. **PUT /api/propostas/{id}** — atualiza texto_tecnico e outros campos da proposta
2. **crudUpdate('propostas', id)** — atualiza proposta no banco via CRUD generico
3. **crudList('propostas')** — recarrega lista de propostas apos edicao
4. **crudGet('propostas', id)** — carrega proposta especifica para edicao
5. **crudCreate('proposta_logs')** — registra entrada de log para auditoria
6. **crudUpdate('propostas', id, { status })** — atualiza status da proposta
7. **crudList('proposta_logs', { proposta_id })** — lista historico de alteracoes

## Dados manipulados
1. **conteudo/texto_tecnico:** string (markdown) — texto completo da proposta em formato Markdown
2. **Status:** "rascunho"|"revisao"|"aprovada"|"enviada" — estado atual da proposta
3. **PropostaLog:** proposta_id, acao ("edicao"|"status"|"geracao"), usuario, timestamp
4. **Selecao de texto:** start (number), end (number) — posicao do cursor para insercao de formatacao
5. **Texto selecionado:** string — trecho selecionado que recebe formatacao da toolbar
6. **Modo de visualizacao:** "edicao"|"preview" — toggle entre TextArea e ReactMarkdown
7. **STATUS_COLORS/STATUS_LABELS:** mapeamento de status para cores e labels visuais

## Fluxo principal
1. Usuario seleciona proposta em status "rascunho" na DataTable e clica "Editar" abrindo area de edicao
2. Sistema carrega texto_tecnico da proposta no TextArea e exibe MarkdownToolbar acima com botoes de formatacao
3. Usuario edita texto diretamente no TextArea ou seleciona trecho e aplica formatacao via toolbar (Bold, Italic, etc)
4. MarkdownToolbar insere tags Markdown na posicao do cursor: **texto** para negrito, *texto* para italico, # para titulo
5. Usuario alterna para modo preview via ToggleRight para visualizar texto renderizado em ReactMarkdown com remarkGfm
6. Usuario clica "Salvar" que dispara PUT /api/propostas/{id} persistindo texto_tecnico atualizado no banco
7. Sistema registra entrada no PropostaLog com acao "edicao", usuario e timestamp, e atualiza DataTable

## Fluxos alternativos
1. **FA-01 — Autosave:** Sistema salva automaticamente a cada 30s; usuario nao precisa clicar "Salvar"
2. **FA-02 — Inserir tabela:** Usuario clica botao Table na toolbar; sistema insere template de tabela Markdown
3. **FA-03 — Alterar status:** Usuario altera status de "rascunho" para "revisao" via SelectInput de status
4. **FA-04 — Desfazer edicao:** Usuario usa Ctrl+Z no TextArea para desfazer alteracoes recentes
5. **FA-05 — Copiar de template:** Usuario aplica outro template ao texto; sistema substitui conteudo
6. **FA-06 — Edicao colaborativa:** Dois usuarios editam mesma proposta; ultima escrita prevalece
7. **FA-07 — Regenerar secao:** Usuario seleciona secao e clica "Regenerar com IA" para reescrever trecho

## Fluxos de excecao
1. **FE-01 — Proposta bloqueada:** Status "aprovada" ou "enviada"; sistema impede edicao com mensagem
2. **FE-02 — Texto vazio:** Usuario tenta salvar com texto vazio; sistema exibe erro "Texto obrigatorio"
3. **FE-03 — Erro de persistencia:** PUT retorna erro 500; frontend exibe toast de erro e mantem texto local
4. **FE-04 — Sessao expirada:** Token JWT invalido; sistema redireciona para login com perda de edicoes nao salvas
5. **FE-05 — Markdown invalido:** Texto com tags abertas sem fechamento; preview exibe parcialmente
6. **FE-06 — Texto muito longo:** Texto excede limite de caracteres; sistema alerta e impede salvamento
7. **FE-07 — Conflito de versao:** Autosave conflita com salvamento manual; sistema prioriza versao mais recente
