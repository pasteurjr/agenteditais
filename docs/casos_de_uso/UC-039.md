# Caso de Uso: UC-039

## Nome
Gerar laudo de recurso com IA

## Objetivo
Gerar laudo formal de recurso administrativo ou tecnico com inteligencia artificial, contendo secoes juridica e tecnica fundamentadas, a partir de inconsistencias identificadas na analise da proposta vencedora (UC-037), para protocolo no orgao licitante.

## Escopo
Modulo Recursos — RecursosPage.tsx (aba Laudos — botao Novo Laudo, tipo "recurso")

## Atores
- Usuario autenticado (Analista Juridico / Comercial)
- Sistema backend (Flask)
- Motor de IA (DeepSeek) via tool_gerar_laudo_recurso
- Templates de recurso (CRUD recurso-templates)

## Gatilho
Usuario clica no botao "Novo Laudo" (icone Plus) na aba Laudos da RecursosPage, seleciona tipo "recurso" e confirma criacao.

## Pre-condicoes
1. Usuario esta autenticado com token JWT valido
2. Backend Flask ativo com DeepSeek configurado
3. Edital salvo com analise da proposta vencedora executada (UC-037)
4. Inconsistencias identificadas disponiveis como base para o laudo
5. Templates de recurso cadastrados via crudList("recurso-templates")
6. Empresa cadastrada com dados juridicos completos (razao social, CNPJ)
7. RecursosPage com aba Laudos acessivel e carregada

## Pos-condicoes
1. Laudo de recurso criado e salvo via crudCreate("recursos-detalhados") com status "rascunho"
2. Conteudo do laudo inclui secao juridica com fundamentacao legal
3. Conteudo do laudo inclui secao tecnica com analise de desvios
4. Laudo associado ao edital e empresa-alvo (empresa vencedora)
5. Subtipo registrado: "administrativo" ou "tecnico"
6. Laudo visivel na DataTable da aba Laudos com status, tipo e data
7. Conteudo em markdown pronto para edicao e exportacao

## Regras de negocio
1. **RN-LAU-001:** Laudo possui tipo "recurso" com subtipo "administrativo" ou "tecnico"
2. **RN-LAU-002:** Secao juridica fundamenta recurso com artigos da Lei 14.133/2021 e legislacao complementar (RN-REC-003)
3. **RN-LAU-003:** Secao tecnica detalha desvios da proposta vencedora em relacao ao edital
4. **RN-LAU-004:** Geracao via POST /api/recursos com tool_gerar_laudo_recurso no DeepSeek
5. **RN-LAU-005:** Templates carregados de crudList("recurso-templates") filtrados por tipo "recurso" e ativo
6. **RN-LAU-006:** Status inicial "rascunho"; pode evoluir para "revisao", "protocolado", "deferido", "indeferido"
7. **RN-LAU-007:** Campo empresa_alvo identifica empresa vencedora contra quem o recurso e dirigido

## Componentes de UI envolvidos
1. ActionButton "Novo Laudo" (icone Plus) na ActionBar da aba Laudos
2. Modal showNovoLaudoModal com campos: SelectInput edital, SelectInput tipo (recurso/contra_razao), SelectInput subtipo, TextInput empresa_alvo, SelectInput template
3. TextArea para conteudo do laudo (laudoConteudo) com pre-preenchimento do template
4. ActionButton "Salvar" (icone Save) com estado loading (laudosSaving)
5. DataTable na aba Laudos com colunas: edital, tipo, subtipo, empresa_alvo, status, data_criacao
6. Badges de status: rascunho (neutral), revisao (warning), protocolado (info), deferido (success), indeferido (error)
7. Icones de acao: Eye (visualizar), Edit3 (editar), Trash2 (excluir), Download (exportar), Send (submeter)

## Endpoints ou servicos envolvidos
1. **POST /api/recursos** — cria recurso e gera laudo com tool_gerar_laudo_recurso
2. **tool_gerar_laudo_recurso** — tool DeepSeek que gera secoes juridica e tecnica
3. **crudCreate("recursos-detalhados")** — persiste laudo no banco de dados
4. **crudList("recursos-detalhados")** — lista laudos existentes para DataTable (loadLaudos)
5. **crudList("recurso-templates")** — carrega templates de recurso ativos (loadTemplates)
6. **crudUpdate("recursos-detalhados", id)** — atualiza laudo (status, conteudo)
7. **crudDelete("recursos-detalhados", id)** — exclui laudo

## Dados manipulados
1. **Laudo:** id, edital_id, edital_numero, tipo ("recurso"), subtipo ("administrativo"|"tecnico"), empresa_alvo, template_id, status, conteudo, data_criacao
2. **laudoEditalId:** string — UUID do edital vinculado ao laudo
3. **laudoTipo:** "recurso" | "contra_razao" — tipo selecionado (neste UC: "recurso")
4. **laudoSubtipo:** "administrativo" | "tecnico" — subtipo do recurso
5. **laudoEmpresaAlvo:** string — nome da empresa vencedora (alvo do recurso)
6. **laudoTemplateId:** string — ID do template escolhido
7. **laudoConteudo:** string — texto editavel do laudo em markdown

## Fluxo principal
1. Usuario acessa aba Laudos na RecursosPage e clica no botao "Novo Laudo" (icone Plus) na ActionBar
2. Modal abre com formulario: SelectInput edital, SelectInput tipo ("recurso"), SelectInput subtipo (administrativo/tecnico), TextInput empresa_alvo, SelectInput template
3. Usuario preenche campos: seleciona edital vinculado, tipo "recurso", subtipo, nome da empresa vencedora e template desejado
4. Sistema pre-preenche TextArea com conteudo_md do template selecionado; usuario edita conforme necessidade
5. Usuario clica em "Salvar"; frontend chama POST /api/recursos ou crudCreate("recursos-detalhados") com dados do formulario
6. Backend executa tool_gerar_laudo_recurso que gera secoes juridica (fundamentacao legal) e tecnica (desvios da proposta) via DeepSeek
7. Laudo salvo com status "rascunho"; modal fecha, lista recarrega via loadLaudos() e novo laudo aparece na DataTable

## Fluxos alternativos
1. **FA-01 — Editar laudo existente:** Usuario clica Edit3 em laudo da lista; modal reabre com dados pre-preenchidos
2. **FA-02 — Alterar status:** Usuario altera status de "rascunho" para "revisao" ou "protocolado" via acao na DataTable
3. **FA-03 — Sem template:** TextArea inicia vazio; usuario redige ou solicita geracao completa pela IA
4. **FA-04 — Exportar laudo:** Usuario clica Download para exportar laudo em DOCX ou PDF para protocolo externo
5. **FA-05 — Submissao assistida:** Usuario clica Send para iniciar fluxo de submissao assistida (UC-041)
6. **FA-06 — Visualizar preview:** Usuario clica Eye para visualizar laudo renderizado em markdown
7. **FA-07 — Duplicar laudo:** Usuario cria novo laudo copiando conteudo de laudo existente como base

## Fluxos de excecao
1. **FE-01 — Nenhum edital selecionado:** Botao Salvar desabilitado se laudoEditalId estiver vazio
2. **FE-02 — Empresa-alvo vazia:** Validacao alerta que campo empresa_alvo e obrigatorio para recurso
3. **FE-03 — Erro ao salvar:** crudCreate retorna erro; frontend exibe mensagem e mantem modal aberto
4. **FE-04 — Template nao carregado:** crudList("recurso-templates") falha; templates vazio, usuario digita manualmente
5. **FE-05 — Erro na IA:** tool_gerar_laudo_recurso falha; backend retorna erro; laudo salvo sem conteudo gerado
6. **FE-06 — Token expirado:** Requisicao retorna 401; frontend redireciona para login
7. **FE-07 — Conteudo vazio:** Validacao impede salvamento se laudoConteudo estiver vazio
