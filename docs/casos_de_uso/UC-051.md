# Caso de Uso: UC-051

## Nome
Gestao de aditivos contratuais

## Objetivo
Permitir ao usuario registrar e gerenciar aditivos contratuais (acrescimo/supressao), com controle automatico de limite legal de 25% (Lei 14.133 Art. 124-126), barra de progresso de consumo e alertas ao atingir 80% do limite.

## Escopo
Modulo Contratos — ProducaoPage.tsx > Aba Aditivos

## Atores
- Usuario autenticado (Gestor de Contratos)
- Sistema backend (Flask)
- Motor de validacao legal (limites Art. 124-126)
- Modulo de alertas (notificacao de proximidade do limite)

## Gatilho
Usuario seleciona um contrato na aba Contratos da ProducaoPage e acessa a aba "Aditivos" para registrar ou consultar aditivos.

## Pre-condicoes
1. Usuario esta autenticado no sistema
2. Contrato cadastrado e selecionado com valor_total preenchido (UC-048)
3. Backend Flask ativo com endpoint POST /api/contratos/{id}/aditivos
4. Endpoint GET /api/contratos/{id}/aditivos retorna aditivos e resumo
5. Valor original do contrato > 0 para calculo de percentual
6. ProducaoPage acessivel no menu do sistema
7. Fundamentacao legal conhecida (Art. 124-126 Lei 14.133)

## Pos-condicoes
1. Aditivo persistido com tipo, valor, justificativa e fundamentacao legal
2. Resumo atualizado: valor_original, total_acrescimos, total_supressoes, limite_25_pct, pct_consumido
3. Barra de progresso exibe percentual consumido do limite de 25%
4. Alerta visual quando percentual consumido atinge 80% do limite
5. Lista de aditivos atualizada na DataTable
6. Valor total do contrato ajustado (original + acrescimos - supressoes)
7. Dashboard Contratado x Realizado reflete alteracao de valor

## Regras de negocio
1. **RN-CON-001:** Limite geral de aditivo = 25% do valor original do contrato (Art. 124-126 Lei 14.133)
2. **RN-CON-002:** Alerta visual quando pct_consumido atinge 80% do limite (barra muda de cor)
3. **RN-ADI-001:** Dois tipos de aditivo: "acrescimo" (aumenta valor) e "supressao" (diminui valor)
4. **RN-ADI-002:** limite_25_pct = valor_original * 0.25 (calculado pelo backend)
5. **RN-ADI-003:** pct_consumido = (total_acrescimos / limite_25_pct) * 100
6. **RN-ADI-004:** Fundamentacao legal obrigatoria (ex: "Art. 124, I, Lei 14.133/2021")
7. **RN-ADI-005:** Para reformas, limite pode ser 50% (Art. 125) — validacao especifica

## Componentes de UI envolvidos
1. TabPanel aba "Aditivos" na ProducaoPage (requer contrato selecionado)
2. Card de resumo: valor_original, total_acrescimos, total_supressoes, limite_25_pct
3. Barra de progresso do percentual consumido (verde/amarelo/vermelho conforme %)
4. ActionButton "Novo Aditivo" com icone Plus para abrir modal
5. Modal de registro: tipo (acrescimo/supressao), valor_aditivo, justificativa, fundamentacao_legal
6. DataTable de aditivos: tipo, valor, justificativa, fundamentacao, data, status
7. Alerta visual quando pct_consumido >= 80% (badge de alerta)

## Endpoints ou servicos envolvidos
1. **GET /api/contratos/{id}/aditivos** — retorna { aditivos: [], resumo: AditivoResumo }
2. **POST /api/contratos/{id}/aditivos** — registra novo aditivo com validacao de limite
3. **AditivoResumo:** valor_original, total_acrescimos, total_supressoes, limite_25_pct, pct_consumido
4. **CRUD /api/crud/contrato-aditivos** — CRUD completo para aditivos
5. **GET /api/contratos/{id}/cronograma** — recarregado ao selecionar contrato
6. **GET /api/dashboard/contratado-realizado** — reflete valores ajustados por aditivos
7. **Auth middleware** — validacao de token JWT

## Dados manipulados
1. **Aditivo:** id, tipo, justificativa, valor_original, valor_aditivo, percentual, data_aditivo, fundamentacao_legal, status
2. **AditivoResumo:** valor_original, total_acrescimos, total_supressoes, limite_25_pct, pct_consumido
3. **Form state:** tipo ("acrescimo"/"supressao"), valor_aditivo (string), justificativa, fundamentacao_legal
4. **Payload POST:** { tipo, valor_aditivo: parseFloat(), justificativa, fundamentacao_legal }
5. **Barra de progresso:** pct_consumido (0-100+) com cores por faixa
6. **Cores da barra:** verde (<50%), amarelo (50-80%), vermelho (>=80%)
7. **Lei referencia:** Art. 124, 125, 126 da Lei 14.133/2021

## Fluxo principal
1. Usuario seleciona contrato e acessa aba "Aditivos" que exibe Card de resumo com valor_original, limite e consumido
2. Sistema exibe barra de progresso mostrando pct_consumido do limite de 25% com cor adequada
3. DataTable lista aditivos existentes com tipo, valor, justificativa, fundamentacao e data
4. Usuario clica em "Novo Aditivo" (icone Plus) que abre Modal com campos tipo, valor, justificativa e fundamentacao
5. Usuario seleciona tipo (acrescimo/supressao), preenche valor e justificativa com fundamentacao legal
6. Usuario clica "Salvar" e frontend envia POST /api/contratos/{id}/aditivos com payload
7. Backend valida limite de 25%, persiste aditivo, recalcula resumo e retorna; frontend atualiza barra e lista

## Fluxos alternativos
1. **FA-01 — Sem contrato selecionado:** Aba exibe mensagem "Selecione um contrato na aba Contratos"
2. **FA-02 — Supressao:** Usuario seleciona tipo "supressao"; valor e subtraido do total
3. **FA-03 — Limite atingido:** pct_consumido >= 100%; sistema exibe alerta de limite esgotado
4. **FA-04 — Alerta 80%:** Barra muda para vermelho ao atingir 80%; badge de alerta visual exibido
5. **FA-05 — Reforma (50%):** Contrato de reforma usa limite de 50% (Art. 125); backend ajusta calculo
6. **FA-06 — Sem aditivos:** Lista vazia; resumo mostra zeros e barra de progresso em 0%
7. **FA-07 — Editar aditivo:** Usuario usa CRUD update para corrigir valor ou justificativa; resumo recalculado

## Fluxos de excecao
1. **FE-01 — Limite excedido:** POST retorna erro indicando que aditivo excede limite de 25%; modal permanece aberto
2. **FE-02 — Token expirado:** Requisicao retorna 401; sistema redireciona para LoginPage
3. **FE-03 — Valor invalido:** parseFloat retorna NaN; valor enviado como 0
4. **FE-04 — Contrato sem valor:** valor_original = 0; calculo de percentual gera divisao por zero; backend trata
5. **FE-05 — Fundamentacao vazia:** Campo obrigatorio nao preenchido; backend pode rejeitar
6. **FE-06 — Erro de rede:** fetch falha; try/catch captura e loga
7. **FE-07 — Concorrencia:** Dois usuarios registram aditivo simultaneo; segundo pode exceder limite
