# Caso de Uso: UC-023

## Nome
Consultar precos historicos PNCP

## Objetivo
Permitir ao usuario consultar o historico de precos praticados em licitacoes anteriores no PNCP para um produto ou item especifico, fornecendo referencia de mercado para embasar a precificacao nas camadas C (referencia) e D (proposta).

## Escopo
Modulo Precificacao — PrecificacaoPage.tsx

## Atores
- Usuario autenticado (Analista Comercial / Precificador)
- Sistema backend (Flask)
- PNCP API (historico de precos)
- Banco de dados

## Gatilho
Usuario clica no botao "Historico PNCP" (icone History) na aba "Custos e Precos" da PrecificacaoPage para um item vinculado, ou sistema consulta automaticamente ao carregar camadas de preco.

## Pre-condicoes
1. Vinculo item-produto confirmado (UC-021 executado)
2. Backend Flask ativo com acesso a PNCP API
3. Produto com descricao ou codigo que permita busca no PNCP
4. Conexao com internet para consulta a API do PNCP
5. PrecificacaoPage na aba "Custos e Precos" com vinculo selecionado
6. Endpoint GET /api/precos-historicos implementado e funcional
7. Camadas de preco em edicao para receber dados de referencia (UC-022)

## Pos-condicoes
1. Lista de precos historicos exibida com produto, preco, data, edital e resultado
2. Preco medio historico calculado e exibido (preco_medio_historico)
3. Preco minimo e maximo historicos identificados (preco_min_historico, preco_max_historico)
4. Quantidade de registros historicos contabilizada (qtd_registros_historico)
5. Camada C (referencia) pre-preenchida com preco_medio_historico como sugestao
6. Dados de historico persistidos no PrecoCamada para consulta futura
7. Grafico de tendencia de precos disponivel para visualizacao

## Regras de negocio
1. **RN-HIS-001:** Historico de precos consulta PNCP por descricao do produto nos ultimos 24 meses
2. **RN-HIS-002:** Preco medio historico = media aritmetica dos precos encontrados
3. **RN-HIS-003:** Resultado de cada registro classificado como "ganho" ou "perdido"
4. **RN-HIS-004:** Precos historicos sao ajustados por inflacao (IPCA) quando periodo > 12 meses
5. **RN-HIS-005:** Minimo de 3 registros para considerar historico estatisticamente relevante
6. **RN-HIS-006:** Precos outliers (> 3 desvios padrao) sao sinalizados mas nao excluidos
7. **RN-HIS-007:** Sistema armazena cache de consultas PNCP por 7 dias para evitar requisicoes redundantes

## Componentes de UI envolvidos
1. Botao "Historico PNCP" (ActionButton com icone History) que dispara a consulta
2. DataTable de historico com colunas: produto, preco (formatado BRL), data, edital, resultado (ganho/perdido)
3. Card de resumo estatistico: preco_medio_historico, preco_min_historico, preco_max_historico, qtd_registros
4. Grafico de barras (BarChart3) com tendencia de precos ao longo do tempo
5. Badge de resultado em cada registro: verde para "ganho", vermelho para "perdido"
6. Indicador de loading (Loader2) durante consulta ao PNCP
7. Botao "Aplicar Media" que copia preco_medio_historico para Camada C (referencia)

## Endpoints ou servicos envolvidos
1. **GET /api/precos-historicos** — consulta historico de precos no PNCP por produto
2. **PNCP API /orgaos/{cnpj}/compras** — dados de licitacoes anteriores com precos
3. **PNCP Search API /api/search/?q=** — busca licitacoes por descricao do produto
4. **crudUpdate('preco_camadas', id)** — atualiza camada C com preco_medio_historico
5. **GET /api/editais/{id}/precificacao** — recarrega dados de precificacao com historico
6. **tool_sugerir_preco** — tool IA que analisa historico e sugere precos otimizados
7. **crudList('historico_precos', { produto_id })** — lista cache de historico local

## Dados manipulados
1. **HistoricoPreco:** id, produto, preco, data, edital, resultado ("ganho"|"perdido")
2. **preco_medio_historico:** number — media dos precos historicos encontrados
3. **preco_min_historico:** number — menor preco encontrado no periodo
4. **preco_max_historico:** number — maior preco encontrado no periodo
5. **qtd_registros_historico:** number — quantidade de registros encontrados
6. **Tendencia:** array de pares (data, preco) para grafico temporal
7. **Cache PNCP:** dados armazenados localmente com TTL de 7 dias

## Fluxo principal
1. Usuario seleciona vinculo item-produto na aba "Custos e Precos" e clica no botao "Historico PNCP" com icone History
2. Sistema dispara GET /api/precos-historicos com descricao do produto e periodo de 24 meses como parametros
3. Backend consulta PNCP Search API por licitacoes com descricao similar e extrai precos praticados
4. Backend calcula estatisticas: preco_medio_historico, preco_min_historico, preco_max_historico e qtd_registros
5. Frontend renderiza DataTable com registros historicos e Card de resumo estatistico com valores calculados
6. Sistema exibe grafico de tendencia (BarChart3) com evolucao de precos ao longo do periodo consultado
7. Usuario clica "Aplicar Media" para copiar preco_medio_historico para Camada C (valor_referencia_edital) automaticamente

## Fluxos alternativos
1. **FA-01 — Cache disponivel:** Consulta recente (< 7 dias) encontrada no cache; sistema exibe dados locais sem nova requisicao
2. **FA-02 — Busca por NCM:** Produto sem descricao suficiente; sistema busca por codigo NCM como alternativa
3. **FA-03 — Filtrar por resultado:** Usuario filtra historico por resultado (apenas "ganho" ou apenas "perdido")
4. **FA-04 — Expandir periodo:** Usuario amplia busca para 36 ou 48 meses quando poucos registros encontrados
5. **FA-05 — Aplicar minimo:** Usuario aplica preco_min_historico em vez do medio para Camada C
6. **FA-06 — Exportar historico:** Usuario exporta dados de historico em CSV para analise externa
7. **FA-07 — Sugestao IA:** tool_sugerir_preco analisa historico e sugere preco otimizado considerando tendencia

## Fluxos de excecao
1. **FE-01 — PNCP indisponivel:** API retorna erro ou timeout; sistema exibe "Historico indisponivel" e sugere cache anterior
2. **FE-02 — Nenhum registro:** PNCP nao retorna resultados; sistema exibe "Sem historico para este produto"
3. **FE-03 — Poucos registros:** Menos de 3 registros encontrados; sistema exibe warning "Amostra insuficiente para media confiavel"
4. **FE-04 — Precos outliers:** Precos com variacao > 3 desvios padrao; sistema sinaliza com icone AlertTriangle
5. **FE-05 — Timeout na busca:** Consulta excede 30s; sistema cancela e exibe dados parciais coletados
6. **FE-06 — Produto generico:** Descricao muito generica retorna resultados imprecisos; sistema alerta e sugere refinar busca
7. **FE-07 — Erro de parsing:** Dados do PNCP em formato inesperado; backend registra erro e retorna lista vazia
