# Caso de Uso: UC-025

## Nome
Definir estrategia competitiva

## Objetivo
Permitir ao usuario consolidar a estrategia competitiva para o edital, reunindo insights de precificacao da IA, beneficios fiscais NCM, comodato de equipamentos e posicionamento estrategico, definindo o plano final de participacao na licitacao.

## Escopo
Modulo Precificacao — PrecificacaoPage.tsx

## Atores
- Usuario autenticado (Analista Comercial / Precificador / Gestor)
- Sistema backend (Flask)
- Motor de IA (DeepSeek)
- Banco de dados

## Gatilho
Usuario acessa a aba "Estrategia" ou "Insights" na PrecificacaoPage apos definir camadas de preco (UC-022) e executar simulacao de disputa (UC-024).

## Pre-condicoes
1. Camadas de preco A-E definidas para todos os itens do edital (UC-022)
2. Simulacao de disputa executada com cenarios avaliados (UC-024, recomendado)
3. Historico de precos PNCP consultado (UC-023, recomendado)
4. Backend Flask ativo com acesso ao DeepSeek para insights
5. Dados de comodato cadastrados quando aplicavel (equipamentos em comodato)
6. Dados de NCM e beneficios fiscais disponiveis para itens vinculados
7. PrecificacaoPage carregada com edital e lotes selecionados

## Pos-condicoes
1. Insights de precificacao gerados pela IA e exibidos ao usuario
2. Beneficios fiscais NCM identificados e quantificados por item
3. Comodato de equipamentos configurado com amortizacao calculada
4. Posicionamento estrategico definido (agressivo, moderado, conservador)
5. Plano final de precos consolidado com lances definidos para todos os itens
6. Dados de estrategia persistidos para referencia na proposta (UC-026)
7. Edital pronto para avancar para etapa de Proposta

## Regras de negocio
1. **RN-EST-001:** Insights de precificacao analisam: margem media, competitividade, riscos e oportunidades
2. **RN-EST-002:** Beneficios fiscais NCM incluem isencao ICMS, reducao IPI e creditos PIS/COFINS
3. **RN-EST-003:** Comodato calcula amortizacao: valor_mensal = valor_equipamento / duracao_meses + custo_manutencao
4. **RN-EST-004:** Custo de comodato e diluido no preco dos reagentes/insumos do lote
5. **RN-EST-005:** Estrategia deve considerar intencao definida na analise de mercado (UC-018): estrategico, defensivo, acompanhamento
6. **RN-EST-006:** IA gera recomendacao textual com pontos fortes e fracos da estrategia
7. **RN-EST-007:** Estrategia aprovada pelo gestor permite avancar para Proposta

## Componentes de UI envolvidos
1. Card de insights IA (icone Lightbulb) com analise textual renderizada em ReactMarkdown
2. Card de beneficios fiscais NCM (icone Shield) com tabela de isencoes por item
3. Card de comodato (icone Package) com nome_equipamento, valor, duracao, amortizacao mensal
4. FormField para posicionamento estrategico: SelectInput com opcoes (agressivo, moderado, conservador)
5. Card de resumo financeiro: custo total, preco total proposto, margem media, margem minima
6. Botao "Gerar Insights" (ActionButton com icone Sparkles) que dispara tool_insights_precificacao
7. Botao "Aprovar Estrategia" que finaliza precificacao e habilita etapa de Proposta

## Endpoints ou servicos envolvidos
1. **tool_insights_precificacao** — tool IA que gera insights estrategicos de precificacao
2. **createSession() + sendMessage()** — sessao IA para geracao de insights via chat
3. **crudList('comodatos', { edital_id })** — lista equipamentos em comodato do edital
4. **crudCreate('comodatos')** — registra novo equipamento em comodato
5. **crudUpdate('comodatos', id)** — atualiza dados de comodato existente
6. **GET /api/editais/{id}/precificacao** — recarrega dados consolidados de precificacao
7. **crudUpdate('editais', id, { status: 'precificado' })** — atualiza status do edital

## Dados manipulados
1. **Insights IA:** string (markdown) — texto com analise estrategica gerada pelo DeepSeek
2. **Beneficios fiscais:** ncm, isencao_icms (boolean), reducao_ipi (%), creditos_pis_cofins (%)
3. **Comodato:** nome_equipamento, valor_equipamento, duracao_meses, valor_mensal_amortizacao, custo_manutencao_mensal
4. **Posicionamento:** "agressivo"|"moderado"|"conservador" — estrategia selecionada
5. **Resumo financeiro:** custo_total, preco_total, margem_media (%), margem_minima (%)
6. **iaResponse:** string — resposta da IA renderizada em ReactMarkdown
7. **iaProcessing:** string | null — indicador de processamento IA em andamento

## Fluxo principal
1. Usuario acessa aba "Estrategia" na PrecificacaoPage com camadas de preco definidas para todos os itens do edital
2. Usuario clica em "Gerar Insights" que dispara tool_insights_precificacao via createSession/sendMessage
3. IA analisa camadas de preco, historico PNCP, simulacao de disputa e perfil do orgao para gerar insights
4. Frontend renderiza insights em Card com ReactMarkdown incluindo pontos fortes, fracos, riscos e oportunidades
5. Sistema exibe Card de beneficios fiscais NCM com isencoes aplicaveis e Card de comodato com amortizacao calculada
6. Usuario seleciona posicionamento estrategico (agressivo/moderado/conservador) e revisa resumo financeiro consolidado
7. Usuario clica "Aprovar Estrategia" que atualiza status do edital para "precificado" e habilita etapa de Proposta

## Fluxos alternativos
1. **FA-01 — Sem comodato:** Edital nao requer equipamento em comodato; Card de comodato oculto
2. **FA-02 — Ajustar apos insights:** Usuario volta para camadas de preco e ajusta lances baseado nos insights IA
3. **FA-03 — Insights via Chat:** Usuario solicita insights pelo FloatingChat; mesma tool e acionada
4. **FA-04 — Adicionar comodato:** Usuario registra novo equipamento em comodato com FormField de valores
5. **FA-05 — Mudar posicionamento:** Usuario altera posicionamento estrategico e sistema recalcula resumo financeiro
6. **FA-06 — Exportar estrategia:** Usuario exporta resumo estrategico em PDF para apresentacao interna
7. **FA-07 — Aprovacao por gestor:** Estrategia requer aprovacao do gestor antes de avancar para Proposta

## Fluxos de excecao
1. **FE-01 — DeepSeek timeout:** Geracao de insights excede 45s; sistema exibe insights parciais ou mensagem de erro
2. **FE-02 — Itens sem camadas:** Nem todos os itens tem camadas definidas; sistema lista itens pendentes
3. **FE-03 — Margem media negativa:** Resumo financeiro com margem negativa; sistema exibe alerta vermelho
4. **FE-04 — Comodato sem amortizacao:** Duracao ou valor do equipamento zerados; sistema impede calculo
5. **FE-05 — Erro de persistencia:** Atualizacao de status falha; frontend exibe toast de erro
6. **FE-06 — Dados de NCM ausentes:** Produto sem NCM cadastrado; beneficios fiscais indisponiveis
7. **FE-07 — Estrategia rejeitada:** Gestor rejeita estrategia; edital volta para ajuste de precificacao

- Secao "Justificativa IA": texto gerado fundamentando a precificacao com base nas atas

### Fluxo pipeline IA (UC-P11) — Sequencia real
1. Usuario seleciona vinculo item-produto na PrecificacaoPage
2. Sistema busca insights salvos no banco para esse vinculo
3. Se nao encontra: dispara pipeline — historico local → atas PNCP → contratos
4. IA gera justificativa com base nos dados coletados
5. Sistema salva insights no banco para reutilizacao futura
6. P11-F01 exibe 5 sugestoes de preco (A-E) com fonte e confianca
7. P11-F02 mostra atas PNCP encontradas, expandiveis
8. P11-F03 e P11-F04 mostram vencedores e concorrentes
9. P11-F05 exibe justificativa gerada pela IA
10. Usuario revisa e clica P11-F06 "Aplicar Sugestoes" para pre-preencher campos A-E
11. Alternativamente, clica P11-F07 "Recalcular" para nova rodada
