# Caso de Uso: UC-022

## Nome
Definir camadas de preco A-E

## Objetivo
Permitir ao usuario definir precos em 5 camadas estrategicas (A=custo, B=markup, C=referencia, D=proposta, E=minimo) para cada item vinculado do edital, considerando tributos NCM, beneficios fiscais, historico de precos e margens desejadas.

## Escopo
Modulo Precificacao — PrecificacaoPage.tsx

## Atores
- Usuario autenticado (Analista Comercial / Precificador)
- Sistema backend (Flask)
- Banco de dados

## Gatilho
Usuario seleciona um vinculo item-produto na aba "Custos e Precos" da PrecificacaoPage e inicia o preenchimento dos campos de preco por camada.

## Pre-condicoes
1. Vinculo item-produto confirmado (UC-021 executado com confirmado = true)
2. Dados do produto disponíveis (custo, NCM, tributos)
3. Backend Flask ativo e banco de dados acessivel
4. Camadas de preco carregadas do banco (se existirem) via GET /api/editais/{id}/precificacao
5. Dados de NCM e tributos do produto cadastrados no portfolio
6. Valor de referencia do edital disponivel (valor_referencia_edital) quando fornecido pelo orgao
7. PrecificacaoPage na aba "Custos e Precos" com vinculo selecionado no SelectInput (vinculoId)

## Pos-condicoes
1. Camada A (custo) definida: custo_unitario, ncm, icms, ipi, pis_cofins, isencao_icms, custo_base_final
2. Camada B (markup) definida: modo_preco_base, markup_percentual, preco_base
3. Camada C (referencia) definida: valor_referencia_edital, target_referencia, margem_sobre_custo
4. Camada D (proposta) definida: lance_inicial com margem planejada
5. Camada E (minimo) definida: lance_minimo, margem_minima (piso de negociacao)
6. Dados persistidos via POST /api/editais/{id}/preco-camada com todas as 5 camadas
7. Status da camada atualizado para "definido" permitindo simulacao de disputa (UC-024)

## Regras de negocio
1. **RN-PRE-001:** 5 camadas de preco obrigatorias: A=custo, B=markup, C=referencia, D=proposta, E=minimo
2. **RN-PRE-002:** Camada A calcula custo_base_final = custo_unitario + impostos (ICMS + IPI + PIS/COFINS) - isencoes
3. **RN-PRE-003:** Camada B calcula preco_base = custo_base_final * (1 + markup_percentual/100)
4. **RN-PRE-004:** Camada C compara preco_base com valor_referencia_edital e calcula margem_sobre_custo
5. **RN-PRE-005:** Camada D define lance_inicial que deve ser <= valor_referencia_edital (quando disponivel)
6. **RN-PRE-006:** Camada E define lance_minimo que deve ser >= custo_base_final (margem_minima >= 0)
7. **RN-PRE-007:** Beneficios fiscais NCM (isencao_icms) reduzem custo_base_final quando aplicaveis

## Componentes de UI envolvidos
1. SelectInput para selecao do vinculo item-produto (vinculoId) a precificar
2. Card "Camada A — Custo" com campos: custo_unitario, ncm, icms, ipi, pis_cofins, isencao_icms, custo_base_final
3. Card "Camada B — Markup" com campos: modo_preco_base (manual/percentual), markup_percentual, preco_base
4. Card "Camada C — Referencia" com campos: valor_referencia_edital, target_referencia, margem_sobre_custo
5. Card "Camada D — Proposta" com campos: lance_inicial e indicador visual de margem planejada
6. Card "Camada E — Minimo" com campos: lance_minimo, margem_minima com alerta se margem < 5%
7. Botao "Salvar Camadas" (ActionButton com icone DollarSign) que persiste todas as 5 camadas

## Endpoints ou servicos envolvidos
1. **POST /api/editais/{id}/preco-camada** — salva/atualiza precos por camada para um vinculo
2. **GET /api/editais/{id}/precificacao** — carrega camadas existentes do banco
3. **crudGet('preco_camadas', vinculo_id)** — busca camada especifica de um vinculo
4. **crudUpdate('preco_camadas', id)** — atualiza campos de uma camada existente
5. **crudList('produtos', { ncm })** — consulta dados tributarios do produto por NCM
6. **GET /api/precos-historicos** — historico PNCP para referencia na camada C (UC-023)
7. **crudCreate('preco_camadas')** — cria registro de camada quando nao existe

## Dados manipulados
1. **PrecoCamada:** estrutura completa com campos das 5 camadas (custo a minimo)
2. **custo_unitario:** number — custo direto do produto (camada A)
3. **NCM e tributos:** ncm (string), icms (%), ipi (%), pis_cofins (%), isencao_icms (boolean)
4. **custo_base_final:** number — custo total com tributos e isencoes aplicadas
5. **markup_percentual:** number — percentual de margem sobre custo (camada B)
6. **lance_inicial:** number — preco proposto para lance inicial (camada D)
7. **lance_minimo:** number — preco minimo aceitavel na disputa (camada E)

## Fluxo principal
1. Usuario seleciona vinculo item-produto no SelectInput e sistema carrega camada existente ou cria nova PrecoCamada
2. Usuario preenche Camada A: custo_unitario do produto, NCM, e sistema calcula tributos (ICMS, IPI, PIS/COFINS) automaticamente
3. Sistema verifica isencao_icms por NCM e calcula custo_base_final = custo + impostos - isencoes exibindo resultado
4. Usuario define Camada B: escolhe modo_preco_base (manual ou percentual) e informa markup_percentual; sistema calcula preco_base
5. Sistema preenche Camada C automaticamente: valor_referencia_edital do item, calcula target_referencia e margem_sobre_custo
6. Usuario define Camada D (lance_inicial) e Camada E (lance_minimo) com alertas visuais se margem_minima < 5%
7. Usuario clica "Salvar Camadas" que dispara POST /api/editais/{id}/preco-camada persistindo todas as 5 camadas

## Fluxos alternativos
1. **FA-01 — Custo automatico:** Sistema pre-preenche custo_unitario do cadastro do produto quando disponivel
2. **FA-02 — NCM com isencao:** Sistema detecta NCM com isencao fiscal e aplica automaticamente isencao_icms = true
3. **FA-03 — Sem referencia do edital:** Valor de referencia nao disponivel; Camada C exibe "Referencia indisponivel"
4. **FA-04 — Copiar camadas:** Usuario copia camadas de outro item do mesmo lote para agilizar preenchimento
5. **FA-05 — Preco sugerido por IA:** tool_sugerir_preco sugere valores para camadas B, D e E baseado em historico
6. **FA-06 — Modo markup manual:** Usuario define preco_base diretamente sem calcular via percentual de markup
7. **FA-07 — Ajuste em lote:** Usuario aplica mesmo markup_percentual a todos os itens do lote simultaneamente

## Fluxos de excecao
1. **FE-01 — Custo zero:** custo_unitario = 0 impede calculo das camadas seguintes; sistema exibe erro inline
2. **FE-02 — Lance acima da referencia:** lance_inicial > valor_referencia_edital; sistema exibe warning amarelo
3. **FE-03 — Lance minimo abaixo do custo:** lance_minimo < custo_base_final; sistema exibe alerta vermelho "Margem negativa"
4. **FE-04 — NCM invalido:** NCM nao encontrado na base de tributos; sistema solicita preenchimento manual de aliquotas
5. **FE-05 — Erro de persistencia:** POST retorna erro 500; frontend exibe toast de erro e mantem dados no formulario
6. **FE-06 — Vinculo nao selecionado:** Campos de camada desabilitados enquanto nenhum vinculo estiver selecionado
7. **FE-07 — Dados inconsistentes:** preco_base < custo_base_final em modo percentual; sistema recalcula automaticamente

## Criterios de validacao
1. Camada A calcula custo_base_final corretamente com tributos e isencoes
2. Camada B calcula preco_base = custo_base_final * (1 + markup/100) corretamente
3. Camada C exibe margem_sobre_custo coerente com valor de referencia
4. Camada D valida lance_inicial <= valor_referencia_edital quando disponivel
5. Camada E valida lance_minimo >= custo_base_final (margem nao negativa)
6. Beneficios fiscais NCM reduzem custo_base_final quando aplicaveis
7. Todas as 5 camadas sao persistidas corretamente no banco
