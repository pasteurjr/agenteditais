# Caso de Uso: UC-056

## Nome
Configurar alertas multi-tier

## Objetivo
Exibir e gerenciar alertas de vencimento multi-tier na ContratadoRealizadoPage, consolidando proximos vencimentos de contratos, ARPs e entregas com classificacao por urgencia (vermelho/laranja/amarelo/verde) e regras configuradas por tipo de entidade.

## Escopo
Modulo Dashboard CR — ContratadoRealizadoPage.tsx > Secao Alertas/Vencimentos

## Atores
- Usuario autenticado (Gestor de Contratos)
- Sistema backend (Flask)
- Scheduler de alertas (verificacao periodica)
- Servico de notificacoes (email/push)

## Gatilho
loadAlertas() dispara automaticamente via useEffect ao montar ContratadoRealizadoPage, consultando vencimentos consolidados.

## Pre-condicoes
1. Usuario esta autenticado no sistema
2. Backend Flask ativo com endpoint GET /api/alertas-vencimento/consolidado
3. Contratos, ARPs ou entregas cadastrados com datas de vencimento
4. Regras de alerta configuradas (UC-043) por tipo de entidade
5. ContratadoRealizadoPage acessivel no menu
6. Endpoint de alertas retorna vencimentos com resumo
7. Dados de vencimento atualizados no banco

## Pos-condicoes
1. Vencimentos consolidados exibidos com tipo_entidade, nome, data_vencimento, dias_restantes, urgencia
2. Resumo de alertas: total, vermelho, laranja, amarelo, verde
3. Indicadores visuais de urgencia com cores por faixa de dias
4. Valor associado a cada vencimento (quando disponivel)
5. Dados sincronizados com alertas da FollowupPage (UC-043)
6. Alertas automaticos disparados conforme regras configuradas
7. Interface carregada automaticamente ao acessar pagina

## Regras de negocio
1. **RN-ALE-002:** Urgencia: vermelho (<=7d), laranja (8-15d), amarelo (16-30d), verde (>30d)
2. **RN-ALE-003:** Tipos de entidade: contrato, arp, entrega
3. **RN-ALE-CONS-001:** Consolidado agrupa todos os tipos em lista unica
4. **RN-ALE-CONS-002:** Resumo contabiliza total e quantidade por faixa de urgencia
5. **RN-ALE-CONS-003:** Vencimentos ordenados por dias_restantes (mais urgente primeiro)
6. **RN-ALE-CONS-004:** Entidade com dias_restantes negativo = ja vencido, urgencia "vermelho"
7. **RN-ALE-CONS-005:** Dados compartilhados entre FollowupPage e ContratadoRealizadoPage

## Componentes de UI envolvidos
1. StatCards de resumo de alertas: total, vermelhos (criticos), laranjas (urgentes), amarelos (atencao)
2. DataTable de vencimentos: tipo_entidade, nome, data_vencimento, dias_restantes, urgencia, valor
3. Badge de urgencia colorido: vermelho, laranja, amarelo, verde
4. Icones Calendar e AlertTriangle para indicar vencimentos
5. Loader2 durante carregamento de alertas (alertasLoading)
6. Formatacao de data formatDate() e moeda formatCurrency()
7. Card wrapper para secao de alertas com titulo descritivo

## Endpoints ou servicos envolvidos
1. **GET /api/alertas-vencimento/consolidado** — retorna { vencimentos: VencimentoRow[], resumo: AlertasResumo }
2. **GET /api/alertas-vencimento/regras** — regras configuradas por tipo (acessivel na FollowupPage)
3. **AlertasResponse:** vencimentos[], resumo { total, vermelho, laranja, amarelo, verde }, erro?
4. **GET /api/contratado-realizado/vencimentos** — endpoint alternativo de vencimentos
5. **Scheduler** — verifica e dispara alertas nos tiers configurados
6. **Auth middleware** — validacao via authHeaders()
7. **Servico de notificacoes** — canais email/push conforme regras

## Dados manipulados
1. **VencimentoRow:** tipo_entidade ("contrato"|"arp"|"entrega"), nome, data_vencimento, dias_restantes, valor, entity_id, urgencia
2. **AlertasResumo:** total (number), vermelho (count), laranja (count), amarelo (count), verde (count)
3. **AlertasResponse:** vencimentos (VencimentoRow[]), resumo (AlertasResumo), erro? (string)
4. **Urgencia:** "vermelho" (<=7d) | "laranja" (8-15d) | "amarelo" (16-30d) | "verde" (>30d)
5. **Estado UI:** alertas (AlertasResponse | null), alertasLoading, alertasError
6. **Derivados:** vencimentos = alertas?.vencimentos ?? [], resumoAlertas = alertas?.resumo
7. **Filtros:** periodo e orgao do dashboard podem afetar vencimentos

## Fluxo principal
1. ContratadoRealizadoPage monta e useEffect dispara loadAlertas() via useCallback
2. Frontend envia GET /api/alertas-vencimento/consolidado com authHeaders() e seta alertasLoading=true
3. Backend consolida vencimentos de contratos, ARPs e entregas, calcula dias_restantes e urgencia
4. Backend gera resumo com total e contagens por faixa de urgencia (vermelho/laranja/amarelo/verde)
5. Frontend recebe AlertasResponse, atualiza states alertas e extrai vencimentos e resumoAlertas
6. StatCards de resumo exibem total e contagens por urgencia com cores correspondentes
7. DataTable de vencimentos lista todos os itens com tipo_entidade, nome, data, dias_restantes, urgencia e valor

## Fluxos alternativos
1. **FA-01 — Sem vencimentos:** Lista vazia; StatCards zeros; mensagem positiva "Nenhum vencimento proximo"
2. **FA-02 — Apenas vermelhos:** Todos vencimentos criticos (<=7d); indicador de risco maximo
3. **FA-03 — Vencimento ja expirado:** dias_restantes negativo; urgencia "vermelho" com indicacao de vencido
4. **FA-04 — Filtrar por tipo:** DataTable pode filtrar por tipo_entidade (contrato/arp/entrega)
5. **FA-05 — Enviar para Chat:** Usuario usa onSendToChat para analisar vencimentos com IA
6. **FA-06 — Atualizar manualmente:** Botao de refresh recarrega alertas
7. **FA-07 — Mesmos dados na FollowupPage:** Dados de vencimento identicos aos da aba Alertas (UC-043)

## Fluxos de excecao
1. **FE-01 — Erro no consolidado:** GET falha; alertasError setado; interface exibe mensagem de erro
2. **FE-02 — Token expirado:** Requisicao retorna 401; sistema redireciona para LoginPage
3. **FE-03 — Resposta com erro:** data.erro presente; throw new Error(data.erro)
4. **FE-04 — Dados de data invalidos:** data_vencimento invalida; formatDate retorna string crua
5. **FE-05 — Valor nulo:** valor null; formatCurrency trata como "—"
6. **FE-06 — Erro de rede:** fetch falha; catch seta alertasError
7. **FE-07 — Backend lento:** alertasLoading=true; Loader2 exibido ate resposta

### Sequencia de eventos real (UC-CR03)
1. Usuario acessa ContratadoRealizadoPage e clica na aba "Vencimentos"
2. Sistema calcula vencimentos proximos (30 dias a frente) de todas as categorias
3. CR03-F01 a CR03-F03 exibem contadores por faixa de urgencia
4. Vencimentos agrupados: CR03-F04 Contratos, CR03-F05 ARPs, CR03-F06 Garantias, CR03-F07 Entregas
5. Badge de urgencia: vermelho (<7d), amarelo (7-15d), verde (15-30d)
6. Itens ordenados por dias restantes (mais urgente primeiro) dentro de cada secao
7. CR03-F09 tabela mostra configuracao atual de regras por tipo
8. Usuario clica "Configurar Regras de Alerta" — modal abre
9. Usuario seleciona CR03-F10 tipo e define canais por limiar CR03-F11 a CR03-F14
10. Escalacao progressiva: 30d=Sistema → 15d=E-mail → 7d=E-mail+WhatsApp → 1d=Push+Telefone
11. CR03-F15 escalacao automatica: CR03-F16 e-mail gerencial obrigatorio
12. CR03-F17 Salvar — regras persistidas; scheduler monitora e dispara conforme regras
