# Caso de Uso: UC-055

## Nome
Monitorar pedidos em atraso

## Objetivo
Permitir ao usuario monitorar entregas atrasadas de contratos com classificacao por severidade (HIGH/MEDIUM/LOW), valor em risco e indicadores visuais de urgencia, possibilitando acao corretiva proativa.

## Escopo
Modulo Dashboard CR — ContratadoRealizadoPage.tsx > Secao Atrasos

## Atores
- Usuario autenticado (Gestor de Contratos)
- Sistema backend (Flask)
- Modulo de entregas (fonte de dados de atraso)
- Modulo de notificacoes (alertas de atraso)

## Gatilho
Dados de atrasos carregados automaticamente como parte do dashboard Contratado x Realizado (UC-054) via loadDashboard().

## Pre-condicoes
1. Usuario esta autenticado no sistema
2. ContratadoRealizadoPage acessada com loadDashboard() executado
3. Ao menos uma entrega com data_prevista < hoje e status != "entregue"
4. Backend calcula atrasos como parte do DashboardResponse
5. Severidade calculada no backend (HIGH/MEDIUM/LOW)
6. Valor de cada entrega disponivel para calculo de risco
7. Dados de entregas atualizados no banco

## Pos-condicoes
1. Lista de atrasos exibida com severidade, contrato, entrega, dias de atraso e valor
2. Contadores derivados: totalAtrasados, altaSeveridade, valorEmRisco
3. Atrasos agrupados por severidade: atrasosHigh, atrasosMedium, atrasosLow
4. Indicadores visuais de urgencia com cores por severidade
5. Valor em risco calculado como soma dos valores de entregas atrasadas
6. Dados atualizados a cada recarga do dashboard
7. Informacoes disponiveis para analise via Chat IA

## Regras de negocio
1. **RN-CR-002:** Severidade: HIGH = >30 dias de atraso, MEDIUM = 15-30 dias, LOW = <15 dias
2. **RN-ATR-001:** dias_atraso = hoje - data_prevista (em dias)
3. **RN-ATR-002:** Atraso somente para entregas com status != "entregue" e data_prevista < hoje
4. **RN-ATR-003:** valorEmRisco = soma de valor de todas as entregas atrasadas
5. **RN-ATR-004:** totalAtrasados = atrasos.length (contagem total)
6. **RN-ATR-005:** altaSeveridade = atrasosHigh.length (contagem HIGH)
7. **RN-ATR-006:** Severidade e um enum restrito: "HIGH" | "MEDIUM" | "LOW"

## Componentes de UI envolvidos
1. StatCards de atraso: Total Atrasados, Alta Severidade, Valor em Risco
2. DataTable de atrasos: contrato_numero, orgao, entrega, data_prevista, dias_atraso, severidade, valor
3. Badge de severidade: HIGH (vermelho), MEDIUM (laranja), LOW (amarelo)
4. Icones AlertTriangle e TrendingDown para indicar riscos
5. Formatacao monetaria formatCurrency() para valores
6. Formatacao de data formatDate() para datas previstas
7. Agrupamento visual por severidade (HIGH primeiro)

## Endpoints ou servicos envolvidos
1. **GET /api/dashboard/contratado-realizado** — retorna atrasos[] dentro do DashboardResponse
2. **GET /api/contratado-realizado/atrasos** — endpoint dedicado de atrasos
3. **DashboardResponse.atrasos:** array de AtrasoRow com severidade calculada
4. **crudList("contrato-entregas")** — fonte de dados de entregas
5. **GET /api/contratos/{id}/cronograma** — cronograma com atrasados (fonte alternativa)
6. **Auth middleware** — validacao de token
7. **Scheduler** — monitoramento automatico de atrasos

## Dados manipulados
1. **AtrasoRow:** contrato_id, contrato_numero, orgao, entrega, data_prevista, dias_atraso, severidade, valor
2. **Severidade:** "HIGH" (>30d) | "MEDIUM" (15-30d) | "LOW" (<15d)
3. **Derivados:** totalAtrasados (count), altaSeveridade (count HIGH), valorEmRisco (soma valores)
4. **Agrupamento:** atrasosHigh = filter(HIGH), atrasosMedium = filter(MEDIUM), atrasosLow = filter(LOW)
5. **Formatacao:** formatCurrency(valor) para exibicao monetaria
6. **Formatacao:** formatDate(data_prevista) para exibicao de data
7. **Contadores:** atrasos.length, atrasosHigh.length

## Fluxo principal
1. loadDashboard() do ContratadoRealizadoPage retorna DashboardResponse contendo array de atrasos
2. Frontend extrai atrasos do dashboard e calcula derivados: totalAtrasados, altaSeveridade, valorEmRisco
3. Frontend agrupa atrasos por severidade: atrasosHigh, atrasosMedium, atrasosLow usando filter
4. StatCards de atraso exibem: Total Atrasados, Alta Severidade (HIGH), Valor em Risco (R$)
5. DataTable de atrasos lista todos os atrasos com contrato, orgao, entrega, data_prevista, dias_atraso
6. Badge de severidade exibe cor por nivel: HIGH (vermelho), MEDIUM (laranja), LOW (amarelo)
7. Usuario identifica entregas criticas (HIGH) e toma acoes corretivas

## Fluxos alternativos
1. **FA-01 — Sem atrasos:** Array de atrasos vazio; StatCards mostram zeros; DataTable exibe mensagem positiva
2. **FA-02 — Apenas LOW:** Todos atrasos com < 15 dias; altaSeveridade = 0; situacao controlada
3. **FA-03 — Todos HIGH:** Atrasos criticos em todos; indicador de saude muda para "critico"
4. **FA-04 — Filtrar por periodo:** Filtro de periodo do dashboard afeta quais atrasos sao exibidos
5. **FA-05 — Atraso corrigido:** Entrega realizada remove item da lista de atrasos no proximo carregamento
6. **FA-06 — Enviar para Chat:** Usuario clica em onSendToChat para analisar atrasos com IA
7. **FA-07 — Atraso com valor zero:** Entrega sem valor; valorEmRisco nao afetado; atraso listado normalmente

## Fluxos de excecao
1. **FE-01 — Erro no dashboard:** loadDashboard() falha; atrasos ficam como array vazio
2. **FE-02 — Token expirado:** Requisicao retorna 401; sistema redireciona para LoginPage
3. **FE-03 — Dados de data invalidos:** data_prevista invalida; dias_atraso pode ser NaN; backend trata
4. **FE-04 — Valor nulo:** valor null em atraso; reduce trata com (a.valor || 0)
5. **FE-05 — Severidade desconhecida:** Valor fora de HIGH/MEDIUM/LOW; tratado como padrao
6. **FE-06 — Erro de rede:** fetch falha; dashError setado
7. **FE-07 — Muitos atrasos:** Lista longa; DataTable com paginacao ou scroll
