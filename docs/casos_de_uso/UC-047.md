# Caso de Uso: UC-047

## Nome
Dashboard de atas consultadas

## Objetivo
Exibir dashboard com todas as atas de registro de preco salvas pelo usuario, separando atas vigentes e vencidas, com estatisticas de acompanhamento e indicadores visuais de prazo de vigencia.

## Escopo
Modulo Atas — AtasPage.tsx > Aba Minhas Atas

## Atores
- Usuario autenticado (Analista Comercial)
- Sistema backend (Flask)
- Scheduler de alertas (monitoramento de vigencia)

## Gatilho
Usuario acessa a aba "Minhas Atas" na AtasPage, que dispara fetchMinhasAtas() automaticamente via useEffect.

## Pre-condicoes
1. Usuario esta autenticado no sistema
2. Backend Flask ativo com endpoint GET /api/atas/minhas
3. Ao menos uma ata salva previamente via busca (UC-045)
4. Banco de dados com tabela de atas consultadas
5. Datas de vigencia preenchidas nas atas salvas
6. Endpoint retorna atas com stats consolidados
7. AtasPage acessivel no menu do sistema

## Pos-condicoes
1. Lista de atas salvas exibida na DataTable com titulo, orgao, UF e indicador de vigencia
2. Stats consolidados exibidos: total, vigentes, vencidas
3. Badges visuais indicam dias restantes de vigencia ou dias vencidos
4. Atas vigentes destacadas em verde, vencidas em vermelho
5. Dashboard atualizado a cada acesso a aba
6. Dados disponiveis para controle de saldo ARP (UC-053)
7. Loading indicator exibido durante carregamento

## Regras de negocio
1. **RN-DASH-ATA-001:** Dashboard separa atas vigentes (data_vigencia_fim > hoje) e vencidas (data_vigencia_fim <= hoje)
2. **RN-DASH-ATA-002:** Badge de vigencia calcula dias restantes: verde se vigente, vermelho se vencida
3. **RN-DASH-ATA-003:** Stats retornam total, vigentes e vencidas como contadores inteiros
4. **RN-DASH-ATA-004:** Calculo de dias: Math.ceil((data_vigencia_fim - Date.now()) / 86400000)
5. **RN-DASH-ATA-005:** Resposta flexivel: data.atas para lista e data.stats para estatisticas
6. **RN-DASH-ATA-006:** DataTable exibe todas as atas (vigentes + vencidas) com ordenacao padrao
7. **RN-DASH-ATA-007:** Dashboard carrega automaticamente no mount via useEffect + useCallback

## Componentes de UI envolvidos
1. StatCards com 3 metricas: Total, Vigentes (verde), Vencidas (vermelho)
2. DataTable de atas com colunas: Titulo, Orgao, UF, Vigencia (badge)
3. Badge de vigencia colorido: verde "Xd restantes" ou vermelho "Vencida ha Xd"
4. Loader2 indicador de carregamento durante fetch
5. Card wrapper para cada secao do dashboard
6. Icone BarChart2 na aba do TabPanel
7. Formatacao de datas em pt-BR (toLocaleDateString)

## Endpoints ou servicos envolvidos
1. **GET /api/atas/minhas** — retorna lista de atas salvas com stats consolidados
2. **CRUD /api/crud/atas-consultadas** — CRUD completo para gestao de atas
3. **GET /api/atas/{id}/saldos** — saldos por item (linkado ao Saldo ARP)
4. **POST /api/atas/salvar** — endpoint que alimenta o dashboard (origem dos dados)
5. **Auth middleware** — validacao de token JWT
6. **Scheduler** — monitoramento de vigencia para alertas automaticos
7. **GET /api/alertas-vencimento/consolidado** — vencimentos de atas incluidos

## Dados manipulados
1. **AtaSalva:** id, titulo, orgao, uf, url_pncp, data_publicacao, data_vigencia_inicio, data_vigencia_fim, created_at
2. **AtasStats:** total (number), vigentes (number), vencidas (number)
3. **Dias restantes:** calculado como Math.ceil((fim.getTime() - Date.now()) / 86400000)
4. **Vigencia:** boolean (dias > 0) para determinar cor do badge
5. **Resposta API:** { atas: AtaSalva[], stats: { total, vigentes, vencidas } }
6. **Estado UI:** minhasAtas (AtaSalva[]), atasStats, loadingAtas (boolean)
7. **Badge texto:** vigente = "{dias}d restantes" | vencida = "Vencida ha {abs(dias)}d"

## Fluxo principal
1. Usuario acessa AtasPage e clica na aba "Minhas Atas" que dispara fetchMinhasAtas() via useCallback
2. Frontend envia GET /api/atas/minhas com headers de autenticacao e seta loadingAtas=true
3. Backend consulta tabela de atas consultadas, calcula stats (total, vigentes, vencidas) e retorna JSON
4. Frontend recebe data.atas e data.stats, atualiza states minhasAtas e atasStats
5. StatCards renderizam com total de atas, vigentes em verde e vencidas em vermelho
6. DataTable exibe lista com titulo, orgao, UF e badge de vigencia calculado em tempo real
7. Badge colorido exibe dias restantes (verde) ou dias vencidos (vermelho) baseado em data_vigencia_fim

## Fluxos alternativos
1. **FA-01 — Sem atas salvas:** Lista vazia; StatCards mostram zeros; mensagem informativa exibida
2. **FA-02 — Todas vencidas:** Stats mostram 0 vigentes; todos os badges em vermelho
3. **FA-03 — Ata sem data de vigencia:** Badge exibe "—" em vez de dias restantes
4. **FA-04 — Ata recém-salva:** Ata aparece imediatamente apos fetchMinhasAtas() ser chamado
5. **FA-05 — Navegar para Saldo ARP:** Usuario seleciona ata e navega para aba "Saldo ARP" para ver detalhes
6. **FA-06 — Excluir ata:** Usuario remove ata da lista via CRUD; dashboard atualiza
7. **FA-07 — Ordenar por vigencia:** DataTable permite ordenacao por coluna de vigencia

## Fluxos de excecao
1. **FE-01 — Erro ao carregar atas:** GET /api/atas/minhas falha; try/catch loga erro e loadingAtas=false
2. **FE-02 — Token expirado:** Requisicao retorna 401; sistema redireciona para LoginPage
3. **FE-03 — Resposta sem stats:** data.stats undefined; fallback { total: 0, vigentes: 0, vencidas: 0 }
4. **FE-04 — Data invalida:** data_vigencia_fim com formato invalido; calculo de dias retorna NaN
5. **FE-05 — Backend lento:** Carregamento demora; Loader2 exibido ate resposta chegar
6. **FE-06 — Erro de rede:** fetch falha por conexao; console.error registra e loading para
7. **FE-07 — Dados corrompidos:** Ata sem titulo ou orgao; DataTable exibe celulas vazias
