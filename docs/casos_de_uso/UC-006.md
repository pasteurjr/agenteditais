# Caso de Uso: UC-006

## Nome
Cadastro de Produto

## Objetivo
Permitir que o usuario cadastre um novo produto no portfolio da empresa, informando dados basicos (nome, NCM, fabricante, modelo) e organizando-o por area, classe e subclasse, com possibilidade de cadastro assistido por IA a partir de documentos.

## Escopo
Modulo Portfolio — PortfolioPage.tsx / Backend CRUD produtos + tools de IA

## Atores
- Ator primario: Usuario autenticado (gestor de portfolio)
- Ator secundario: IA DeepSeek (cadastro assistido a partir de documentos)

## Gatilho
O usuario acessa o menu "Portfolio" e inicia o cadastro de um novo produto (manual ou via upload assistido por IA).

## Pre-condicoes
1. O usuario esta autenticado com JWT valido
2. Existe pelo menos uma empresa cadastrada vinculada ao usuario
3. O sistema (frontend e backend) esta operacional
4. Para cadastro via IA: DeepSeek esta configurado e acessivel

## Pos-condicoes
1. O produto esta persistido na tabela "produtos"
2. O produto possui dados basicos preenchidos (nome, fabricante, modelo)
3. O produto esta vinculado a empresa do usuario
4. Se cadastrado via IA, especificacoes tecnicas foram extraidas automaticamente
5. Score de completude do produto e calculado

## Regras de negocio
- RN-PROD-001: Produtos sao scoped por empresa do usuario
- RN-PROD-002: Produtos podem ser cadastrados manualmente ou via IA (upload de documento)
- RN-PROD-003: O cadastro via IA aceita 6 tipos de fonte: Manual Tecnico, Instrucoes de Uso/IFU, Nota Fiscal, Plano de Contas/ERP, Folder/Catalogo, Website do Fabricante
- RN-PROD-004: A IA (DeepSeek) extrai especificacoes tecnicas automaticamente do documento
- RN-PROD-005: Cada produto possui score de completude calculado por tool_completude_produto
- RN-PROD-006: Produtos sao organizados em hierarquia: Area → Classe → Subclasse
- RN-PROD-007: Metadados podem ser reprocessados a qualquer momento via tool_reprocessar_metadados

## Componentes de UI envolvidos
- PortfolioPage com 3 abas: "Produtos", "Cadastro IA", "Classificacao"
- Aba "Produtos":
  - DataTable com listagem de produtos (filtros e busca)
  - Colunas: nome, fabricante, modelo, classe, score de completude, acoes
  - FilterBar com campo de busca textual
  - ScoreBar para exibir completude visual
  - ActionButton com icone Eye para ver detalhes
  - ActionButton com icone Edit2 para editar
  - ActionButton com icone Trash2 para excluir
  - ActionButton com icone RefreshCw para reprocessar metadados
- Aba "Cadastro IA":
  - Lista de UPLOAD_TYPES (6 opcoes) com botao de upload para cada
  - Icone Sparkles para indicar funcionalidade de IA
  - Campo de URL para "Website do Fabricante"
- Aba "Classificacao":
  - Hierarquia de Areas, Classes e Subclasses
  - Icones FolderOpen, ChevronDown, ChevronRight para arvore hierarquica
- Modal de detalhes do produto com especificacoes tecnicas
- Modal de edicao com campos editaveis

## Endpoints ou servicos envolvidos
- GET /api/produtos — lista produtos com filtros e paginacao
- GET /api/produtos/{id} — detalhe do produto com especificacoes
- GET /api/produtos/{id}/completude — score de completude
- POST /api/produtos/{id}/reprocessar — reprocessa metadados com IA
- CRUD /api/crud/produtos — CRUD basico de produtos
- CRUD /api/crud/produtos-especificacoes — especificacoes tecnicas
- CRUD /api/crud/produtos-documentos — documentos do produto
- CRUD /api/crud/areas-produto — areas da hierarquia
- CRUD /api/crud/classes-produto-v2 — classes da hierarquia
- CRUD /api/crud/subclasses-produto — subclasses da hierarquia
- tool_cadastrar_produto — cadastra produto a partir de documento via IA
- tool_extrair_especificacoes — extrai especificacoes de documento
- tool_reprocessar_metadados — reprocessa metadados existentes
- tool_completude_produto — calcula score de completude

## Dados manipulados
- Entrada (manual): nome (string), fabricante (string), modelo (string), ncm (string), area_id, classe_id, subclasse_id
- Entrada (IA): arquivo (File) ou URL (string), tipo de fonte (string), prompt de instrucao
- Saida: objeto Produto com id, nome, fabricante, modelo, ncm, especificacoes tecnicas, score de completude
- Armazenamento: tabela "produtos" + "produtos_especificacoes" + "produtos_documentos" no MySQL

## Fluxo principal

### Passo 1: Acesso a pagina Portfolio
- **Acao:** O ator clica no menu "Portfolio" na barra de navegacao lateral
- **Elemento UI:** Item de menu com icone Briefcase e texto "Portfolio"
- **Resposta visual:** O sistema renderiza a PortfolioPage com a aba "Produtos" ativa por padrao; exibe DataTable com produtos existentes, FilterBar para busca, e ScoreBar em cada linha indicando completude
- **Efeito funcional:** GET /api/produtos e invocado para carregar lista de produtos
- **Requisicao:** GET /api/produtos com header Authorization: Bearer {token}
- **Persistencia:** Nenhuma (apenas leitura)
- **Evidencia:** Screenshot da pagina Portfolio (runtime/screenshots/UC-006/01_portfolio.png)

### Passo 2: Iniciar cadastro manual de produto
- **Acao:** O ator clica no botao "Novo Produto" ou "+"
- **Elemento UI:** ActionButton com icone Plus
- **Resposta visual:** O sistema abre modal ou formulario com campos vazios: Nome, Fabricante, Modelo, NCM, Area, Classe, Subclasse
- **Efeito funcional:** Estado de criacao e ativado
- **Requisicao:** Nenhuma
- **Persistencia:** Nenhuma
- **Evidencia:** Screenshot do formulario vazio (runtime/screenshots/UC-006/02_formulario_novo.png)

### Passo 3: Preenchimento dos dados basicos
- **Acao:** O ator preenche nome do produto, fabricante, modelo e NCM
- **Elemento UI:** TextInput para cada campo (Nome, Fabricante, Modelo, NCM)
- **Resposta visual:** Cada campo exibe o valor digitado
- **Efeito funcional:** Estado local do formulario e atualizado
- **Requisicao:** Nenhuma
- **Persistencia:** Nenhuma
- **Evidencia:** Screenshot do formulario preenchido (runtime/screenshots/UC-006/03_dados_basicos.png)

### Passo 4: Selecao de classificacao (opcional)
- **Acao:** O ator seleciona Area, Classe e Subclasse nos dropdowns correspondentes
- **Elemento UI:** SelectInput para Area, Classe e Subclasse (hierarquicos — a selecao de Area filtra Classes; a selecao de Classe filtra Subclasses)
- **Resposta visual:** Dropdowns exibem opcoes selecionadas
- **Efeito funcional:** Estado local atualizado; dropdowns inferiores filtrados conforme selecao superior
- **Requisicao:** Pode haver chamadas para carregar classes e subclasses baseadas na area selecionada
- **Persistencia:** Nenhuma
- **Evidencia:** Screenshot com classificacao selecionada (runtime/screenshots/UC-006/04_classificacao.png)

### Passo 5: Salvar produto
- **Acao:** O ator clica no botao "Salvar"
- **Elemento UI:** ActionButton "Salvar" com icone Save
- **Resposta visual:** O modal fecha; a listagem de produtos e atualizada com o novo produto; ScoreBar do novo produto indica completude inicial (provavelmente baixa sem especificacoes)
- **Efeito funcional:** Os dados sao enviados ao backend para criacao do produto
- **Requisicao:** POST /api/crud/produtos com body { nome, fabricante, modelo, ncm, area_id, classe_id, subclasse_id, empresa_id }
- **Persistencia:** Registro criado na tabela "produtos" com id gerado
- **Evidencia:** Screenshot apos salvar (runtime/screenshots/UC-006/05_salvo.png)

### Passo 6: Verificacao na listagem
- **Acao:** O ator observa o produto na tabela
- **Elemento UI:** DataTable com nova linha contendo nome, fabricante, modelo, ScoreBar e acoes
- **Resposta visual:** O produto recem-cadastrado aparece na tabela; ScoreBar exibe a completude calculada
- **Efeito funcional:** GET /api/produtos/{id}/completude pode ser chamado para calcular score
- **Requisicao:** GET /api/produtos e/ou GET /api/produtos/{id}/completude
- **Persistencia:** Nenhuma
- **Evidencia:** Screenshot da listagem com produto (runtime/screenshots/UC-006/06_listagem.png)

## Fluxos alternativos

### FA-001: Cadastro assistido por IA (upload de documento)
1. O ator clica na aba "Cadastro IA"
2. O sistema exibe 6 opcoes de tipo de fonte (Manual Tecnico, IFU, NF, Plano de Contas, Folder, Website)
3. O ator seleciona o tipo de fonte desejado
4. O ator faz upload do arquivo correspondente (ou informa URL para Website)
5. O sistema envia o arquivo para a IA (DeepSeek) via tool_cadastrar_produto
6. A IA extrai nome, fabricante, modelo, NCM e especificacoes tecnicas do documento
7. O produto e criado automaticamente com os dados extraidos
8. O ator verifica os dados na aba "Produtos" e pode editar se necessario

### FA-002: Cadastro em lote via Nota Fiscal
1. O ator seleciona tipo "Nota Fiscal (NFS)" na aba "Cadastro IA"
2. O ator faz upload de arquivo PDF ou planilha da NF
3. A IA extrai TODOS os itens/produtos da nota fiscal
4. Cada item e cadastrado como produto individual
5. A listagem e atualizada com todos os novos produtos

### FA-003: Cadastro via Website do Fabricante
1. O ator seleciona tipo "Website do Fabricante"
2. O ator informa a URL do website
3. A IA acessa o website e extrai informacoes dos produtos
4. Produtos sao cadastrados com dados obtidos do site

### FA-004: Edicao de produto existente
1. O ator clica no botao Edit2 de um produto na listagem
2. O sistema carrega os dados do produto no formulario de edicao
3. O ator altera os campos desejados
4. O ator clica em "Salvar"
5. PUT /api/crud/produtos/{id} e invocado
6. Listagem e atualizada

### FA-005: Reprocessar metadados com IA
1. O ator clica no botao RefreshCw de um produto
2. O sistema aciona tool_reprocessar_metadados
3. A IA reanalisa os documentos do produto e atualiza especificacoes
4. Score de completude e recalculado

## Fluxos de excecao

### FE-001: Nome do produto vazio
1. O ator tenta salvar sem preencher o nome do produto
2. O sistema exibe mensagem de validacao
3. O produto nao e salvo

### FE-002: Erro no processamento de IA
1. O ator faz upload de documento mas a IA (DeepSeek) nao consegue extrair dados
2. O sistema exibe mensagem de erro com detalhes do problema
3. O ator pode tentar com outro documento ou cadastrar manualmente

### FE-003: Formato de arquivo nao suportado
1. O ator tenta upload de arquivo com formato nao aceito pelo tipo selecionado
2. O sistema exibe mensagem indicando formatos validos (ex: ".pdf,.doc,.docx" para Manual Tecnico)
3. O upload nao e processado

### FE-004: Erro de rede
1. O ator clica em "Salvar" mas o backend nao responde
2. O sistema exibe mensagem de erro
3. Os dados permanecem no formulario
