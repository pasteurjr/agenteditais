# Caso de Uso: UC-006

## Nome
Listar, Filtrar e Inspecionar Produtos

## Objetivo
Permitir que o usuario liste, filtre e inspecione os produtos do portfolio da empresa, com visualizacao de detalhe completo (classificacao, NCM, preco de referencia, status pipeline, ANVISA, especificacoes e metadados de captacao) e acesso a acoes de manutencao.

## Escopo
Modulo Portfolio — PortfolioPage.tsx > aba "Meus Produtos" / Backend CRUD produtos + endpoints de detalhe, completude e metadados

## Atores
- Ator primario: Usuario autenticado (gestor de portfolio)
- Ator secundario: IA DeepSeek (cadastro assistido a partir de documentos)

## Gatilho
O usuario acessa o menu "Portfolio" e inicia o cadastro de um novo produto (manual ou via upload assistido por IA).

## Pre-condicoes
1. O usuario esta autenticado com JWT valido
2. Produtos cadastrados no sistema
3. Hierarquia Area -> Classe -> Subclasse disponivel
4. O sistema (frontend e backend) esta operacional

## Pos-condicoes
1. Usuario consegue localizar produtos por texto e classificacao
2. Usuario consegue abrir detalhe enriquecido do produto
3. Usuario tem acesso as acoes de manutencao (Editar, Reprocessar IA, Verificar Completude, Excluir)

## Regras de negocio
- RN-PROD-001: Produtos sao scoped por empresa do usuario
- RN-PROD-006: Produtos sao organizados em hierarquia: Area -> Classe -> Subclasse
- RN-PROD-007: Metadados podem ser reprocessados a qualquer momento via endpoint proprio
- RN-PROD-008: O detalhe do produto exibe classificacao, NCM, preco de referencia, status pipeline, ANVISA, especificacoes e metadados de captacao

## Componentes de UI envolvidos
- PortfolioPage com 3 abas reais: `Meus Produtos`, `Cadastro por IA`, `Classificacao`
- Aba `Meus Produtos`:
  - Botao `Atualizar` — recarrega a lista real de produtos via `getProdutos(...)`
  - DataTable com listagem de produtos (filtros por Area, Classe, Subclasse e texto)
  - Colunas: nome, fabricante, modelo, classe, score de completude, acoes
  - Botao `Visualizar` — carrega detalhe completo via `getProduto(id)`
  - Botao `Editar` — abre modal, resolve hierarquia Area -> Classe -> Subclasse
  - Botao `Reprocessar IA` — solicita reextracao das especificacoes
  - Botao `Verificar Completude` — consulta `getProdutoCompletude(produto.id)`
  - Botao `Excluir` — remove produto via CRUD
- Painel de detalhe do produto (apos selecionar):
  - Botao `Reprocessar IA` no card de detalhes
  - Botao `Verificar Completude` no card de detalhes
  - Botao `Precos de Mercado` — envia instrucao ao chat para buscar precos no PNCP
  - Toggle `Metadados de Captacao` — expande bloco de CATMAT, CATSER, termos de busca
  - Botao `Reprocessar Metadados` — chama endpoint proprio de enriquecimento
  - Botao `Excluir` no card de detalhes

## Endpoints ou servicos envolvidos
- GET /api/produtos (via `getProdutos(...)`) — lista produtos com filtros
- GET /api/produtos/{id} (via `getProduto(id)`) — detalhe completo do produto
- GET /api/produtos/{id}/completude (via `getProdutoCompletude(produto.id)`) — score de completude
- POST /api/produtos/{id}/reprocessar-metadados (via `reprocessarMetadados(produtoId)`) — enriquecimento CATMAT/CATSER/termos
- DELETE /api/crud/produtos/{id} — remove produto

## Dados manipulados
- Filtros de entrada: Area, Classe, Subclasse, texto livre
- Saida do detalhe: classificacao, NCM, preco de referencia, status pipeline, ANVISA, especificacoes, metadados de captacao (CATMAT, CATSER, termos de busca, ultima atualizacao)
- Armazenamento: tabela `produtos` + `produtos_especificacoes` no MySQL

- **Efeito funcional:** Sistema carrega produtos reais via `getProdutos(...)`
- **Requisicao:** GET /api/produtos com header Authorization: Bearer {token}
- **Persistencia:** Nenhuma (apenas leitura)
- **Evidencia:** Screenshot da pagina Portfolio (runtime/screenshots/UC-006/01_portfolio.png)

## Fluxos alternativos

### FA-001: Buscar ANVISA
1. O ator clica no botao `Buscar ANVISA` na lista ou detalhe
2. Sistema abre modal para informar numero de registro ou nome do produto
3. Ao confirmar, sistema cria sessao `busca-anvisa` e envia prompt textual apropriado
4. O retorno e mostrado inline em `iaResponse`

### FA-002: Buscar na Web
1. O ator clica no botao `Buscar na Web` na lista ou detalhe
2. Sistema abre modal para informar produto e, opcionalmente, fabricante
3. Ao confirmar, sistema cria sessao `busca-web` e envia prompt ao chat
4. O retorno e exibido inline

### FA-003: Precos de Mercado
1. O ator clica no botao `Precos de Mercado` no detalhe do produto
2. Sistema envia instrucao ao chat para buscar precos do item no PNCP
3. Resultado e exibido inline (nao persiste diretamente na pagina)

### FA-004: Editar produto
1. O ator clica no botao `Editar` em um produto
2. Sistema abre modal e resolve a hierarquia Area -> Classe -> Subclasse do item
3. Sistema carrega especificacoes existentes via `getProduto(id)`
4. O ator altera dados basicos, classificacao e campos tecnicos
5. Ao salvar: `crudUpdate("produtos", ...)` + upsert em `crudUpdate/crudCreate("produtos-especificacoes", ...)` para cada especificacao preenchida

### FA-005: Reprocessar especificacoes via IA
1. O ator clica no botao `Reprocessar IA` (na tabela ou no detalhe)
2. Sistema chama `onSendToChat("Reprocesse as especificacoes do produto ...")`
3. O fluxo delega o trabalho ao subsistema de chat/IA
4. Apos alguns segundos, a pagina executa `fetchProdutos()` para refletir possiveis alteracoes

## Fluxos de excecao

### FE-001: Nome do produto vazio
1. O ator tenta salvar sem preencher o nome do produto
2. O sistema exibe mensagem de validacao
3. O produto nao e salvo

### FE-002: Erro no processamento de IA
1. O ator faz upload de documento mas a IA (DeepSeek) nao consegue extrair dados
2. O sistema exibe mensagem de erro com detalhes do problema
3. O ator pode tentar com outro documento ou cadastrar manualmente

### FE-003: Formato de arquivo nao suportado
1. O ator tenta upload de arquivo com formato nao aceito pelo tipo selecionado
2. O sistema exibe mensagem indicando formatos validos (ex: ".pdf,.doc,.docx" para Manual Tecnico)
3. O upload nao e processado

### FE-004: Erro de rede
1. O ator clica em "Salvar" mas o backend nao responde
2. O sistema exibe mensagem de erro
3. Os dados permanecem no formulario
