# Caso de Uso: UC-021

## Nome
Vincular produto a item do edital

## Objetivo
Permitir ao usuario vincular produtos do portfolio da empresa aos itens do edital, de forma manual ou assistida por IA, estabelecendo a correspondencia entre o que o orgao solicita e o que a empresa oferece, incluindo calculo de volumetria e rendimento.

## Escopo
Modulo Precificacao — PrecificacaoPage.tsx

## Atores
- Usuario autenticado (Analista Comercial / Precificador)
- Sistema backend (Flask)
- Motor de IA (DeepSeek)
- Banco de dados

## Gatilho
Usuario clica no botao "Vincular Produto" em um item do edital na aba "Custos e Precos" da PrecificacaoPage, ou solicita vinculacao automatica por IA.

## Pre-condicoes
1. Lotes organizados com itens associados (UC-020 executado)
2. Portfolio de produtos cadastrado com especificacoes tecnicas (UC-006 a UC-008)
3. Backend Flask ativo com acesso ao DeepSeek para sugestao automatica
4. Dados dos itens do edital disponiveis (descricao, quantidade, unidade de medida)
5. Lista de produtos carregada via getProdutos() no frontend
6. PrecificacaoPage na aba "Custos e Precos" com edital e lote selecionados
7. Vinculos anteriores carregados do banco (se existirem)

## Pos-condicoes
1. Vinculo criado entre item do edital e produto do portfolio com match_score
2. Campos de volumetria preenchidos: volume_edital, rendimento_produto, repeticoes
3. Volume real ajustado e quantidade de kits calculados automaticamente
4. Formula de calculo registrada para auditoria
5. Vinculo confirmado pelo usuario (confirmado = true) apos revisao
6. Dados persistidos via POST /api/editais/{id}/vincular-produto
7. Camada de preco A (custo) pre-preenchida com base no produto vinculado

## Regras de negocio
1. **RN-VIN-001:** IA sugere produtos com match_score baseado em similaridade de descricao e especificacoes
2. **RN-VIN-002:** Usuario pode aceitar sugestao IA ou vincular manualmente via SelectInput de produtos
3. **RN-VIN-003:** Volumetria calcula: volume_real_ajustado = volume_edital / rendimento_produto * repeticoes
4. **RN-VIN-004:** Quantidade de kits = ceil(volume_real_ajustado / rendimento_por_kit)
5. **RN-VIN-005:** Repeticoes incluem 3 tipos: amostras, calibradores e controles
6. **RN-VIN-006:** Vinculo deve ser confirmado explicitamente pelo usuario antes de prosseguir
7. **RN-VIN-007:** Formula de calculo e registrada como string para rastreabilidade

## Componentes de UI envolvidos
1. Modal de selecao de produto (showSelecaoModal) com lista de sugestoes IA e busca manual
2. DataTable de sugestoes IA com colunas: nome, fabricante, match_score (ordenado por score)
3. SelectInput para selecao manual de produto do portfolio
4. FormField com campos de volumetria: volume_edital, rendimento_produto, repeticoes (amostras, calibradores, controles)
5. Card de resumo do vinculo mostrando produto, match_score, volume_real_ajustado e quantidade_kits
6. Botao "Confirmar Vinculo" (ActionButton com icone Check) para validar a associacao
7. Botao "Sugerir por IA" (ActionButton com icone Sparkles) que dispara tool_vincular_item_produto

## Endpoints ou servicos envolvidos
1. **POST /api/editais/{id}/vincular-produto** — cria ou atualiza vinculo item-produto
2. **tool_vincular_item_produto** — tool IA que sugere produto do portfolio para item do edital
3. **getProdutos()** — lista todos os produtos do portfolio para selecao manual
4. **crudList('edital_item_produtos', { edital_id })** — lista vinculos existentes
5. **crudUpdate('edital_item_produtos', id)** — atualiza dados de volumetria do vinculo
6. **createSession() + sendMessage()** — sessao IA para sugestao automatica via chat
7. **GET /api/editais/{id}/precificacao** — recarrega dados completos de precificacao

## Dados manipulados
1. **Vinculo:** id, edital_item_id, produto_id, match_score, confirmado
2. **Volume edital:** number — volume total solicitado pelo orgao licitante
3. **Rendimento produto:** number — rendimento por unidade/kit do produto
4. **Repeticoes:** amostras (number), calibradores (number), controles (number)
5. **Volume real ajustado:** number — volume calculado considerando rendimento e repeticoes
6. **Quantidade kits:** number — numero de kits necessarios para atender volume
7. **Formula calculo:** string — expressao usada no calculo para auditoria

## Fluxo principal
1. Usuario seleciona um item do edital na aba "Custos e Precos" e clica em "Vincular Produto" abrindo o Modal de selecao
2. Sistema dispara tool_vincular_item_produto via IA enviando descricao do item e lista de produtos do portfolio
3. IA retorna lista de sugestoes ordenada por match_score na DataTable do Modal com nome, fabricante e percentual
4. Usuario analisa sugestoes, seleciona o produto mais adequado e clica em "Selecionar" na linha correspondente
5. Sistema preenche campos de volumetria (volume_edital do item, rendimento_produto do cadastro) e usuario ajusta repeticoes
6. Sistema calcula volume_real_ajustado e quantidade_kits automaticamente e exibe no Card de resumo
7. Usuario confirma o vinculo clicando em "Confirmar Vinculo" que dispara POST /api/editais/{id}/vincular-produto

## Fluxos alternativos
1. **FA-01 — Vinculacao manual:** Usuario ignora sugestoes IA e seleciona produto diretamente no SelectInput
2. **FA-02 — Sem sugestao compativel:** IA nao encontra produto com match_score > 50%; usuario busca manualmente
3. **FA-03 — Alterar vinculo:** Usuario substitui vinculo existente por outro produto; sistema atualiza dados
4. **FA-04 — Vincular todos por IA:** Usuario clica "Vincular Todos" para que IA processe todos os itens do lote
5. **FA-05 — Ajustar volumetria:** Usuario modifica repeticoes ou rendimento apos vinculacao; sistema recalcula
6. **FA-06 — Buscar na Web:** Usuario clica "Buscar na Web" (showBuscaWebModal) para encontrar produto nao cadastrado
7. **FA-07 — Verificar ANVISA:** Usuario clica "ANVISA" (showAnvisaModal) para validar registro do produto sugerido

## Fluxos de excecao
1. **FE-01 — Portfolio vazio:** Nenhum produto cadastrado; sistema exibe alerta "Cadastre produtos antes de vincular"
2. **FE-02 — DeepSeek timeout:** Sugestao IA excede 30s; sistema exibe opcao de vinculacao manual
3. **FE-03 — Rendimento zero:** Produto sem rendimento cadastrado; sistema impede calculo e solicita preenchimento
4. **FE-04 — Produto descontinuado:** Produto vinculado esta inativo; sistema exibe warning com sugestao de alternativa
5. **FE-05 — Erro de persistencia:** POST retorna erro 500; frontend exibe toast de erro e mantem dados locais
6. **FE-06 — Match score muito baixo:** Todas sugestoes com score < 30%; sistema alerta "Nenhum produto adequado encontrado"
7. **FE-07 — Item ja vinculado:** Item ja possui vinculo confirmado; sistema pergunta se deseja substituir

## Criterios de validacao
1. IA sugere produtos com match_score ordenado decrescente e valores coerentes
2. Usuario pode selecionar sugestao IA ou vincular manualmente via SelectInput
3. Volumetria e calculada corretamente: volume_real_ajustado = volume / rendimento * repeticoes
4. Quantidade de kits e arredondada para cima (ceil) corretamente
5. Vinculo persiste no banco com todos os campos preenchidos
6. Formula de calculo e registrada como string auditavel
7. Vinculo confirmado atualiza interface com status visual de confirmacao
