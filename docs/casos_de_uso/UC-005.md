# Caso de Uso: UC-005

## Nome
Cadastro de Responsaveis

## Objetivo
Permitir que o usuario cadastre e gerencie os responsaveis da empresa (representantes legais, prepostos e responsaveis tecnicos), com seus dados de identificacao e contato.

## Escopo
Modulo Empresa — EmpresaPage.tsx > aba Responsaveis / Backend CRUD empresa-responsaveis

## Atores
- Ator primario: Usuario autenticado (gestor da empresa)

## Gatilho
O usuario acessa o menu "Empresa" e clica na aba "Responsaveis".

## Pre-condicoes
1. O usuario esta autenticado com JWT valido
2. Existe pelo menos uma empresa cadastrada vinculada ao usuario
3. O sistema (frontend e backend) esta operacional

## Pos-condicoes
1. Os responsaveis estao persistidos na tabela "empresa_responsaveis"
2. Cada responsavel possui nome, tipo/cargo, email e telefone
3. Os responsaveis aparecem na listagem da aba Responsaveis

## Regras de negocio
- RN-RESP-001: Responsaveis sao scoped por empresa do usuario
- RN-RESP-002: Tipos de responsavel sao predefinidos: Representante Legal, Preposto, Responsavel Tecnico
- RN-RESP-003: Nome do responsavel e campo obrigatorio
- RN-RESP-004: Email e telefone sao opcionais mas recomendados
- RN-RESP-005: Um responsavel pode ter cargo livre alem do tipo predefinido

## Componentes de UI envolvidos
- EmpresaPage > aba "Responsaveis"
- DataTable com colunas: nome, tipo, cargo, email, telefone, acoes
- ActionButton "Novo Responsavel" com icone Plus
- ActionButton com icone Pencil para edicao
- ActionButton com icone Trash2 para exclusao
- Modal ou formulario inline com campos:
  - TextInput: Nome
  - SelectInput: Tipo (Representante Legal, Preposto, Responsavel Tecnico)
  - TextInput: Cargo
  - TextInput: Email
  - TextInput: Telefone
- Constante TIPO_RESPONSAVEL_OPTIONS com opcoes predefinidas

## Endpoints ou servicos envolvidos
- GET /api/crud/empresa-responsaveis — lista responsaveis da empresa
- POST /api/crud/empresa-responsaveis — cria novo responsavel
- PUT /api/crud/empresa-responsaveis/{id} — atualiza responsavel
- DELETE /api/crud/empresa-responsaveis/{id} — remove responsavel

## Dados manipulados
- Entrada: nome (string, obrigatorio), tipo (string, enum), cargo (string, opcional), email (string, opcional), telefone (string, opcional)
- Saida: objeto Responsavel com campos: id, nome, tipo, cargo, email, telefone
- Armazenamento: tabela "empresa_responsaveis" no MySQL

## Fluxo principal

### Passo 1: Acesso a aba Responsaveis
- **Acao:** O ator clica na aba "Responsaveis" dentro da EmpresaPage
- **Elemento UI:** Tab "Responsaveis" na barra de abas
- **Resposta visual:** O sistema renderiza a lista de responsaveis da empresa em DataTable; se nao ha responsaveis, exibe estado vazio com botao para adicionar
- **Efeito funcional:** GET /api/crud/empresa-responsaveis e invocado para carregar responsaveis existentes
- **Requisicao:** GET /api/crud/empresa-responsaveis com filtro por empresa_id
- **Persistencia:** Nenhuma (apenas leitura)
- **Evidencia:** Screenshot da aba Responsaveis (runtime/screenshots/UC-005/01_aba_responsaveis.png)

- **Efeito funcional:** Estado de criacao e ativado; campos sao limpos
- **Requisicao:** Nenhuma
- **Persistencia:** Nenhuma
- **Evidencia:** Screenshot do formulario vazio (runtime/screenshots/UC-005/02_formulario_novo.png)

- **Requisicao:** Nenhuma

- **Persistencia:** Nenhuma

- **Acao:** O ator clica no botao "Salvar"
- **Elemento UI:** ActionButton "Salvar" com icone Save

- **Efeito funcional:** Os dados do formulario sao enviados ao backend
- **Requisicao:** POST /api/crud/empresa-responsaveis com body { nome, tipo, cargo, email, telefone, empresa_id }
- **Persistencia:** Registro criado na tabela "empresa_responsaveis" com id gerado

- **Resposta visual:** O responsavel recem-cadastrado aparece na tabela com todos os dados

- **Persistencia:** Nenhuma

## Fluxos alternativos

### FA-001: Edicao de responsavel existente
1. O ator clica no botao de edicao (icone Pencil) de um responsavel na listagem
2. O sistema carrega os dados do responsavel no formulario
3. O ator altera os campos desejados (ex: atualizar email ou telefone)
4. O ator clica em "Salvar"
5. PUT /api/crud/empresa-responsaveis/{id} e invocado com os dados atualizados
6. Listagem e atualizada com os novos dados

### FA-002: Exclusao de responsavel
1. O ator clica no botao de exclusao (icone Trash2) de um responsavel
2. O sistema exibe confirmacao de exclusao
3. O ator confirma a exclusao
4. DELETE /api/crud/empresa-responsaveis/{id} e invocado
5. Responsavel e removido da listagem

### FA-003: Cadastro sem campos opcionais
1. O ator preenche apenas o nome e o tipo (campos obrigatorios)
2. Campos cargo, email e telefone ficam vazios
3. O sistema aceita o cadastro normalmente
4. Responsavel aparece na listagem com campos opcionais em branco

## Fluxos de excecao

### FE-001: Nome vazio
1. O ator tenta salvar sem preencher o campo nome
2. O sistema exibe mensagem de validacao indicando campo obrigatorio
3. O responsavel nao e salvo

### FE-002: Erro de rede
1. O ator clica em "Salvar" mas o backend nao responde
2. O sistema exibe mensagem de erro
3. Os dados permanecem no formulario para nova tentativa

### FE-003: Email em formato invalido
1. O ator informa um email com formato invalido
2. O sistema pode exibir validacao de formato (dependendo da implementacao)
3. O ator corrige o email e tenta novamente

1. A aba "Responsaveis" renderiza com DataTable e botao de novo

3. Dropdown de tipo contém as 3 opcoes predefinidas (Representante Legal, Preposto, Responsavel Tecnico)
4. POST /api/crud/empresa-responsaveis salva o responsavel com sucesso
5. Responsavel salvo aparece na listagem
6. Edicao e exclusao funcionam corretamente
7. Validacao impede salvar sem nome
