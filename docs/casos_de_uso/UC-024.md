# Caso de Uso: UC-024

## Nome
Simular disputa de precos

## Objetivo
Permitir ao usuario simular cenarios de disputa de precos em pregao eletronico, testando diferentes combinacoes de lance_inicial e lance_minimo para avaliar margens, probabilidade de vitoria e posicionamento competitivo antes da licitacao.

## Escopo
Modulo Precificacao — PrecificacaoPage.tsx

## Atores
- Usuario autenticado (Analista Comercial / Precificador)
- Sistema backend (Flask)
- Motor de IA (DeepSeek)
- PNCP API (dados historicos de disputas)

## Gatilho
Usuario clica no botao "Simular Disputa" (icone Target) na aba "Custos e Precos" da PrecificacaoPage apos definir camadas de preco D (proposta) e E (minimo).

## Pre-condicoes
1. Camadas de preco A-E definidas para o item (UC-022 executado)
2. Historico de precos PNCP consultado (UC-023 executado, recomendado)
3. lance_inicial (Camada D) e lance_minimo (Camada E) preenchidos
4. Backend Flask ativo com acesso ao DeepSeek para simulacao
5. Dados de concorrentes historicos disponíveis (quando existentes)
6. PrecificacaoPage na aba "Custos e Precos" com vinculo selecionado
7. custo_base_final (Camada A) definido para calculo de margens

## Pos-condicoes
1. Cenarios de simulacao gerados com diferentes faixas de lance
2. Probabilidade estimada de vitoria por cenario exibida
3. Margem projetada por cenario calculada e exibida
4. Grafico de cenarios comparativo renderizado
5. Cenario recomendado pela IA destacado visualmente
6. Dados de simulacao disponiveis para consulta na estrategia competitiva (UC-025)
7. Resultado da simulacao persistido para historico e auditoria

## Regras de negocio
1. **RN-SIM-001:** Simulacao gera minimo 3 cenarios: conservador, moderado e agressivo
2. **RN-SIM-002:** Cenario conservador: lance_inicial proximo ao valor de referencia do edital
3. **RN-SIM-003:** Cenario moderado: lance entre preco_medio_historico e lance_inicial definido
4. **RN-SIM-004:** Cenario agressivo: lance proximo ao lance_minimo com margem_minima
5. **RN-SIM-005:** Probabilidade de vitoria estimada com base em historico PNCP e numero de concorrentes
6. **RN-SIM-006:** Margem projetada = (lance - custo_base_final) / custo_base_final * 100
7. **RN-SIM-007:** IA analisa padroes de disputa do orgao para refinar estimativa de probabilidade

## Componentes de UI envolvidos
1. Botao "Simular Disputa" (ActionButton com icone Target) que inicia a simulacao
2. Card de cenarios com 3 opcoes: conservador (azul), moderado (amarelo), agressivo (vermelho)
3. Para cada cenario: lance proposto, margem projetada (%), probabilidade de vitoria (%)
4. Grafico de barras comparativo (BarChart3) com margem vs probabilidade por cenario
5. Badge de recomendacao IA no cenario sugerido (icone Sparkles com "Recomendado")
6. FormField com sliders para ajustar lance_inicial e lance_minimo interativamente
7. Indicador de loading (Loader2) durante processamento da simulacao

## Endpoints ou servicos envolvidos
1. **POST /api/editais/{id}/simular-disputa** — executa simulacao de disputa com cenarios
2. **tool_simulacao_disputa** — tool IA que simula cenarios de disputa com base em historico
3. **GET /api/precos-historicos** — historico de precos para alimentar simulacao
4. **PNCP API** — dados de disputas anteriores do mesmo orgao
5. **crudList('simulacoes', { edital_id, item_id })** — lista simulacoes anteriores
6. **crudCreate('simulacoes')** — persiste resultado da simulacao
7. **tool_insights_precificacao** — gera insights sobre resultado da simulacao

## Dados manipulados
1. **Cenario:** tipo (conservador|moderado|agressivo), lance (number), margem (%), probabilidade (%)
2. **lance_inicial:** number — valor do lance proposto no cenario
3. **margem_projetada:** number — percentual de margem sobre custo_base_final
4. **probabilidade_vitoria:** number — estimativa percentual de chance de ganhar
5. **concorrentes_estimados:** number — numero estimado de participantes na disputa
6. **recomendacao_ia:** string — cenario recomendado pela IA com justificativa
7. **historico_disputas:** array de disputas anteriores do orgao com resultados

## Fluxo principal
1. Usuario verifica camadas D (lance_inicial) e E (lance_minimo) preenchidas e clica em "Simular Disputa"
2. Sistema dispara POST /api/editais/{id}/simular-disputa com lance_inicial, lance_minimo, custo_base_final e historico
3. Backend aciona tool_simulacao_disputa no DeepSeek enviando dados de precos, historico PNCP e perfil do orgao
4. IA gera 3 cenarios (conservador, moderado, agressivo) com lance, margem e probabilidade para cada um
5. Backend calcula margem_projetada para cada cenario e consolida resultados em resposta JSON estruturada
6. Frontend renderiza Cards de cenarios com cores distintas e grafico comparativo de margem vs probabilidade
7. IA destaca cenario recomendado com badge Sparkles e justificativa; usuario pode ajustar lances via sliders

## Fluxos alternativos
1. **FA-01 — Cenario personalizado:** Usuario cria cenario adicional com lance customizado alem dos 3 padrao
2. **FA-02 — Ajuste interativo:** Usuario ajusta lance via slider e cenarios recalculam em tempo real
3. **FA-03 — Sem historico PNCP:** Simulacao usa apenas dados internos e estimativas genericas
4. **FA-04 — Multiplos itens:** Usuario simula disputa para todos os itens do lote simultaneamente
5. **FA-05 — Comparar simulacoes:** Usuario compara simulacao atual com simulacoes anteriores do mesmo edital
6. **FA-06 — Exportar simulacao:** Usuario exporta resultados da simulacao em PDF para apresentacao interna
7. **FA-07 — Simulacao via Chat:** Usuario solicita simulacao pelo FloatingChat; mesma tool e acionada

## Fluxos de excecao
1. **FE-01 — Camadas nao definidas:** lance_inicial ou lance_minimo vazios; sistema exibe "Defina as camadas D e E antes de simular"
2. **FE-02 — DeepSeek timeout:** Simulacao excede 45s; sistema retorna cenarios basicos sem analise IA
3. **FE-03 — Margem negativa:** Cenario com lance < custo_base_final; sistema exibe alerta vermelho "Prejuizo"
4. **FE-04 — Dados insuficientes:** Sem historico nem referencia; probabilidade marcada como "indeterminada"
5. **FE-05 — Erro de calculo:** Divisao por zero ou NaN; sistema trata e exibe "Erro no calculo — verifique custos"
6. **FE-06 — Erro de persistencia:** POST retorna erro 500; frontend exibe toast e mantem resultados na tela
7. **FE-07 — Lance inicial < lance minimo:** Inconsistencia nos dados; sistema corrige automaticamente e alerta
