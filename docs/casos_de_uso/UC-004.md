# Caso de Uso: UC-004

## Nome
Busca Automatica de Certidoes

## Objetivo
Permitir que o usuario solicite a busca automatica de certidoes da empresa (INSS, FGTS, tributos federais) em portais governamentais, com resolucao automatica de CAPTCHAs, obtendo os documentos e seus status de validade.

## Escopo
Modulo Empresa — EmpresaPage.tsx > aba Certidoes / Backend tool_buscar_certidoes + certidao_browser.py

## Atores
- Ator primario: Usuario autenticado (gestor documental da empresa)
- Ator secundario: Sistema automatico (browser headless com CAPSolver)

## Gatilho
O usuario acessa a aba "Certidoes" na EmpresaPage e clica no botao de buscar certidoes.

## Pre-condicoes
1. O usuario esta autenticado com JWT valido
2. Existe pelo menos uma empresa cadastrada com CNPJ valido
3. O sistema (frontend e backend) esta operacional
4. CAPSolver esta configurado no .env (chave de API valida)
5. Fontes de certidoes estao cadastradas na tabela fontes_certidoes

## Pos-condicoes
1. As certidoes obtidas estao persistidas na tabela "empresa_certidoes"
2. Cada certidao possui status (valida, vencida, pendente, erro, nao_disponivel)
3. Arquivos PDF das certidoes estao salvos no servidor (path_arquivo)
4. Datas de obtencao e validade estao registradas

## Regras de negocio
- RN-CERT-001: Certidoes tem validade e o sistema calcula o status automaticamente (valida, vencida, pendente)
- RN-CERT-002: A busca automatica usa browser headless (certidao_browser.py) com CAPSolver para resolver CAPTCHAs
- RN-CERT-003: Nem todas as certidoes permitem busca automatica (campo permite_busca_automatica)
- RN-CERT-004: O status "buscando" e exibido durante o processamento
- RN-CERT-005: Fontes de certidoes sao configuraveis via tabela fontes_certidoes
- RN-CERT-006: Cada certidao registra orgao emissor, numero, URL de consulta e dados extras

## Componentes de UI envolvidos
- EmpresaPage > aba "Certidoes"
- DataTable com colunas: certidao, tipo, status, data_obtencao, validade, orgao_emissor, acoes
- StatusBadge com cores por status:
  - valida: verde
  - vencida: vermelho
  - pendente: amarelo
  - buscando: azul com animacao (Loader2)
  - erro: vermelho com icone AlertCircle
  - nao_disponivel: cinza
- ActionButton "Buscar Certidoes" com icone Search ou RefreshCw
- ActionButton com icone Eye para visualizar certidao
- ActionButton com icone Download para baixar PDF
- Icone Sparkles para indicar busca automatica com IA

## Endpoints ou servicos envolvidos
- GET /api/crud/empresa-certidoes — lista certidoes da empresa
- POST /api/crud/empresa-certidoes — cria registro de certidao manualmente
- PUT /api/crud/empresa-certidoes/{id} — atualiza certidao
- DELETE /api/crud/empresa-certidoes/{id} — remove certidao
- GET /api/crud/fontes-certidoes — lista fontes configuraveis
- tool_buscar_certidoes — tool de IA que aciona certidao_browser.py para busca automatica

## Dados manipulados
- Entrada: empresa_id, CNPJ da empresa, fontes de certidao selecionadas
- Saida: objeto CertidaoAutomatica com campos: id, certidao, tipo, status, data_obtencao, validade, path_arquivo, url_consulta, orgao_emissor, fonte_certidao_id, fonte_nome, numero, permite_busca_automatica, mensagem, dados_extras
- Armazenamento: tabela "empresa_certidoes" no MySQL + arquivos PDF no filesystem

## Fluxo principal

### Passo 1: Acesso a aba Certidoes
- **Acao:** O ator clica na aba "Certidoes" dentro da EmpresaPage
- **Elemento UI:** Tab "Certidoes" na barra de abas
- **Resposta visual:** O sistema renderiza a lista de certidoes da empresa; exibe DataTable com certidoes ja obtidas (se existirem) e seus status; exibe botao "Buscar Certidoes"
- **Efeito funcional:** GET /api/crud/empresa-certidoes e invocado para carregar certidoes existentes
- **Requisicao:** GET /api/crud/empresa-certidoes com filtro por empresa_id
- **Persistencia:** Nenhuma (apenas leitura)
- **Evidencia:** Screenshot da aba Certidoes (runtime/screenshots/UC-004/01_aba_certidoes.png)

### Passo 2: Iniciar busca automatica
- **Acao:** O ator clica no botao "Buscar Certidoes"
- **Elemento UI:** ActionButton com icone Search/RefreshCw e texto "Buscar Certidoes"
- **Resposta visual:** O botao entra em estado de loading; os StatusBadge das certidoes em processamento mudam para "buscando" (azul com icone Loader2 girando)
- **Efeito funcional:** O sistema aciona a tool_buscar_certidoes que inicia o certidao_browser.py em background
- **Requisicao:** Chamada a tool_buscar_certidoes via backend (pode ser via chat ou endpoint direto)
- **Persistencia:** Status das certidoes atualizado para "buscando" no banco
- **Evidencia:** Screenshot do estado de busca em andamento (runtime/screenshots/UC-004/02_buscando.png)

### Passo 3: Processamento automatico (sistema)
- **Acao:** Nenhuma do ator — o sistema executa automaticamente
- **Elemento UI:** StatusBadge "buscando" permanece visivel
- **Resposta visual:** O sistema mantem indicadores de loading enquanto o browser headless acessa os portais governamentais
- **Efeito funcional:** certidao_browser.py acessa sites do INSS, FGTS, Receita Federal; CAPSolver resolve CAPTCHAs automaticamente; PDFs sao baixados e salvos
- **Requisicao:** Requisicoes HTTP do browser headless aos portais governamentais
- **Persistencia:** Arquivos PDF salvos no filesystem; registros atualizados progressivamente na tabela empresa_certidoes
- **Evidencia:** Nenhuma visual direta (processamento em background)

### Passo 4: Resultado da busca
- **Acao:** O ator observa os resultados apos conclusao da busca (pode precisar atualizar a pagina)
- **Elemento UI:** DataTable atualizada com novas certidoes
- **Resposta visual:** Os StatusBadge mudam de "buscando" para o status final: "valida" (verde), "vencida" (vermelho), "erro" (vermelho com icone) ou "nao_disponivel" (cinza); campos de data_obtencao e validade sao preenchidos
- **Efeito funcional:** GET /api/crud/empresa-certidoes e re-executado para carregar resultados atualizados
- **Requisicao:** GET /api/crud/empresa-certidoes
- **Persistencia:** Nenhuma (leitura dos dados ja persistidos no passo 3)
- **Evidencia:** Screenshot do resultado da busca (runtime/screenshots/UC-004/03_resultado_busca.png)

### Passo 5: Visualizar certidao obtida
- **Acao:** O ator clica no botao Eye de uma certidao com status "valida"
- **Elemento UI:** ActionButton com icone Eye
- **Resposta visual:** O sistema abre o PDF da certidao em nova aba ou modal de preview
- **Efeito funcional:** O arquivo PDF e servido a partir do path_arquivo salvo no servidor
- **Requisicao:** GET do arquivo PDF
- **Persistencia:** Nenhuma
- **Evidencia:** Screenshot do preview da certidao (runtime/screenshots/UC-004/04_preview_certidao.png)

### Passo 6: Verificacao de validade
- **Acao:** O ator observa as datas de validade e os status de cada certidao
- **Elemento UI:** Colunas validade e status no DataTable
- **Resposta visual:** Certidoes validas exibem badge verde; certidoes proximas do vencimento exibem alerta; certidoes vencidas exibem badge vermelho
- **Efeito funcional:** Nenhum (apenas visualizacao)
- **Requisicao:** Nenhuma
- **Persistencia:** Nenhuma
- **Evidencia:** Screenshot dos status de validade (runtime/screenshots/UC-004/05_status_validade.png)

## Fluxos alternativos

### FA-001: Cadastro manual de certidao
1. O ator clica em "Nova Certidao" para cadastrar manualmente
2. O sistema abre modal com campos: nome da certidao, tipo, data de obtencao, validade, upload de arquivo
3. O ator preenche os campos e faz upload do PDF
4. POST /api/crud/empresa-certidoes e invocado
5. Certidao aparece na listagem com status correspondente

### FA-002: Busca individual de certidao
1. O ator clica no botao de atualizar de uma certidao especifica
2. O sistema busca apenas aquela certidao no portal correspondente
3. Resultado e atualizado apenas para aquela certidao

## Fluxos de excecao

### FE-001: Falha na resolucao de CAPTCHA
1. O CAPSolver nao consegue resolver o CAPTCHA de um portal
2. O status da certidao muda para "erro" com mensagem explicativa
3. O ator pode tentar novamente mais tarde

### FE-002: Portal governamental indisponivel
1. O site do orgao emissor esta fora do ar
2. O status da certidao muda para "erro" com mensagem "Portal indisponivel"
3. O ator pode tentar novamente

### FE-003: CNPJ invalido ou nao encontrado
1. O CNPJ da empresa nao e encontrado no portal consultado
2. O status da certidao muda para "nao_disponivel"
3. Mensagem explicativa e registrada no campo mensagem

### FE-004: CAPSolver nao configurado
1. A chave de API do CAPSolver nao esta no .env
2. O sistema retorna erro informando que a busca automatica nao esta disponivel
3. O ator deve cadastrar certidoes manualmente
