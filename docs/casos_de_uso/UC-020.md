# Caso de Uso: UC-020

## Nome
Organizar itens em lotes

## Objetivo
Permitir ao usuario organizar os itens de um edital em lotes logicos, agrupando-os por especialidade, tipo de amostra ou equipamento, facilitando a precificacao e a vinculacao de produtos do portfolio a cada item do edital.

## Escopo
Modulo Precificacao — PrecificacaoPage.tsx

## Atores
- Usuario autenticado (Analista Comercial / Precificador)
- Sistema backend (Flask)
- Banco de dados

## Gatilho
Usuario acessa a aba "Lotes" na PrecificacaoPage apos selecionar um edital com decisao "go" (UC-019) e clica em "Criar Lote" ou "Organizar Itens".

## Pre-condicoes
1. Edital com status "go" (decisao de participacao registrada via UC-019)
2. Itens do edital importados e disponiveis no banco de dados
3. Backend Flask ativo e banco de dados acessivel
4. Usuario com permissao de precificacao (papel Analista ou Precificador)
5. PrecificacaoPage carregada com edital selecionado no SelectInput
6. Dados dos itens do edital contendo descricao, unidade de medida e quantidade
7. Portfolio de produtos cadastrado para futura vinculacao (UC-006 a UC-009)

## Pos-condicoes
1. Lotes criados com numero sequencial, nome e especialidade definidos
2. Itens do edital associados aos lotes correspondentes
3. Volume exigido e valor estimado consolidados por lote
4. Status do lote definido como "rascunho" (pronto para vinculacao de produtos)
5. Mapa loteItensMap atualizado com itens de cada lote
6. Dados de lotes persistidos via POST /api/editais/{id}/lotes
7. Aba "Custos e Precos" habilitada para lotes com itens vinculados

## Regras de negocio
1. **RN-LOT-001:** Cada lote deve ter numero sequencial unico dentro do edital
2. **RN-LOT-002:** Um item do edital pode pertencer a apenas um lote por vez
3. **RN-LOT-003:** Lote deve conter pelo menos um item para ser salvo
4. **RN-LOT-004:** Campos opcionais do lote: especialidade, volume_exigido, tipo_amostra, equipamento, observacoes
5. **RN-LOT-005:** Valor estimado do lote e a soma dos valores estimados dos itens contidos
6. **RN-LOT-006:** Lote pode ser expandido/colapsado via botao ChevronDown/ChevronUp para visualizar itens
7. **RN-LOT-007:** Status do lote segue fluxo: rascunho -> vinculado -> precificado -> aprovado

## Componentes de UI envolvidos
1. SelectInput para selecao do edital (lista de editais com status "go")
2. Card de lote com numero, nome, especialidade, volume e valor estimado
3. Botao "Criar Lote" (ActionButton com icone Layers) para adicionar novo lote
4. FormField com TextInput para nome do lote, especialidade, volume exigido, tipo de amostra, equipamento e observacoes
5. DataTable com itens do edital (numero_item, descricao, unidade_medida, quantidade, valor_unitario_estimado)
6. Botao ChevronDown/ChevronUp para expandir/colapsar detalhes do lote (loteExpandido)
7. Botao "Salvar Lotes" que dispara POST /api/editais/{id}/lotes com dados consolidados

## Endpoints ou servicos envolvidos
1. **GET /api/editais/{id}/precificacao** — carrega dados de precificacao incluindo lotes existentes
2. **POST /api/editais/{id}/lotes** — cria ou reorganiza lotes do edital
3. **crudList('edital_itens', { edital_id })** — lista itens do edital para associacao
4. **crudUpdate('lotes', id)** — atualiza dados de um lote especifico
5. **getEditais()** — lista editais disponiveis com status "go" para selecao
6. **crudList('lote_itens', { lote_id })** — lista itens associados a um lote
7. **crudCreate('lotes')** — cria novo lote no banco de dados

## Dados manipulados
1. **Lote:** id, edital_id, numero_lote, nome, especialidade, volume_exigido, valor_estimado, status
2. **EditalItem:** id, numero_item, descricao, unidade_medida, quantidade, valor_unitario_estimado, valor_total_estimado
3. **loteItensMap:** Record<string, string[]> — mapeamento de lote_id para lista de item_ids
4. **loteEspecialidade:** string — especialidade do lote sendo editado
5. **loteVolume:** string — volume exigido do lote sendo editado
6. **loteTipoAmostra:** string — tipo de amostra do lote
7. **loteEquipamento:** string — equipamento associado ao lote

## Fluxo principal
1. Usuario acessa PrecificacaoPage e seleciona um edital com status "go" no SelectInput de editais disponiveis
2. Sistema carrega dados de precificacao via GET /api/editais/{id}/precificacao incluindo lotes e itens existentes
3. Usuario clica em "Criar Lote" que adiciona novo Card de lote com numero sequencial e campos editaveis
4. Usuario preenche nome do lote, especialidade, volume exigido, tipo de amostra e equipamento nos FormFields
5. Usuario arrasta ou seleciona itens do edital (DataTable) para associar ao lote criado via checkbox
6. Sistema calcula valor_estimado do lote somando valores dos itens associados e atualiza o Card
7. Usuario clica em "Salvar Lotes" que dispara POST /api/editais/{id}/lotes persistindo a organizacao no banco

## Fluxos alternativos
1. **FA-01 — Lotes pre-existentes:** Edital ja possui lotes definidos no PDF; sistema importa automaticamente na carga
2. **FA-02 — Lote unico:** Edital com item unico; sistema cria lote automatico com todos os itens
3. **FA-03 — Reorganizar lotes:** Usuario move itens entre lotes existentes arrastando ou via botao "Mover"
4. **FA-04 — Excluir lote:** Usuario remove lote vazio; itens ficam disponiveis para reassociacao
5. **FA-05 — Lote por especialidade:** Sistema sugere agrupamento por especialidade quando detecta itens relacionados
6. **FA-06 — Expandir/colapsar:** Usuario clica ChevronDown para ver detalhes do lote e ChevronUp para colapsar
7. **FA-07 — Duplicar lote:** Usuario duplica estrutura de lote existente para criar outro com mesma configuracao

## Fluxos de excecao
1. **FE-01 — Lote sem itens:** Sistema impede salvamento e exibe alerta "Lote deve conter pelo menos um item"
2. **FE-02 — Item ja associado:** Sistema avisa que item ja pertence a outro lote e oferece opcao de mover
3. **FE-03 — Edital sem itens importados:** Sistema exibe mensagem "Importe os itens do edital antes de criar lotes"
4. **FE-04 — Erro de persistencia:** POST retorna erro 500; frontend exibe toast de erro e mantem dados locais
5. **FE-05 — Numero de lote duplicado:** Backend rejeita lote com numero duplicado; frontend exibe erro de validacao
6. **FE-06 — Edital nao selecionado:** Botao "Criar Lote" desabilitado enquanto nenhum edital estiver selecionado
7. **FE-07 — Sessao expirada:** Token JWT invalido; sistema redireciona para login preservando estado local

## Criterios de validacao
1. Lotes sao criados com numero sequencial unico e campos editaveis
2. Itens do edital podem ser associados a lotes via selecao ou arraste
3. Valor estimado do lote e calculado corretamente como soma dos itens
4. Lote pode ser expandido e colapsado com ChevronDown/ChevronUp
5. Dados de lotes sao persistidos corretamente no banco via POST
6. Um item nao pode pertencer a dois lotes simultaneamente
7. Lote vazio nao pode ser salvo (validacao frontend e backend)
