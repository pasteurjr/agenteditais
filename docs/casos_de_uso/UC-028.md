# Caso de Uso: UC-028

## Nome
Validar registro ANVISA

## Objetivo
Permitir ao usuario validar os registros ANVISA dos produtos vinculados a proposta, verificando se cada produto possui registro vigente, identificando registros vencidos ou proximos do vencimento, garantindo conformidade regulatoria antes da submissao.

## Escopo
Modulo Proposta — PropostaPage.tsx

## Atores
- Usuario autenticado (Analista Comercial / Regulatorio)
- Sistema backend (Flask)
- Motor de IA (DeepSeek)
- ANVISA (consulta de registros)
- Banco de dados

## Gatilho
Usuario clica no botao "Validar ANVISA" (icone Shield) na PropostaPage para verificar registros dos produtos da proposta.

## Pre-condicoes
1. Proposta criada com produto vinculado (UC-026 executado)
2. Produto com registro ANVISA cadastrado no portfolio (UC-006)
3. Backend Flask ativo com acesso ao DeepSeek para validacao
4. Endpoint POST /api/propostas/{id}/validar-anvisa implementado
5. Dados do produto incluindo numero de registro ANVISA e validade
6. PropostaPage carregada com proposta selecionada
7. tool_validar_anvisa disponivel no backend

## Pos-condicoes
1. Lista de AnvisaItem exibida com produto, registro, validade e status
2. Status de cada registro classificado: "valido", "vencido" ou "proximo_vencimento"
3. Registros vencidos sinalizados com icone XCircle vermelho
4. Registros proximos do vencimento sinalizados com icone AlertTriangle amarelo
5. Registros validos sinalizados com icone CheckCircle verde
6. Resultado da validacao persistido para auditoria documental (UC-029)
7. Proposta bloqueada para status "aprovada" se houver registro vencido

## Regras de negocio
1. **RN-PRO-003:** Validacao ANVISA verifica se produtos tem registro vigente
2. **RN-ANV-001:** Status "valido": registro com validade > 90 dias da data de abertura do edital
3. **RN-ANV-002:** Status "proximo_vencimento": registro com validade entre 30 e 90 dias da abertura
4. **RN-ANV-003:** Status "vencido": registro expirado ou com validade < 30 dias da abertura
5. **RN-ANV-004:** Produto com registro vencido impede aprovacao da proposta (bloqueio de status)
6. **RN-ANV-005:** Validacao pode ser feita por consulta automatica ou informacao manual do usuario
7. **RN-ANV-006:** Resultado da validacao e registrado com data para rastreabilidade

## Componentes de UI envolvidos
1. Botao "Validar ANVISA" (ActionButton com icone Shield) que dispara validacao
2. DataTable de AnvisaItem com colunas: produto, registro, validade, status
3. Icone CheckCircle verde para registro "valido" na coluna status
4. Icone AlertTriangle amarelo para registro "proximo_vencimento" na coluna status
5. Icone XCircle vermelho para registro "vencido" na coluna status
6. Card de resumo ANVISA com contadores: X validos, Y proximos, Z vencidos
7. Indicador de loading (Loader2) durante validacao dos registros

## Endpoints ou servicos envolvidos
1. **POST /api/propostas/{id}/validar-anvisa** — valida registros ANVISA dos produtos da proposta
2. **tool_validar_anvisa** — tool IA que verifica registros ANVISA dos produtos
3. **crudList('produtos', { proposta_id })** — lista produtos vinculados a proposta
4. **crudGet('produtos', produto_id)** — busca dados do produto incluindo registro ANVISA
5. **GET /api/editais/{id}** — dados do edital incluindo data de abertura para calculo de validade
6. **crudUpdate('propostas', id)** — atualiza flag de validacao ANVISA na proposta
7. **crudCreate('validacoes_anvisa')** — persiste resultado da validacao para auditoria

## Dados manipulados
1. **AnvisaItem:** produto (string), registro (string), validade (string/date), status ("valido"|"vencido"|"proximo_vencimento")
2. **Numero registro ANVISA:** string — numero do registro do produto na ANVISA
3. **Data de validade:** date — data de vencimento do registro ANVISA
4. **Data de abertura:** date — data de abertura do edital para calculo de margem
5. **Dias ate vencimento:** number — diferenca entre validade e data de abertura
6. **Contadores:** validos (number), proximos (number), vencidos (number)
7. **Flag validacao:** boolean — indica se proposta passou na validacao ANVISA

## Fluxo principal
1. Usuario acessa PropostaPage com proposta selecionada e clica no botao "Validar ANVISA" com icone Shield
2. Sistema dispara POST /api/propostas/{id}/validar-anvisa enviando lista de produtos da proposta
3. Backend aciona tool_validar_anvisa que consulta registros ANVISA de cada produto vinculado a proposta
4. Para cada produto, sistema calcula dias entre validade do registro e data de abertura do edital
5. Sistema classifica status: "valido" (>90 dias), "proximo_vencimento" (30-90 dias), "vencido" (<30 dias)
6. Frontend renderiza DataTable de AnvisaItem com icones coloridos (CheckCircle, AlertTriangle, XCircle) por status
7. Sistema exibe Card de resumo com contadores e bloqueia aprovacao da proposta se houver registro vencido

## Fluxos alternativos
1. **FA-01 — Registro manual:** Usuario informa numero de registro e validade manualmente para produto sem dados
2. **FA-02 — Atualizar registro:** Produto com registro vencido; usuario atualiza dados do registro no portfolio
3. **FA-03 — Consulta ANVISA online:** Sistema consulta portal ANVISA via showAnvisaModal para validacao em tempo real
4. **FA-04 — Todos validos:** Todos os registros validos; sistema libera proposta para aprovacao
5. **FA-05 — Ignorar warning:** Registro proximo do vencimento; usuario reconhece risco e prossegue
6. **FA-06 — Produto isento:** Produto nao requer registro ANVISA; sistema marca como "isento"
7. **FA-07 — Revalidar:** Usuario solicita nova validacao apos atualizar dados de registro

## Fluxos de excecao
1. **FE-01 — Produto sem registro:** Produto sem numero de registro ANVISA cadastrado; sistema exibe "Registro nao informado"
2. **FE-02 — ANVISA indisponivel:** Consulta online falha; sistema usa dados do cadastro local
3. **FE-03 — Registro invalido:** Numero de registro com formato invalido; sistema exibe erro de validacao
4. **FE-04 — Data de abertura ausente:** Edital sem data de abertura; sistema usa data atual como referencia
5. **FE-05 — Erro de persistencia:** POST retorna erro 500; frontend exibe toast de erro
6. **FE-06 — Multiplos registros:** Produto com mais de um registro ANVISA; sistema valida todos e exibe lista
7. **FE-07 — Timeout na validacao:** Validacao excede 30s; sistema retorna resultados parciais
