# Caso de Uso: UC-045

## Nome
Buscar atas no PNCP

## Objetivo
Permitir ao usuario buscar atas de registro de preco no PNCP por termo e UF, visualizando resultados com titulo, orgao, data de publicacao e vigencia, com opcoes de salvar e extrair dados.

## Escopo
Modulo Atas — AtasPage.tsx > Aba Buscar

## Atores
- Usuario autenticado (Analista Comercial)
- Sistema backend (Flask)
- PNCP API (busca de atas de registro de preco)
- Motor de IA (DeepSeek) para extracao posterior

## Gatilho
Usuario digita um termo de busca na aba "Buscar" da AtasPage e clica no botao "Buscar" ou pressiona Enter.

## Pre-condicoes
1. Usuario esta autenticado no sistema (editais_ia_access_token)
2. Backend Flask ativo na porta 5007
3. PNCP API acessivel para busca de atas
4. Conexao com internet disponivel
5. Termo de busca com minimo de 3 caracteres
6. Endpoint GET /api/atas/buscar disponivel
7. AtasPage acessivel no menu do sistema

## Pos-condicoes
1. Lista de atas exibida na DataTable com titulo, orgao, UF, data de publicacao
2. Cada resultado possui botoes "Salvar" e "Extrair" para acoes subsequentes
3. Atas podem ser filtradas por UF (27 estados + DF)
4. Resultados incluem url_pncp para acesso direto ao portal
5. Ata salva fica disponivel na aba "Minhas Atas" (UC-047)
6. URL da ata pode ser enviada para extracao de dados via IA (UC-046)
7. Indicador de loading exibido durante busca

## Regras de negocio
1. **RN-ATA-BUSCA-001:** Busca retorna atas de registro de preco do PNCP com filtro por termo
2. **RN-ATA-BUSCA-002:** Termo minimo de 3 caracteres para disparar busca (validacao frontend)
3. **RN-ATA-BUSCA-003:** Filtro opcional por UF (27 estados + DF)
4. **RN-ATA-BUSCA-004:** Resultados incluem: titulo, orgao, uf, url_pncp, data_publicacao, data_vigencia_fim
5. **RN-ATA-BUSCA-005:** Botao "Salvar" persiste ata na base local para acompanhamento
6. **RN-ATA-BUSCA-006:** Botao "Extrair" preenche URL no formulario de extracao (tab 2) para processamento IA
7. **RN-ATA-BUSCA-007:** Dados retornados podem vir como data.atas, data.resultados ou array direto (frontend trata 3 formatos)

## Componentes de UI envolvidos
1. TextInput para termo de busca com placeholder descritivo
2. SelectInput para filtro de UF (27 estados + vazio para todos)
3. ActionButton "Buscar" com icone Search e estado de loading
4. DataTable com colunas: Titulo, Orgao, UF, Publicacao, Acoes
5. ActionButton "Salvar" por linha para persistir ata localmente
6. ActionButton "Extrair" por linha para enviar URL para extracao IA
7. Loader2 indicador de carregamento durante busca

## Endpoints ou servicos envolvidos
1. **GET /api/atas/buscar?termo={termo}&uf={uf}** — busca atas no PNCP por termo e UF opcional
2. **POST /api/atas/salvar** — salva ata selecionada na base local
3. **PNCP API** — fonte de dados de atas de registro de preco
4. **GET /api/atas/minhas** — lista atas ja salvas (para verificar duplicatas)
5. **POST /api/atas/extrair-pdf** — endpoint de extracao (ata selecionada vai para esta tab)
6. **tool_extrair_ata** — tool de IA para extracao de dados estruturados
7. **Auth middleware** — validacao de token JWT em todas as requisicoes

## Dados manipulados
1. **Termo de busca:** string com palavras-chave (min 3 caracteres)
2. **UF filtro:** string opcional — sigla do estado (AC a TO)
3. **AtaResult:** titulo, orgao, uf, url_pncp, data_publicacao, data_vigencia_fim
4. **URLSearchParams:** termo + uf concatenados para query string
5. **Resposta flexivel:** data.atas || data.resultados || Array.isArray(data) (3 formatos aceitos)
6. **AtaSalva:** id, titulo, orgao, uf, url_pncp, datas de vigencia, created_at
7. **Estado de UI:** searching (boolean), searchResults (AtaResult[])

## Fluxo principal
1. Usuario acessa AtasPage e visualiza aba "Buscar" com campo de texto para termo e SelectInput para UF
2. Usuario digita termo de busca (ex: "seringa descartavel") com minimo de 3 caracteres
3. Usuario opcionalmente seleciona UF para filtrar resultados por estado
4. Usuario clica em "Buscar" e frontend envia GET /api/atas/buscar?termo={termo}&uf={uf}
5. Backend consulta PNCP API, filtra atas de registro de preco e retorna lista com dados estruturados
6. Frontend trata resposta flexivel (atas/resultados/array) e renderiza DataTable com resultados
7. Cada linha exibe titulo, orgao, UF, data de publicacao e botoes "Salvar" e "Extrair" para acoes

## Fluxos alternativos
1. **FA-01 — Busca sem UF:** Usuario nao seleciona UF; busca retorna atas de todos os estados
2. **FA-02 — Salvar ata:** Usuario clica "Salvar"; POST /api/atas/salvar persiste ata e atualiza fetchMinhasAtas()
3. **FA-03 — Extrair dados:** Usuario clica "Extrair"; URL da ata preenche campo extractUrl e foco vai para aba Extrair
4. **FA-04 — Sem resultados:** Busca retorna array vazio; DataTable exibe mensagem "Nenhuma ata encontrada"
5. **FA-05 — Termo muito curto:** Termo < 3 caracteres; frontend nao dispara requisicao (guard clause)
6. **FA-06 — Busca via Chat:** Usuario solicita busca de atas pelo FloatingChat; mesma logica de busca acionada
7. **FA-07 — Multiplas buscas:** Usuario refina termo e busca novamente; resultados substituem anteriores

## Fluxos de excecao
1. **FE-01 — PNCP indisponivel:** Backend falha ao consultar PNCP; frontend captura erro no catch e exibe mensagem
2. **FE-02 — Timeout de busca:** Requisicao excede timeout; sistema exibe erro e sugere tentar novamente
3. **FE-03 — Token expirado:** Requisicao retorna 401; sistema redireciona para LoginPage
4. **FE-04 — Erro de rede:** fetch falha; try/catch captura e loga no console.error
5. **FE-05 — Resposta malformada:** JSON nao contem atas/resultados/array; setSearchResults recebe array vazio
6. **FE-06 — Erro ao salvar ata:** POST /api/atas/salvar falha; try/catch captura sem feedback visual
7. **FE-07 — Rate limit PNCP:** API retorna 429; backend aplica backoff e retorna erro ao frontend

1. Busca dispara somente com termo de 3+ caracteres
2. Resultados exibem titulo, orgao, UF e data de publicacao corretamente
3. Filtro por UF restringe resultados ao estado selecionado
4. Botao "Salvar" persiste ata e atualiza lista de "Minhas Atas"
5. Botao "Extrair" preenche URL no formulario de extracao
6. Loader exibido durante busca e removido ao completar
7. Frontend trata 3 formatos de resposta (atas/resultados/array) sem erros
