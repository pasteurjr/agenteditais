# Caso de Uso: UC-014

## Nome
Filtrar e selecionar editais

## Objetivo
Permitir ao usuario aplicar filtros combinados (UF, categoria, modalidade, periodo, faixa de valor) sobre os resultados de busca e selecionar editais de interesse para analise detalhada ou salvamento.

## Escopo
Modulo Captacao — CaptacaoPage.tsx

## Atores
- Usuario autenticado (Analista Comercial)
- Sistema frontend (React/Vite)

## Gatilho
Usuario interage com os controles de filtro (SelectInput de UF, categoria, modalidade) ou marca checkboxes de selecao nos editais da DataTable.

## Pre-condicoes
1. Usuario esta autenticado no sistema
2. Busca de editais foi realizada previamente (UC-013)
3. DataTable possui ao menos um resultado carregado
4. Filtros de UF, categoria e modalidade estao disponiveis na interface
5. Dados dos editais contem campos UF, classificacaoTipo e modalidade preenchidos
6. Portfolio de produtos cadastrado para classificacao de categoria
7. Interface CaptacaoPage renderizada com componentes de filtro visiveis

## Pos-condicoes
1. DataTable exibe apenas editais que atendem aos filtros selecionados
2. StatCards recalculados com metricas dos resultados filtrados
3. Editais selecionados (checkbox marcado) ficam destacados na tabela
4. Contagem de editais selecionados exibida na interface
5. Editais selecionados ficam disponiveis para acao em lote (salvar, exportar)
6. Filtros aplicados sao persistidos durante a sessao
7. Ordenacao da tabela e mantida apos aplicacao de filtros

## Regras de negocio
1. **RN-FIL-001:** Filtros sao cumulativos (AND) — UF + categoria + modalidade aplicados simultaneamente
2. **RN-FIL-002:** Filtro de UF aceita selecao multipla (ex: SP, RJ, MG)
3. **RN-FIL-003:** Categorias de produto sao inferidas automaticamente pelo sistema (tipoProdutoInferido)
4. **RN-FIL-004:** Modalidades incluem: Pregao Eletronico, Concorrencia, Dispensa, Inexigibilidade
5. **RN-FIL-005:** Filtro de periodo aplica-se a dataAbertura do edital
6. **RN-FIL-006:** Filtro de valor aplica-se ao valorTotalEstimado (faixa minima e maxima)
7. **RN-FIL-007:** Selecao via checkbox altera campo `selected` na interface EditalBusca

## Componentes de UI envolvidos
1. SelectInput de UF com lista de estados brasileiros
2. SelectInput de categoria/tipo de produto
3. SelectInput de modalidade de licitacao
4. DatePicker para periodo (data inicio e data fim)
5. Campos de valor minimo e maximo (TextInput numerico)
6. Checkbox em cada linha da DataTable para selecao individual
7. Checkbox no cabecalho da DataTable para selecao total (selecionar/deselecionar todos)

## Endpoints ou servicos envolvidos
1. **Filtragem client-side:** useMemo no React para filtrar array de editais em memoria
2. **GET /api/editais/buscar** — dados ja carregados na memoria do frontend
3. **crudList('fontes_editais')** — lista de fontes para filtro de origem
4. **crudList('classes_produtos')** — lista de classes para filtro de categoria
5. **Funcao de ordenacao:** sort() aplicado na DataTable por coluna clicada
6. **Estado local React:** useState para gerenciar filtros ativos e selecao
7. **useMemo:** recalculo otimizado de resultados filtrados e metricas

## Dados manipulados
1. **Filtro UF:** string[] — lista de UFs selecionadas
2. **Filtro categoria:** string — tipo de produto (classificacaoTipo)
3. **Filtro modalidade:** string — modalidade da licitacao
4. **Filtro periodo:** { inicio: Date, fim: Date } — faixa de datas
5. **Filtro valor:** { min: number, max: number } — faixa de valores
6. **Selecao:** boolean[] — array de editais selecionados (campo selected em EditalBusca)
7. **Metricas filtradas:** total filtrado, media de score filtrada, valor total filtrado

## Fluxo principal
1. Usuario visualiza a DataTable com resultados da busca e os controles de filtro acima da tabela
2. Usuario seleciona um ou mais estados no SelectInput de UF (ex: "SP", "RJ")
3. Sistema aplica filtro client-side via useMemo, exibindo apenas editais com UF correspondente
4. Usuario seleciona uma categoria no SelectInput de tipo de produto (ex: "Material Hospitalar")
5. Sistema aplica filtro cumulativo (UF + categoria), reduzindo a lista exibida
6. Usuario marca checkbox em editais de interesse na DataTable para seleciona-los individualmente
7. Sistema atualiza StatCards com metricas recalculadas e exibe contagem de editais selecionados

## Fluxos alternativos
1. **FA-01 — Selecionar todos:** Usuario clica no checkbox do cabecalho da DataTable; todos os editais visiveis (filtrados) sao marcados
2. **FA-02 — Limpar filtros:** Usuario clica em botao "Limpar filtros" e todos os resultados originais sao exibidos novamente
3. **FA-03 — Filtro sem resultados:** Combinacao de filtros nao retorna nenhum edital; sistema exibe mensagem informativa
4. **FA-04 — Ordenar por coluna:** Usuario clica no cabecalho de uma coluna da DataTable para ordenar (score, valor, data)
5. **FA-05 — Filtro por fonte:** Usuario filtra por fonte especifica (apenas PNCP, apenas Brave)
6. **FA-06 — Deselecionar individualmente:** Usuario desmarca checkbox de um edital previamente selecionado
7. **FA-07 — Filtro por potencial:** Usuario filtra por potencialGanho (elevado, medio, baixo)

## Fluxos de excecao
1. **FE-01 — Erro de renderizacao:** Componente de filtro falha ao renderizar; React error boundary captura e exibe fallback
2. **FE-02 — Dados incompletos:** Edital sem UF preenchida aparece em todas as filtragens de UF com badge "UF nao informada"
3. **FE-03 — Performance com muitos resultados:** Mais de 500 resultados sem filtro causa lentidao; sistema sugere aplicar filtros
4. **FE-04 — Filtro de valor com formato invalido:** Usuario digita texto no campo de valor; frontend valida e exibe erro inline
5. **FE-05 — Sessao expirada durante filtragem:** Token JWT expira; sistema redireciona para login mantendo estado de busca
6. **FE-06 — Modalidade desconhecida:** Edital com modalidade nao mapeada e categorizado como "Outros"
7. **FE-07 — Perda de selecao ao refiltrar:** Se filtro exclui edital previamente selecionado, sistema mantem selecao em memoria

7. Ordenacao por coluna funciona corretamente com filtros aplicados
