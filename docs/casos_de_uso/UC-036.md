# Caso de Uso: UC-036

## Nome
Monitorar janela de recurso

## Objetivo
Monitorar a janela de recurso dos editais participados, configurando alertas multi-canal (WhatsApp, Email, in-app) para notificar o usuario quando a janela abrir, permitindo acao tempestiva de recurso ou contra-razao.

## Escopo
Modulo Recursos — RecursosPage.tsx (aba Monitoramento)

## Atores
- Usuario autenticado (Analista Comercial / Juridico)
- Sistema backend (Flask + APScheduler)
- Servico de notificacoes (Email, WhatsApp, in-app)
- Motor de IA (DeepSeek) para consulta de status

## Gatilho
Usuario seleciona edital na aba Monitoramento e clica no botao "Ativar Monitoramento" para iniciar rastreamento da janela de recurso.

## Pre-condicoes
1. Usuario esta autenticado com token JWT valido
2. Backend Flask ativo com APScheduler configurado para monitoramento a cada hora (RN-REC-001)
3. Edital salvo no sistema com resultado de licitacao registrado
4. Servicos de notificacao configurados (Email, WhatsApp) no .env
5. RecursosPage com aba Monitoramento acessivel
6. Edital possui janela de recurso identificavel (prazo definido pelo orgao)
7. Conexao de rede estavel para recebimento de notificacoes

## Pos-condicoes
1. Monitoramento ativado para o edital selecionado com canais configurados
2. Status da janela rastreado: "aguardando", "aberta" ou "encerrada"
3. Alertas disparados automaticamente quando janela muda de status
4. Canais de notificacao respeitam configuracao do usuario (WhatsApp, Email, Sistema)
5. Estado do monitoramento persistido e visivel na aba
6. Tempo restante exibido quando janela esta aberta
7. Intencao de recurso pode ser registrada via botao dedicado

## Regras de negocio
1. **RN-MON-001:** Scheduler (APScheduler) verifica janelas de recurso a cada hora (RN-REC-001)
2. **RN-MON-002:** Alertas multi-canal: in-app (sempre), email (configuravel), WhatsApp (configuravel)
3. **RN-MON-003:** Status da janela: "aguardando" (resultado pendente), "aberta" (prazo em curso), "encerrada" (prazo expirou)
4. **RN-MON-004:** MonitoramentoState inclui: editalId, status, alertaWhatsapp, alertaEmail, alertaSistema, ativado, tempoRestante
5. **RN-MON-005:** Ativacao cria sessao de chat via createSession("monitorar-janela") para consulta de status
6. **RN-MON-006:** Intencao de recurso registrada via onSendToChat enviando mensagem ao FloatingChat
7. **RN-MON-007:** Desativacao do monitoramento suspende alertas mas mantem historico

## Componentes de UI envolvidos
1. SelectInput para selecao de edital a monitorar (editalId) com opcoes "numero - orgao"
2. Checkboxes para canais de alerta: WhatsApp (Smartphone), Email (Mail), Sistema (Bell)
3. ActionButton "Ativar Monitoramento" com icone Activity e estado loading (monitorLoading)
4. Badge de status da janela: "aguardando" (warning), "aberta" (success), "encerrada" (neutral)
5. Campo de tempo restante (Timer) quando janela esta aberta
6. ActionButton "Registrar Intencao de Recurso" com icone Gavel
7. Card de resumo do monitoramento com status, canais ativos e ultimo check

## Endpoints ou servicos envolvidos
1. **createSession("monitorar-janela")** — cria sessao de chat para consulta de status da janela
2. **sendMessage(sid, prompt)** — envia prompt para IA verificar status da janela de recurso
3. **GET /api/editais** — lista editais para SelectInput (getEditais)
4. **APScheduler** — scheduler backend que verifica janelas a cada hora
5. **notifications.py** — servico de notificacoes (email, WhatsApp, in-app)
6. **onSendToChat(mensagem)** — envia mensagem ao FloatingChat para registrar intencao
7. **setMonitoramento(state)** — atualiza estado local do monitoramento

## Dados manipulados
1. **MonitoramentoState:** editalId, status (JanelaStatus), alertaWhatsapp, alertaEmail, alertaSistema, ativado, tempoRestante
2. **JanelaStatus:** "aguardando" | "aberta" | "encerrada" — status da janela de recurso
3. **editalId:** string — UUID do edital selecionado para monitoramento
4. **monitorLoading:** boolean — estado de loading durante ativacao
5. **alertaWhatsapp:** boolean — flag de canal WhatsApp ativo
6. **alertaEmail:** boolean — flag de canal Email ativo (default true)
7. **alertaSistema:** boolean — flag de canal in-app ativo (default true)

## Fluxo principal
1. Usuario acessa RecursosPage e navega para aba "Monitoramento" que exibe seletor de edital e opcoes de alerta
2. Usuario seleciona edital no dropdown e configura canais de alerta: WhatsApp (checkbox), Email (checkbox), Sistema (checkbox)
3. Usuario clica em "Ativar Monitoramento"; frontend cria sessao via createSession("monitorar-janela")
4. Sistema envia prompt a IA solicitando status da janela de recurso do edital; IA retorna status atual
5. Frontend atualiza MonitoramentoState com editalId, ativado=true e status retornado ("aguardando"/"aberta"/"encerrada")
6. APScheduler no backend verifica janelas a cada hora; quando status muda, dispara alertas nos canais configurados
7. Badge de status atualiza em tempo real; se janela "aberta", exibe tempo restante e habilita botao "Registrar Intencao de Recurso"

## Fluxos alternativos
1. **FA-01 — Janela ja aberta:** Status retorna "aberta" imediatamente; sistema exibe tempo restante e alerta urgente
2. **FA-02 — Janela encerrada:** Status retorna "encerrada"; sistema informa que prazo expirou e desativa monitoramento
3. **FA-03 — Registrar intencao:** Usuario clica "Registrar Intencao de Recurso"; mensagem enviada ao FloatingChat via onSendToChat
4. **FA-04 — Desativar monitoramento:** Usuario desativa alertas; sistema suspende notificacoes mas mantem historico
5. **FA-05 — Alterar canais:** Usuario modifica checkboxes de canais a qualquer momento; alteracao efetiva imediatamente
6. **FA-06 — Multiplos editais:** Usuario ativa monitoramento para varios editais simultaneamente
7. **FA-07 — Alerta manual:** Usuario solicita verificacao manual do status sem aguardar proxima execucao do scheduler

## Fluxos de excecao
1. **FE-01 — Nenhum edital selecionado:** Botao "Ativar Monitoramento" inativo se editalId estiver vazio
2. **FE-02 — Erro na sessao de chat:** createSession falha; console.error registra e monitoramento nao ativa
3. **FE-03 — Scheduler indisponivel:** APScheduler falha; alertas automaticos nao disparam; verificacao manual ainda disponivel
4. **FE-04 — Email nao configurado:** Servico de email indisponivel; alerta via email silenciosamente falha; in-app permanece
5. **FE-05 — WhatsApp nao configurado:** API WhatsApp nao integrada; checkbox visivel mas alerta nao dispara
6. **FE-06 — Token expirado:** Requisicoes retornam 401; frontend redireciona para login
7. **FE-07 — Edital sem resultado:** Janela de recurso indefinida; status permanece "aguardando" indefinidamente
