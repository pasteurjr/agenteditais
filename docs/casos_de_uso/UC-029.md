# Caso de Uso: UC-029

## Nome
Auditoria documental

## Objetivo
Permitir ao usuario verificar a completude documental da proposta, identificando documentos presentes, ausentes e vencidos que sao obrigatorios para a submissao, garantindo que o dossie esteja completo antes da exportacao e envio ao orgao licitante.

## Escopo
Modulo Proposta — PropostaPage.tsx

## Atores
- Usuario autenticado (Analista Comercial / Regulatorio)
- Sistema backend (Flask)
- Motor de IA (DeepSeek)
- Banco de dados

## Gatilho
Usuario clica no botao "Auditoria Documental" (icone FileCheck) na PropostaPage para verificar completude do dossie da proposta.

## Pre-condicoes
1. Proposta criada com texto tecnico gerado (UC-026 executado)
2. Validacao ANVISA realizada (UC-028, recomendado)
3. Documentos da empresa cadastrados (UC-003 — gestao de documentos)
4. Backend Flask ativo com acesso ao DeepSeek para analise
5. Requisitos documentais do edital identificados e catalogados
6. PropostaPage carregada com proposta selecionada
7. tool_auditoria_documental disponivel no backend

## Pos-condicoes
1. Lista de DocItem exibida com nome, tamanho_kb, status e path
2. Status de cada documento classificado: "presente", "ausente" ou "vencido"
3. Documentos ausentes sinalizados com icone XCircle vermelho
4. Documentos vencidos sinalizados com icone AlertTriangle amarelo
5. Documentos presentes sinalizados com icone CheckCircle verde
6. Percentual de completude documental calculado e exibido
7. Proposta liberada para exportacao (UC-030) quando 100% completa

## Regras de negocio
1. **RN-AUD-001:** Auditoria verifica documentos obrigatorios definidos no edital contra acervo da empresa
2. **RN-AUD-002:** Documentos obrigatorios tipicos: proposta tecnica, certidoes, ANVISA, contrato social, atestados
3. **RN-AUD-003:** Status "presente": documento encontrado e vigente no acervo
4. **RN-AUD-004:** Status "ausente": documento obrigatorio nao encontrado no acervo
5. **RN-AUD-005:** Status "vencido": documento encontrado mas com validade expirada
6. **RN-AUD-006:** Completude = (presentes / total_obrigatorios) * 100 — deve ser 100% para exportacao
7. **RN-AUD-007:** IA analisa edital para identificar documentos obrigatorios nao catalogados manualmente

## Componentes de UI envolvidos
1. Botao "Auditoria Documental" (ActionButton com icone FileCheck) que dispara verificacao
2. DataTable de DocItem com colunas: nome, tamanho_kb (formatado), status (icone colorido), path
3. Icone CheckCircle verde para documento "presente" na coluna status
4. Icone AlertTriangle amarelo para documento "vencido" na coluna status
5. Icone XCircle vermelho para documento "ausente" na coluna status
6. Card de resumo com barra de progresso: X presentes, Y vencidos, Z ausentes, completude %
7. Botao "Upload Documento" para adicionar documento ausente diretamente da tela de auditoria

## Endpoints ou servicos envolvidos
1. **tool_auditoria_documental** — tool IA que verifica completude documental da proposta
2. **crudList('documentos_empresa')** — lista documentos cadastrados da empresa
3. **crudList('certidoes')** — lista certidoes da empresa com validade
4. **GET /api/editais/{id}** — dados do edital com requisitos documentais
5. **POST /api/propostas/{id}/auditoria** — executa auditoria e retorna resultado
6. **crudCreate('documentos_empresa')** — upload de documento faltante
7. **crudUpdate('propostas', id)** — atualiza flag de auditoria na proposta

## Dados manipulados
1. **DocItem:** nome (string), tamanho_kb (number), status ("presente"|"ausente"|"vencido"), path (string?)
2. **Documentos obrigatorios:** lista de documentos exigidos pelo edital
3. **Acervo empresa:** documentos cadastrados com validade e path
4. **Certidoes:** CND, FGTS, trabalhista, fiscal — com data de validade
5. **Completude:** number (%) — percentual de documentos presentes sobre total obrigatorios
6. **Contadores:** presentes (number), vencidos (number), ausentes (number)
7. **Flag auditoria:** boolean — indica se proposta passou na auditoria documental

## Fluxo principal
1. Usuario acessa PropostaPage com proposta selecionada e clica no botao "Auditoria Documental" com icone FileCheck
2. Sistema dispara tool_auditoria_documental via backend enviando dados do edital e lista de documentos da empresa
3. IA analisa requisitos documentais do edital e cruza com acervo de documentos e certidoes da empresa
4. Para cada documento obrigatorio, sistema verifica: existencia no acervo, validade e path do arquivo
5. Sistema classifica cada DocItem como "presente", "ausente" ou "vencido" com tamanho_kb quando disponivel
6. Frontend renderiza DataTable de DocItem com icones coloridos e Card de resumo com barra de progresso e completude %
7. Sistema libera exportacao (UC-030) quando completude = 100% ou usuario reconhece pendencias explicitas

## Fluxos alternativos
1. **FA-01 — Upload direto:** Usuario identifica documento ausente e faz upload diretamente da tela de auditoria
2. **FA-02 — Certidao vencida:** Sistema identifica certidao vencida e sugere reemissao via UC-004
3. **FA-03 — Documento opcional:** Edital lista documentos opcionais; sistema marca como "opcional" sem impactar completude
4. **FA-04 — Re-auditoria:** Usuario solicita nova auditoria apos upload de documentos faltantes
5. **FA-05 — Exportar checklist:** Usuario exporta lista de documentos pendentes em PDF para providencias
6. **FA-06 — Auditoria parcial:** Edital sem requisitos claros; IA infere documentos tipicos do tipo de licitacao
7. **FA-07 — Documento compartilhado:** Mesmo documento atende multiplos requisitos; sistema vincula uma vez

## Fluxos de excecao
1. **FE-01 — Edital sem requisitos documentais:** Sistema lista documentos tipicos padrao com aviso
2. **FE-02 — DeepSeek timeout:** Auditoria excede 30s; sistema retorna verificacao basica sem analise IA
3. **FE-03 — Acervo vazio:** Nenhum documento cadastrado; todos marcados como "ausente"
4. **FE-04 — Path invalido:** Documento com path que nao existe mais; sistema marca como "ausente" com nota
5. **FE-05 — Erro de persistencia:** POST retorna erro 500; frontend exibe toast de erro
6. **FE-06 — Documento corrompido:** Arquivo existe mas esta corrompido; sistema marca como "ausente" com nota
7. **FE-07 — Multiplas versoes:** Mais de uma versao do mesmo documento; sistema usa mais recente
