# Caso de Uso: UC-035

## Nome
Controle de prazo de impugnacao

## Objetivo
Exibir e monitorar prazos de impugnacao dos editais salvos, calculando automaticamente o prazo limite de 3 dias uteis antes da data de abertura conforme Art. 164 da Lei 14.133/2021, com sinalizacao visual por cores de urgencia.

## Escopo
Modulo Impugnacao — ImpugnacaoPage.tsx (aba Prazos)

## Atores
- Usuario autenticado (Analista Comercial / Juridico)
- Sistema frontend (calculo automatico de prazos)
- Editais salvos no sistema (fonte de dados)

## Gatilho
Usuario acessa a aba "Prazos" da ImpugnacaoPage; sistema calcula automaticamente prazos de todos os editais com data de abertura.

## Pre-condicoes
1. Usuario esta autenticado com token JWT valido
2. Editais salvos no sistema com campo data_abertura preenchido
3. Frontend carregado com funcao calcularPrazoLimite operacional
4. Backend Flask ativo para fornecer lista de editais (GET /api/editais)
5. Data do sistema correta para calculo preciso de dias restantes
6. Ao menos um edital com data de abertura futura para exibicao de prazo ativo
7. ImpugnacaoPage com aba Prazos renderizada no TabPanel

## Pos-condicoes
1. Lista de prazos exibida na DataTable ordenada por dias restantes (menor primeiro)
2. Cada edital exibe: numero, orgao, data abertura, prazo limite, dias restantes, status
3. Cores de urgencia aplicadas: vermelho (<3 dias), amarelo (3-5 dias), verde (>5 dias)
4. Prazos expirados marcados com status "expirado" e cor vermelha
5. Prazos ativos marcados com status "ativo" e cor correspondente a urgencia
6. Calculo exclui sabados e domingos na contagem de dias uteis
7. Lista atualizada automaticamente ao recarregar pagina

## Regras de negocio
1. **RN-PRA-001:** Prazo de impugnacao = 3 dias uteis antes da data de abertura do edital (Art. 164, Lei 14.133/2021)
2. **RN-PRA-002:** Calculo de dias uteis exclui sabados (dia 6) e domingos (dia 0) usando getDay()
3. **RN-PRA-003:** Status "expirado" quando diasRestantes < 0; "ativo" quando >= 0
4. **RN-PRA-004:** Cor de urgencia: vermelho (diasRestantes < 3), amarelo (3 <= dias <= 5), verde (dias > 5)
5. **RN-PRA-005:** Funcao getPrazoColor retorna classes CSS: text-danger, text-warning, text-success
6. **RN-PRA-006:** Lista ordenada por dias_restantes crescente (mais urgente primeiro)
7. **RN-PRA-007:** Editais sem data_abertura sao filtrados da lista de prazos (filter e.data_abertura)

## Componentes de UI envolvidos
1. TabPanel aba "Prazos" na ImpugnacaoPage
2. DataTable com colunas: numero, orgao, data_abertura, prazo_limite, dias_restantes, status
3. Badges de status: "ativo" (success) e "expirado" (error)
4. Texto colorido de dias restantes: text-danger (vermelho), text-warning (amarelo), text-success (verde)
5. Icone Clock para cabecalho e indicadores de prazo
6. Icone AlertTriangle para prazos proximos do vencimento
7. Indicador de loading (loadingEditais) enquanto editais sao carregados

## Endpoints ou servicos envolvidos
1. **GET /api/editais** — carrega editais salvos com data_abertura (getEditais)
2. **calcularPrazoLimite(dataAbertura)** — funcao frontend que calcula prazo de 3 dias uteis
3. **getPrazoColor(dias)** — funcao frontend que retorna classe CSS por urgencia
4. **setEditais(data)** — atualiza estado de editais no componente
5. **setPrazos(prazosList)** — atualiza estado de prazos calculados
6. **Array.sort((a, b) => a.dias_restantes - b.dias_restantes)** — ordenacao por urgencia
7. **filter((e) => e.data_abertura)** — filtra editais sem data de abertura

## Dados manipulados
1. **PrazoEdital:** id, numero, orgao, data_abertura, prazo_limite, dias_restantes, status ("ativo"|"expirado")
2. **dataAbertura:** string ISO — data de abertura do edital
3. **prazoLimite:** string formatada (dd/MM/yyyy) — data limite para impugnacao
4. **diasRestantes:** number — dias corridos ate o prazo limite (negativo se expirado)
5. **status:** "ativo" | "expirado" — derivado de diasRestantes
6. **prazos:** PrazoEdital[] — lista completa de prazos ordenada
7. **loadingEditais:** boolean — estado de carregamento dos editais

## Fluxo principal
1. Usuario acessa ImpugnacaoPage e clica na aba "Prazos" do TabPanel para visualizar calendario de impugnacoes
2. Componente executa loadEditais() no useEffect inicial, buscando editais via GET /api/editais
3. Para cada edital com data_abertura, sistema executa calcularPrazoLimite que subtrai 3 dias uteis (excluindo sabado/domingo)
4. Sistema gera array de PrazoEdital com numero, orgao, data_abertura formatada, prazo_limite, diasRestantes e status
5. Array e ordenado por dias_restantes crescente (mais urgente primeiro) e salvo em setPrazos
6. DataTable renderiza lista com cores de urgencia: vermelho (< 3 dias), amarelo (3-5 dias), verde (> 5 dias)
7. Editais com prazo expirado aparecem com badge "expirado" vermelho e prazos ativos com badge "ativo" verde

## Fluxos alternativos
1. **FA-01 — Todos prazos expirados:** Lista exibe todos com badge "expirado" e cor vermelha; nenhuma acao de impugnacao disponivel
2. **FA-02 — Edital sem data de abertura:** Edital filtrado da lista de prazos; nao aparece na DataTable
3. **FA-03 — Prazo hoje:** diasRestantes = 0; badge "ativo" com cor vermelha (urgente, ultimo dia)
4. **FA-04 — Clicar em edital:** Usuario clica em linha para navegar a aba Validacao Legal do mesmo edital
5. **FA-05 — Exportar prazos:** Usuario exporta lista de prazos em CSV para controle externo
6. **FA-06 — Alerta proativo:** Sistema envia notificacao quando prazo atinge 3 dias restantes
7. **FA-07 — Atualizar prazos:** Usuario recarrega pagina para atualizar calculo com data corrente

## Fluxos de excecao
1. **FE-01 — Nenhum edital salvo:** Lista de prazos vazia; DataTable exibe mensagem "Nenhum edital com prazo cadastrado"
2. **FE-02 — Data de abertura invalida:** calcularPrazoLimite recebe string invalida; retorna prazoLimite "--", diasRestantes 0, status "expirado"
3. **FE-03 — Erro ao carregar editais:** getEditais lanca excecao; console.error registra e lista permanece vazia
4. **FE-04 — Token expirado:** GET /api/editais retorna 401; frontend redireciona para LoginPage
5. **FE-05 — Data do sistema incorreta:** Calculo de diasRestantes produz resultado inesperado; requer correcao de relogio
6. **FE-06 — Feriados nao considerados:** Calculo considera apenas sabado/domingo; feriados nacionais/estaduais nao descontados
7. **FE-07 — Erro de renderizacao:** DataTable falha ao renderizar; React exibe fallback de erro

## Criterios de validacao
1. Prazo calculado como exatamente 3 dias uteis antes da data de abertura (excluindo sabado/domingo)
2. Cores de urgencia aplicadas corretamente: vermelho < 3, amarelo 3-5, verde > 5
3. Status "expirado" para diasRestantes negativo, "ativo" para >= 0
4. Lista ordenada por dias restantes crescente (mais urgente primeiro)
5. Editais sem data_abertura nao aparecem na lista de prazos
6. Data formatada em pt-BR (dd/MM/yyyy) via toLocaleDateString
7. Recarregar pagina atualiza calculo com data corrente
