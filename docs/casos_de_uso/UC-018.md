# Caso de Uso: UC-018

## Nome
Analise de mercado via IA

## Objetivo
Executar analise de mercado e concorrencia para um edital especifico, identificando sinais de mercado, historico de vencedores do orgao, estimativa de numero de concorrentes, tendencias de precos e riscos competitivos, auxiliando a tomada de decisao estrategica.

## Escopo
Modulo Validacao — ValidacaoPage.tsx

## Atores
- Usuario autenticado (Analista Comercial)
- Sistema backend (Flask)
- Motor de IA (DeepSeek)
- PNCP API (historico de vencedores e precos)

## Gatilho
Usuario seleciona um edital na ValidacaoPage e, na aba "Mercado", clica no botao "Analisar Mercado do Orgao" (ou "Reanalisar Mercado" se ja executado anteriormente).

## Pre-condicoes
1. Edital salvo e com score calculado (UC-016 executado)
2. Backend Flask ativo com acesso ao DeepSeek
3. Acesso a PNCP API para consulta de historico de vencedores
4. Dados do orgao licitante disponiveis (CNPJ, historico)
5. Portfolio de produtos cadastrado para referencia de competitividade
6. Historico de participacoes da empresa em licitacoes anteriores
7. Dados de precos do edital (valor estimado, itens com valores)

## Pos-condicoes
1. Sinais de mercado listados e analisados
2. Historico de vencedores do orgao exibido com resultados anteriores
3. Reputacao do orgao avaliada (pregoeiro, pagador, historico)
4. Estimativa de concorrentes e nivel de competitividade
5. Analise de margem e viabilidade comercial disponivel
6. Riscos competitivos identificados e classificados
7. Dados de mercado persistidos para referencia na decisao (UC-019)

## Regras de negocio
1. **RN-MER-001:** Analise de mercado consulta historico de vencedores do mesmo orgao nos ultimos 24 meses
2. **RN-MER-002:** Sinais de mercado incluem: numero estimado de concorrentes, presenca de incumbentes, barreiras de entrada
3. **RN-MER-003:** Reputacao do orgao avalia 3 aspectos: qualidade do pregoeiro, pontualidade de pagamento, regularidade do processo
4. **RN-MER-004:** HistoricoItem registra resultado (vencida/perdida/cancelada) e motivo para cada licitacao anterior
5. **RN-MER-005:** Margem de expectativa e calculada comparando valor estimado com custos da empresa
6. **RN-MER-006:** Riscos competitivos incluem: preco muito baixo, exigencias atipicas, orgao com historico de cancelamentos
7. **RN-MER-007:** Analise integra dados de todas as 6 dimensoes para visao holistica de mercado

## Componentes de UI envolvidos

Na aba "Mercado" da ValidacaoPage:
1. Botao "Analisar Mercado do Orgao" / "Reanalisar Mercado"
2. Card de dados do orgao contratante
3. Card de reputacao do orgao
4. Card de volume de compras do orgao
5. Card de compras similares ao edital
6. Card de historico interno da empresa com o orgao
7. Analise textual de mercado gerada pela IA

## Endpoints ou servicos envolvidos
1. **POST /api/editais/{id}/analisar-mercado** — endpoint real para analise de mercado (tanto para primeira analise quanto para reanalisar)
2. **DeepSeek API** — motor IA para analise de mercado e geracao de insights textuais
3. **PNCP API** — dados de historico do orgao contratante (consultado internamente pelo backend)
4. **GET /api/precos-historicos** — historico de precos PNCP para referencia (se disponivel)
5. **crudList('participacoes')** — historico de participacoes da propria empresa (referencia interna)

## Dados manipulados
1. **SinaisMercado:** string[] — lista de indicadores de mercado e competitividade
2. **HistoricoItem:** nome (string), resultado ("vencida"|"perdida"|"cancelada"), motivo (string)
3. **ReputacaoOrgao:** pregoeiro (string), pagador (string), historico (string) — classificacao qualitativa
4. **MargemExpectativa:** number — percentual de margem estimada sobre o valor do edital
5. **PotencialGanho:** "elevado"|"medio"|"baixo" — classificacao do potencial de retorno
6. **IntencaoEstrategica:** "estrategico"|"defensivo"|"acompanhamento"|"aprendizado" — posicionamento recomendado
7. **Riscos competitivos:** string[] — lista de riscos identificados na analise

## Fluxo principal (sequencia real observada — UC-CV12)
1. Usuario abre a aba "Mercado" na ValidacaoPage
2. Usuario clica em "Analisar Mercado do Orgao"
3. Sistema chama POST /api/editais/{id}/analisar-mercado
4. Interface mostra dados do orgao, reputacao, volume de compras, compras similares e historico interno
5. Sistema exibe a analise textual de mercado produzida pela IA
6. Usuario pode clicar em "Reanalisar Mercado" para reprocessar a analise com dados atualizados

## Fluxos alternativos
1. **FA-01 — Orgao sem historico:** PNCP nao retorna licitacoes anteriores; sistema exibe "Primeiro registro deste orgao" e omite historico
2. **FA-02 — Empresa ja participou com orgao:** Sistema destaca participacoes anteriores da empresa com resultados
3. **FA-03 — Analise via Chat:** Usuario solicita analise de mercado pelo FloatingChat; mesma tool e acionada
4. **FA-04 — Comparar com editais similares:** Sistema identifica editais similares ja analisados e compara metricas
5. **FA-05 — Atualizar com dados externos:** Usuario informa dados adicionais de mercado (ex: preco de concorrente conhecido)
6. **FA-06 — Exportar analise:** Usuario exporta analise de mercado em PDF para apresentacao interna
7. **FA-07 — Intencao estrategica personalizada:** Usuario sobrescreve intencaoEstrategica sugerida pela IA com classificacao propria

## Fluxos de excecao
1. **FE-01 — PNCP sem historico de precos:** Analise comercial fica limitada; sistema avisa que referencia de precos indisponivel
2. **FE-02 — DeepSeek timeout:** Analise excede 45s; backend retorna analise parcial com dados ja coletados
3. **FE-03 — Erro na consulta de vencedores:** PNCP retorna erro; historico indisponivel com mensagem ao usuario
4. **FE-04 — Orgao com CNPJ invalido:** Sistema nao consegue consultar historico; alerta exibido com orientacao
5. **FE-05 — Dados insuficientes para margem:** Sem valor estimado no edital; margem calculada como "indeterminada"
6. **FE-06 — Multiplas fontes conflitantes:** Dados de precos divergentes entre fontes; sistema exibe faixa (min-max)
7. **FE-07 — Analise desatualizada:** Se analise tem mais de 7 dias, sistema sugere reprocessamento
