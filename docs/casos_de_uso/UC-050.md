# Caso de Uso: UC-050

## Nome
Acompanhar cronograma de entregas

## Objetivo
Exibir cronograma semanal de entregas de um contrato com indicacao visual de entregas atrasadas, proximas (7 dias), pendentes e entregues, permitindo gestao proativa do calendario de obrigacoes contratuais.

## Escopo
Modulo Contratos — ProducaoPage.tsx > Aba Cronograma

## Atores
- Usuario autenticado (Gestor de Contratos)
- Sistema backend (Flask)
- Modulo de notificacoes (alertas de atraso)

## Gatilho
Usuario seleciona um contrato na aba Contratos da ProducaoPage e acessa a aba "Cronograma".

## Pre-condicoes
1. Usuario esta autenticado no sistema
2. Contrato selecionado (UC-048) com ao menos uma entrega cadastrada (UC-049)
3. Backend Flask ativo com endpoint GET /api/contratos/{id}/cronograma
4. Entregas possuem data_prevista preenchida
5. ProducaoPage acessivel no menu do sistema
6. Endpoint de cronograma retorna dados estruturados por semana
7. Dados de entregas atualizados no banco de dados

## Pos-condicoes
1. Cronograma semanal exibido com entregas agrupadas por semana
2. Stats consolidados: pendentes, entregues, atrasados, total
3. Lista de entregas atrasadas com dias de atraso calculados
4. Lista de proximas entregas (7 dias) com dias restantes
5. Indicadores visuais de atraso destacados em vermelho
6. Gestao proativa de prazos possibilitada pela visao consolidada
7. Dados sincronizados com dashboard Contratado x Realizado

## Regras de negocio
1. **RN-CRON-001:** Cronograma agrupa entregas por semana com base em data_prevista
2. **RN-CRON-002:** Entregas atrasadas = data_prevista < hoje E status != "entregue", com dias_atraso calculados
3. **RN-CRON-003:** Proximas entregas = data_prevista dentro de 7 dias a partir de hoje
4. **RN-CRON-004:** Stats: pendentes + entregues + atrasados = total de entregas do contrato
5. **RN-CRON-005:** Cronograma carregado ao selecionar contrato via selectContrato()
6. **RN-CRON-006:** Atrasados destacados com badge vermelho e dias_atraso
7. **RN-CRON-007:** Proximos 7 dias destacados com badge amarelo e dias_restantes

## Componentes de UI envolvidos
1. TabPanel aba "Cronograma" com icone Clock na ProducaoPage
2. StatCards com 4 metricas: Pendentes, Entregues, Atrasados, Total
3. Secao "Atrasados" com lista de entregas vencidas e dias_atraso
4. Secao "Proximos 7 dias" com lista de entregas iminentes e dias_restantes
5. Cronograma semanal com entregas agrupadas por semana (Record<string, EntregaAPI[]>)
6. Badge de status: pendente (amarelo), entregue (verde), atrasado (vermelho)
7. Mensagem "Selecione um contrato" quando nenhum contrato ativo

## Endpoints ou servicos envolvidos
1. **GET /api/contratos/{id}/cronograma** — retorna cronograma semanal com stats, semanas, atrasados e proximos_7d
2. **crudList("contrato-entregas")** — entregas brutas do contrato
3. **CRUD /api/crud/contrato-entregas** — atualizacao de status de entrega
4. **GET /api/dashboard/contratado-realizado** — dashboard que consome dados de entregas
5. **GET /api/alertas-vencimento/consolidado** — alertas de entregas incluidos
6. **Auth middleware** — validacao de token JWT
7. **Scheduler** — monitoramento automatico de prazos

## Dados manipulados
1. **CronogramaData:** stats, semanas (Record<string, EntregaAPI[]>), atrasados, proximos_7d
2. **Stats:** pendentes (number), entregues (number), atrasados (number), total (number)
3. **Semanas:** chave string (ex: "2026-W13") com array de entregas daquela semana
4. **Atrasados:** EntregaAPI + dias_atraso (number) — entregas vencidas com calculo
5. **Proximos 7d:** EntregaAPI + dias_restantes (number) — entregas iminentes
6. **EntregaAPI:** id, contrato_id, descricao, quantidade, valor, data_prevista, data_realizada, status
7. **Estado UI:** cronograma (CronogramaData | null)

## Fluxo principal
1. Usuario seleciona contrato na aba "Contratos" via botao "Selecionar" que chama selectContrato()
2. selectContrato() dispara GET /api/contratos/{id}/cronograma em paralelo com entregas, aditivos e designacoes
3. Backend calcula cronograma: agrupa entregas por semana, identifica atrasados e proximos 7 dias, gera stats
4. Frontend recebe CronogramaData e seta state cronograma
5. Aba "Cronograma" exibe StatCards (pendentes, entregues, atrasados, total) no topo
6. Secao "Atrasados" lista entregas vencidas com badge vermelho e dias_atraso
7. Secao "Proximos 7 dias" lista entregas iminentes e cronograma semanal agrupa restante por semana

## Fluxos alternativos
1. **FA-01 — Sem contrato selecionado:** Aba exibe mensagem "Selecione um contrato na aba Contratos"
2. **FA-02 — Contrato sem entregas:** Cronograma vazio; stats mostram zeros; mensagem informativa
3. **FA-03 — Todas entregas realizadas:** Stats mostram 0 pendentes e 0 atrasados; secao "Atrasados" vazia
4. **FA-04 — Multiplos atrasados:** Lista de atrasados ordenada por dias_atraso decrescente (mais critico primeiro)
5. **FA-05 — Entrega iminente (1 dia):** Destaque especial na secao "Proximos 7 dias" com urgencia alta
6. **FA-06 — Atualizar cronograma:** Usuario registra nova entrega (UC-049); selectContrato() recarrega cronograma
7. **FA-07 — Exportar cronograma:** Dados de cronograma disponiveis para analise no Chat IA

## Fluxos de excecao
1. **FE-01 — Erro ao carregar cronograma:** GET falha; try/catch loga erro e cronograma fica null
2. **FE-02 — Token expirado:** Requisicao retorna 401; sistema redireciona para LoginPage
3. **FE-03 — Dados de datas invalidos:** data_prevista nula; entrega nao aparece no cronograma semanal
4. **FE-04 — Resposta malformada:** JSON sem stats/semanas; cronograma setado como null
5. **FE-05 — Contrato excluido:** contrato_id invalido; backend retorna 404
6. **FE-06 — Erro de rede:** fetch falha; try/catch captura e loga
7. **FE-07 — Muitas entregas:** Cronograma com dezenas de semanas; scroll necessario para visualizar tudo
