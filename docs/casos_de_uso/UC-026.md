# Caso de Uso: UC-026

## Nome
Gerar proposta tecnica com IA

## Objetivo
Permitir ao usuario gerar automaticamente o texto da proposta tecnica e comercial utilizando IA (DeepSeek), com base nas especificacoes do produto, requisitos do edital, template selecionado e dados de precificacao, produzindo um documento estruturado pronto para revisao.

## Escopo
Modulo Proposta — PropostaPage.tsx

## Atores
- Usuario autenticado (Analista Comercial / Redator de Proposta)
- Sistema backend (Flask)
- Motor de IA (DeepSeek)
- Banco de dados

## Gatilho
Usuario clica no botao "Gerar Proposta" (icone Sparkles) na PropostaPage apos selecionar edital, produto, lote e template, ou clica em "Gerar Texto" em uma proposta existente em status "rascunho".

## Pre-condicoes
1. Edital com status "precificado" (UC-025 — estrategia aprovada)
2. Vinculo item-produto confirmado com precificacao definida (UC-021 e UC-022)
3. Template de proposta selecionado ou template padrao disponivel
4. Backend Flask ativo com acesso ao DeepSeek
5. Dados do produto no portfolio: especificacoes tecnicas, fabricante, registro ANVISA
6. Dados do edital: requisitos, lotes, itens, valores
7. PropostaPage carregada com SelectInputs de edital, produto, lote e template

## Pos-condicoes
1. Texto da proposta tecnica gerado pela IA e armazenado no campo conteudo/texto_tecnico
2. Proposta criada no banco com status "rascunho" via POST /api/propostas
3. Texto estruturado com secoes do template: introducao, especificacoes, conformidade, precos
4. Dados de preco unitario e quantidade preenchidos automaticamente da precificacao
5. Log de criacao registrado no PropostaLog com timestamp e usuario
6. Editor rico habilitado para revisao e edicao do texto gerado (UC-027)
7. Status "rascunho" permite reescrita parcial ou total do texto

## Regras de negocio
1. **RN-PRO-001:** Fluxo de status: rascunho -> revisao -> aprovada -> enviada
2. **RN-PRO-002:** Todas as alteracoes sao registradas em PropostaLog com timestamp e usuario
3. **RN-PRO-004:** Geracao de proposta e inline (sem chat), corrigida no commit f21aa2b
4. **RN-GER-001:** IA usa template selecionado como estrutura base para o texto gerado
5. **RN-GER-002:** Texto gerado inclui especificacoes do produto, conformidade com edital e diferenciais
6. **RN-GER-003:** Precos e quantidades sao inseridos automaticamente no texto da proposta
7. **RN-GER-004:** Geracao inline dispara POST /api/propostas/{id}/gerar-texto sem criar sessao de chat

## Componentes de UI envolvidos
1. SelectInput para edital (lista editais com status "precificado" ou "go")
2. SelectInput para produto (getProdutos) e SelectInput para lote (LoteOption)
3. SelectInput para template (TemplateOption com id, nome e conteudo_md)
4. Botao "Gerar Proposta" (ActionButton com icone Sparkles) que dispara geracao inline
5. Indicador de loading (Loader2) durante geracao do texto pela IA
6. Card de preview com texto gerado renderizado em ReactMarkdown
7. DataTable de propostas existentes com colunas: edital, orgao, produto, valor, status

## Endpoints ou servicos envolvidos
1. **POST /api/propostas** — cria nova proposta no banco com dados basicos
2. **POST /api/propostas/{id}/gerar-texto** — gera texto tecnico com IA (inline, sem chat)
3. **tool_gerar_proposta** — tool IA que gera texto da proposta tecnica baseado em especificacoes
4. **getEditais()** — lista editais para selecao no formulario
5. **getProdutos()** — lista produtos para selecao no formulario
6. **crudList('proposta-templates')** — lista templates de proposta disponiveis
7. **crudList('propostas')** — lista propostas existentes para DataTable

## Dados manipulados
1. **Proposta:** id, edital_id, produto_id, lote_id, template_id, preco_unitario, quantidade, valor_total, status, conteudo
2. **Template:** id, nome, conteudo_md — modelo de texto com placeholders para geracao
3. **Texto gerado:** string (markdown) — texto da proposta tecnica produzido pela IA
4. **Preco unitario:** number — preco do lance_inicial (Camada D) da precificacao
5. **Quantidade:** number — quantidade do item do edital
6. **Valor total:** number — preco_unitario * quantidade
7. **PropostaLog:** proposta_id, acao, usuario, timestamp — registro de auditoria

## Fluxo principal
1. Usuario acessa PropostaPage e seleciona edital, produto, lote e template nos SelectInputs do formulario de criacao
2. Sistema pre-preenche preco_unitario e quantidade dos dados de precificacao (Camada D) automaticamente
3. Usuario clica "Gerar Proposta" que dispara POST /api/propostas criando registro com status "rascunho"
4. Sistema dispara POST /api/propostas/{id}/gerar-texto enviando dados do edital, produto e template para IA
5. Backend aciona tool_gerar_proposta inline (sem chat) enviando especificacoes, requisitos e template como contexto
6. IA retorna texto estruturado em markdown com secoes: introducao, especificacoes, conformidade, precos e diferenciais
7. Frontend exibe texto gerado no Card de preview com ReactMarkdown e habilita edicao no editor rico (UC-027)

## Fluxos alternativos
1. **FA-01 — Template padrao:** Usuario nao seleciona template; sistema usa template padrao do sistema
2. **FA-02 — Regenerar texto:** Usuario clica "Regenerar" para que IA produza nova versao do texto
3. **FA-03 — Proposta existente:** Usuario edita proposta ja criada em vez de gerar nova
4. **FA-04 — Sem template:** Nenhum template cadastrado; IA gera texto com estrutura propria
5. **FA-05 — Multiplos produtos:** Edital com varios itens; sistema gera proposta consolidada por lote
6. **FA-06 — Texto parcial:** Usuario solicita geracao apenas de secoes especificas (ex: so especificacoes)
7. **FA-07 — Proposta manual:** Usuario redige texto inteiramente manual sem usar geracao IA

## Fluxos de excecao
1. **FE-01 — DeepSeek timeout:** Geracao excede 60s; sistema exibe erro e sugere tentar novamente
2. **FE-02 — Produto sem especificacoes:** Produto sem dados tecnicos; IA gera texto generico com aviso
3. **FE-03 — Template invalido:** Template corrompido ou vazio; sistema usa template padrao
4. **FE-04 — Edital sem precificacao:** Preco_unitario = 0; sistema alerta "Defina precos antes de gerar proposta"
5. **FE-05 — Erro de persistencia:** POST retorna erro 500; frontend exibe toast de erro
6. **FE-06 — Texto muito longo:** IA gera texto > limite de caracteres; sistema trunca e avisa
7. **FE-07 — Sessao expirada:** Token JWT invalido; sistema redireciona para login
