# Caso de Uso: UC-054

## Nome
Dashboard contratado x realizado

## Objetivo
Exibir dashboard comparativo entre valores contratados e realizados de contratos, com indicadores de saude do portfolio (verde/amarelo/vermelho), variacao percentual por contrato com semaforo de cores, e filtros por periodo e orgao.

## Escopo
Modulo Dashboard CR — ContratadoRealizadoPage.tsx

## Atores
- Usuario autenticado (Gestor de Contratos / Diretor Comercial)
- Sistema backend (Flask)
- Modulo de contratos (fonte de dados)
- Modulo de entregas (valores realizados)

## Gatilho
Usuario acessa ContratadoRealizadoPage que dispara loadDashboard() automaticamente via useEffect ao montar o componente.

## Pre-condicoes
1. Usuario esta autenticado no sistema (editais_ia_access_token)
2. Backend Flask ativo com endpoint GET /api/dashboard/contratado-realizado
3. Ao menos um contrato cadastrado com entregas registradas
4. Valores contratados e realizados preenchidos para calculo de variacao
5. ContratadoRealizadoPage acessivel no menu do sistema
6. Filtros de periodo e orgao opcionais
7. Dados de entregas atualizados para calculo correto

## Pos-condicoes
1. Dashboard exibe totais: contratado, realizado, variacao_pct, contratos_ativos
2. DataTable de contratos com variacao percentual e semaforo de cores
3. Saude do portfolio calculada: saudavel, atencao ou critico
4. Lista de atrasos com severidade (HIGH/MEDIUM/LOW)
5. Proximos vencimentos consolidados
6. Filtros de periodo e orgao aplicados aos dados
7. Dados atualizados a cada mudanca de filtro

## Regras de negocio
1. **RN-CR-001:** Variacao <=5% = verde (saudavel), 5-15% = amarelo (atencao), >15% = vermelho (critico)
2. **RN-CR-002:** Severidade de atraso: HIGH (>30 dias), MEDIUM (15-30 dias), LOW (<15 dias)
3. **RN-CR-003:** Saude do portfolio: "saudavel" | "atencao" | "critico" baseado em entregas em dia
4. **RN-CR-004:** Variacao calculada: abs(contratado - realizado) / contratado * 100
5. **RN-CR-005:** Filtro de periodo aceita: 1m, 3m, 6m, 12m, todos
6. **RN-CR-006:** Filtro de orgao filtra por nome do orgao contratante
7. **RN-CR-007:** Dashboard recarrega automaticamente ao alterar filtros (useEffect + useCallback)

## Componentes de UI envolvidos
1. StatCards com 4 metricas: Total Contratado, Total Realizado, Variacao %, Contratos Ativos
2. SelectInput para filtro de periodo (1m/3m/6m/12m/todos)
3. TextInput para filtro de orgao
4. DataTable de contratos: numero, orgao, contratado, realizado, variacao_pct (com semaforo), status, data_fim
5. Badge de variacao com cor: verde (<=5%), amarelo (5-15%), vermelho (>15%)
6. Indicador de saude do portfolio (saudavel/atencao/critico) com icone Shield
7. Loader2 indicador de carregamento com RefreshCw para atualizar

## Endpoints ou servicos envolvidos
1. **GET /api/dashboard/contratado-realizado?periodo={}&orgao={}** — dados comparativos com filtros
2. **GET /api/contratado-realizado** — endpoint alternativo de dados comparativos
3. **GET /api/contratado-realizado/saude** — saude do portfolio
4. **GET /api/contratado-realizado/atrasos** — pedidos em atraso
5. **GET /api/contratado-realizado/vencimentos** — proximos vencimentos
6. **GET /api/alertas-vencimento/consolidado** — alertas de vencimento (UC-056)
7. **Auth middleware** — validacao de token via authHeaders()

## Dados manipulados
1. **DashboardTotais:** contratado (number), realizado (number), variacao_pct (number), contratos_ativos (number)
2. **ContratoRow:** contrato_id, numero, orgao, objeto, valor_contratado, valor_realizado, variacao_pct, status, data_fim
3. **AtrasoRow:** contrato_id, contrato_numero, orgao, entrega, data_prevista, dias_atraso, severidade, valor
4. **DashboardResponse:** totais, contratos[], atrasos[], proximos_vencimentos[], saude_portfolio, erro?
5. **Filtros:** periodo (string "6m" default), orgao (string)
6. **Saude:** "saudavel" | "atencao" | "critico"
7. **Semaforo CSS:** status-badge-success (verde), status-badge-warning (amarelo), status-badge-danger (vermelho)

## Fluxo principal
1. Usuario acessa ContratadoRealizadoPage e loadDashboard() dispara automaticamente via useEffect
2. Frontend envia GET /api/dashboard/contratado-realizado?periodo=6m com headers de autenticacao
3. Backend calcula totais (contratado vs realizado), variacao por contrato, atrasos e saude do portfolio
4. Frontend recebe DashboardResponse e atualiza states: dashboard, dashLoading, dashError
5. StatCards renderizam: Total Contratado, Total Realizado, Variacao %, Contratos Ativos
6. DataTable exibe contratos com semaforo de variacao (verde/amarelo/vermelho) e status
7. Indicador de saude do portfolio (icone Shield) mostra estado geral: saudavel, atencao ou critico

## Fluxos alternativos
1. **FA-01 — Filtrar por periodo:** Usuario altera SelectInput de periodo (1m/3m/6m/12m); loadDashboard() recarrega
2. **FA-02 — Filtrar por orgao:** Usuario digita nome do orgao no TextInput; loadDashboard() recarrega com filtro
3. **FA-03 — Sem contratos:** Dashboard exibe zeros nos totais e DataTable vazia
4. **FA-04 — Todos saudaveis:** Variacao <=5% em todos; saude = "saudavel" com indicador verde
5. **FA-05 — Contrato critico:** Variacao >15%; badge vermelho e saude pode mudar para "atencao" ou "critico"
6. **FA-06 — Atualizar manualmente:** Usuario clica em RefreshCw para recarregar dados
7. **FA-07 — Exportar dados:** Dados do dashboard disponiveis para analise via Chat IA (onSendToChat)

## Fluxos de excecao
1. **FE-01 — Erro no endpoint:** GET falha; dashError setado com mensagem; interface exibe alerta
2. **FE-02 — Token expirado:** Requisicao retorna 401; sistema redireciona para LoginPage
3. **FE-03 — Resposta com erro:** data.erro presente; throw new Error(data.erro) capturado
4. **FE-04 — Backend lento:** dashLoading=true; Loader2 exibido ate resposta chegar
5. **FE-05 — Divisao por zero:** contratado = 0; variacao_pct calculado como 0 no backend
6. **FE-06 — Erro de rede:** fetch falha; catch seta dashError com mensagem descritiva
7. **FE-07 — Dados inconsistentes:** realizado > contratado * 2; sistema exibe normalmente com variacao alta
