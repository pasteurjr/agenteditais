# Caso de Uso: UC-038

## Nome
Chatbox de analise interativo

## Objetivo
Fornecer chatbox interativo na aba Analise da RecursosPage que permite ao usuario explorar desvios identificados pela analise da proposta vencedora (UC-037), fazer perguntas complementares e aprofundar investigacao de inconsistencias via conversa com IA.

## Escopo
Modulo Recursos — RecursosPage.tsx (aba Analise > Chatbox — aparece apos UC-037)

## Atores
- Usuario autenticado (Analista Comercial / Juridico)
- Motor de IA (DeepSeek) via sessao de chat
- Contexto da analise anterior (UC-037)

## Gatilho
Apos conclusao da analise da proposta vencedora (UC-037), chatbox interativo aparece na aba Analise; usuario digita pergunta e envia.

## Pre-condicoes
1. Analise da proposta vencedora (UC-037) executada com sucesso
2. Sessao de chat criada (chatSessionId preenchido) apos analise
3. Resultado da analise disponivel (analiseResultadoMd e inconsistenciasVencedora)
4. Usuario autenticado com token JWT valido
5. Backend Flask ativo com DeepSeek acessivel
6. Chatbox renderizado na aba Analise apos resultado da analise
7. Historico de mensagens inicializado (chatMessages: ChatMsg[])

## Pos-condicoes
1. Historico de conversa exibido com mensagens do usuario e respostas da IA
2. Respostas da IA contextualizam desvios encontrados na analise anterior
3. Informacoes complementares disponiveis para fundamentar laudo de recurso
4. Sessao de chat mantida para continuidade da conversa
5. Mensagens renderizadas em markdown com formatacao adequada
6. Dados explorados no chatbox podem ser aproveitados na geracao de laudo (UC-039)
7. Conversa persistida durante a sessao para referencia

## Regras de negocio
1. **RN-CHT-001:** Chatbox aparece somente apos analise concluida (UC-037) com chatSessionId valido
2. **RN-CHT-002:** Mensagens armazenadas em chatMessages: ChatMsg[] com role "user" e "assistant"
3. **RN-CHT-003:** Envio de mensagem via sendMessage(chatSessionId, chatInput) usando sessao existente
4. **RN-CHT-004:** Respostas da IA em markdown renderizadas via ReactMarkdown com remarkGfm
5. **RN-CHT-005:** Estado chatLoading controla indicador de loading durante resposta da IA
6. **RN-CHT-006:** Contexto da analise anterior mantido na sessao para respostas coerentes
7. **RN-CHT-007:** Campo de input (chatInput) limpo apos envio bem-sucedido da mensagem

## Componentes de UI envolvidos
1. Container do chatbox com historico de mensagens (chatMessages) — aparece apos analise
2. Bolhas de mensagem do usuario (role "user") alinhadas a direita
3. Bolhas de resposta da IA (role "assistant") alinhadas a esquerda com markdown renderizado
4. TextInput para digitar pergunta (chatInput) com placeholder de orientacao
5. ActionButton "Enviar" (icone Send) com estado loading (chatLoading/Loader2)
6. Icone MessageSquare no cabecalho do chatbox
7. Scroll automatico para ultima mensagem no historico

## Endpoints ou servicos envolvidos
1. **sendMessage(chatSessionId, chatInput)** — envia mensagem do usuario para IA e recebe resposta
2. **createSession("analise-recurso")** — sessao criada no UC-037, reutilizada aqui
3. **setChatMessages([...prev, userMsg, assistantMsg])** — atualiza historico de mensagens
4. **setChatInput("")** — limpa campo de input apos envio
5. **setChatLoading(true/false)** — controla estado de loading
6. **setChatSessionId(sid)** — sessao definida no UC-037
7. **ReactMarkdown + remarkGfm** — renderiza respostas em markdown

## Dados manipulados
1. **ChatMsg:** role ("user"|"assistant"), content (string) — mensagem individual
2. **chatMessages:** ChatMsg[] — historico completo da conversa
3. **chatInput:** string — texto digitado pelo usuario no campo de input
4. **chatSessionId:** string — ID da sessao de chat criada no UC-037
5. **chatLoading:** boolean — estado de loading durante resposta da IA
6. **analiseResultadoMd:** string — contexto da analise anterior disponivel na sessao
7. **inconsistenciasVencedora:** InconsistenciaVencedora[] — desvios identificados como contexto

## Fluxo principal
1. Apos conclusao da analise (UC-037), chatbox aparece na aba Analise com campo de input e historico vazio
2. Usuario digita pergunta complementar sobre desvio identificado (ex: "Explique melhor o desvio no item 3") no TextInput
3. Usuario clica "Enviar" (icone Send); frontend adiciona mensagem do usuario ao chatMessages e ativa chatLoading
4. Frontend chama sendMessage(chatSessionId, chatInput) que envia pergunta a IA com contexto da analise anterior
5. IA processa pergunta considerando analise previa e retorna resposta detalhada em markdown
6. Frontend adiciona resposta da IA ao chatMessages, desativa chatLoading e limpa chatInput
7. Historico exibe conversa com bolhas de usuario e IA; scroll automatico posiciona na ultima mensagem

## Fluxos alternativos
1. **FA-01 — Pergunta sobre item especifico:** Usuario pergunta sobre inconsistencia especifica; IA detalha com fundamentacao legal
2. **FA-02 — Solicitar sugestao de recurso:** Usuario pede sugestao de argumentacao para recurso; IA elabora texto juridico
3. **FA-03 — Comparar com edital:** Usuario pede comparacao entre proposta vencedora e requisito do edital; IA detalha
4. **FA-04 — Multiplas perguntas:** Usuario faz sequencia de perguntas; cada resposta mantem contexto das anteriores
5. **FA-05 — Copiar resposta:** Usuario seleciona e copia texto da resposta da IA para uso em laudo
6. **FA-06 — Enviar ao FloatingChat:** Usuario transfere conversa para FloatingChat via onSendToChat
7. **FA-07 — Nova analise:** Se usuario executa nova analise (UC-037), chatbox reinicia com novo chatSessionId

## Fluxos de excecao
1. **FE-01 — Sessao de chat invalida:** chatSessionId expirado ou invalido; sendMessage retorna erro; exibe mensagem ao usuario
2. **FE-02 — Input vazio:** Botao Enviar desabilitado ou handler retorna sem acao se chatInput estiver vazio
3. **FE-03 — Erro na IA:** sendMessage retorna erro; mensagem de erro adicionada ao historico como resposta
4. **FE-04 — Timeout de resposta:** Resposta da IA excede tempo limite; chatLoading desativado com mensagem de timeout
5. **FE-05 — Token expirado:** Requisicao retorna 401; frontend redireciona para login
6. **FE-06 — Chatbox nao visivel:** UC-037 nao executado; chatbox nao renderizado; usuario deve executar analise primeiro
7. **FE-07 — Erro de rede:** fetch lanca excecao; mensagem de erro no historico e chatLoading desativado
