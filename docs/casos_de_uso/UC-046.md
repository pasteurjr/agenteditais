# Caso de Uso: UC-046

## Nome
Extrair dados de ata PDF via IA

## Objetivo
Permitir ao usuario extrair dados estruturados (itens, vencedores, precos, quantidades) de uma ata de registro de preco em PDF, utilizando IA (DeepSeek) para processamento e estruturacao automatica do conteudo.

## Escopo
Modulo Atas — AtasPage.tsx > Aba Extrair

## Atores
- Usuario autenticado (Analista Comercial)
- Sistema backend (Flask)
- Motor de IA (DeepSeek) para extracao de dados de PDF
- PNCP (fonte do PDF da ata)

## Gatilho
Usuario preenche a URL da ata ou cola texto do PDF na aba "Extrair" da AtasPage e clica no botao "Extrair Dados".

## Pre-condicoes
1. Usuario esta autenticado no sistema
2. Backend Flask ativo com endpoint POST /api/atas/extrair-pdf
3. URL valida de ata do PNCP ou texto do PDF disponivel
4. DeepSeek API acessivel para processamento de texto
5. Tool tool_extrair_ata registrada no backend
6. Conexao com internet para download do PDF (quando via URL)
7. AtasPage acessivel no menu do sistema

## Pos-condicoes
1. Itens da ata extraidos com descricao, vencedor, valor_unitario e quantidade
2. DataTable exibe itens extraidos em formato estruturado
3. Dados disponibilizados para calculo de saldo ARP (UC-053)
4. Extracao persistida vinculada a ata consultada
5. Itens podem ser revisados e editados pelo usuario
6. Dados de vencedor e preco utilizaveis em analise competitiva
7. Indicador de loading exibido durante processamento da IA

## Regras de negocio
1. **RN-EXT-001:** Extracao aceita dois inputs: URL do PDF ou texto colado manualmente
2. **RN-EXT-002:** IA extrai campos obrigatorios: descricao, vencedor, valor_unitario, quantidade por item
3. **RN-EXT-003:** Tool tool_extrair_ata processa o conteudo e retorna JSON estruturado
4. **RN-EXT-004:** Resposta flexivel: data.itens || data.items (frontend trata 2 formatos)
5. **RN-EXT-005:** URL pode ser pre-preenchida via botao "Extrair" da aba Buscar (UC-045)
6. **RN-EXT-006:** Extracao por URL faz download automatico do PDF antes de processar
7. **RN-EXT-007:** Extracao por texto aceita conteudo colado diretamente sem download

## Componentes de UI envolvidos
1. TextInput para URL do PDF da ata (extractUrl)
2. TextArea para texto colado do PDF (extractTexto)
3. ActionButton "Extrair Dados" com icone FileText e estado de loading
4. DataTable de itens extraidos com colunas: descricao, vencedor, valor_unitario, quantidade
5. Loader2 indicador de processamento durante extracao IA
6. Card informativo com instrucoes de uso (URL ou texto)
7. Formatacao monetaria para valores unitarios (R$ format)

## Endpoints ou servicos envolvidos
1. **POST /api/atas/extrair-pdf** — recebe URL ou texto e retorna itens extraidos via IA
2. **tool_extrair_ata** — tool de IA que processa conteudo do PDF e extrai dados estruturados
3. **DeepSeek API** — motor de IA para parsing e estruturacao do conteudo
4. **PNCP (download)** — download automatico do PDF quando fornecida URL
5. **GET /api/atas/{id}/saldos** — vinculacao posterior dos itens extraidos ao saldo ARP
6. **POST /api/atas/salvar** — persistencia da ata com dados extraidos
7. **Auth middleware** — validacao de token JWT

## Dados manipulados
1. **Input URL:** string com URL do PDF da ata no PNCP
2. **Input texto:** string com conteudo colado do PDF (alternativo a URL)
3. **ExtractedItem:** descricao (string), vencedor (string), valor_unitario (number), quantidade (number)
4. **Payload POST:** { url?: string, texto?: string } — ao menos um obrigatorio
5. **Resposta flexivel:** data.itens || data.items (2 formatos aceitos)
6. **Estado de UI:** extracting (boolean), extractedItems (ExtractedItem[])
7. **Dados processados pela IA:** JSON estruturado com array de itens da ata

## Fluxo principal
1. Usuario acessa aba "Extrair" da AtasPage e visualiza campos de URL e texto para input
2. Usuario preenche URL do PDF (ex: url_pncp da ata encontrada na busca) ou cola texto do PDF manualmente
3. Usuario clica em "Extrair Dados" que seta extracting=true e exibe Loader2
4. Frontend envia POST /api/atas/extrair-pdf com body { url, texto } ao backend
5. Backend executa tool_extrair_ata via DeepSeek que faz download do PDF (se URL) e processa conteudo
6. IA extrai itens estruturados com descricao, vencedor, valor_unitario e quantidade de cada item
7. Frontend recebe resposta, trata formato (itens/items), renderiza DataTable com itens extraidos e seta extracting=false

## Fluxos alternativos
1. **FA-01 — URL pre-preenchida via Buscar:** Usuario clicou "Extrair" em ata da busca (UC-045); extractUrl ja preenchida automaticamente
2. **FA-02 — Extracao por texto colado:** Usuario cola texto do PDF diretamente; backend processa sem download
3. **FA-03 — URL e texto simultaneos:** Ambos preenchidos; backend prioriza URL para extracao
4. **FA-04 — Ata com muitos itens:** DataTable exibe todos os itens com scroll; paginacao se necessario
5. **FA-05 — Revisao manual:** Usuario identifica erro na extracao e corrige manualmente na DataTable
6. **FA-06 — Extracao parcial:** IA extrai apenas parte dos itens; usuario complementa manualmente
7. **FA-07 — Extracao via Chat:** Usuario solicita extracao pelo FloatingChat; mesma tool acionada

## Fluxos de excecao
1. **FE-01 — URL invalida:** Backend nao consegue acessar PDF; retorna erro e frontend exibe mensagem
2. **FE-02 — PDF corrompido:** Download funciona mas conteudo ilegivel; IA retorna itens vazios
3. **FE-03 — DeepSeek indisponivel:** Backend retorna erro de conexao com IA; frontend exibe alerta
4. **FE-04 — Timeout de extracao:** Processamento excede timeout; backend cancela e retorna erro
5. **FE-05 — Token expirado:** Requisicao retorna 401; sistema redireciona para LoginPage
6. **FE-06 — Resposta malformada:** JSON sem itens/items; setExtractedItems recebe array vazio
7. **FE-07 — PDF protegido:** PDF com senha ou restricao de leitura; IA nao consegue processar

### Sequencia de eventos real (UC-AT02)
1. Usuario acessa AtasPage e clica na aba "Extrair"
2. Se veio de UC-AT01, AT02-F02 ja esta pre-preenchido com URL da ata
3. Usuario escolhe fonte: upload PDF em AT02-F01 OU URL em AT02-F02

8. AT02-F05 exibe tabela com dados extraidos — celulas editaveis para correcao

11. AT02-F09 registra concorrentes na base competitiva
12. AT02-F10 exporta CSV para backup externo
