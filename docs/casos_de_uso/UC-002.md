# Caso de Uso: UC-002

## Nome
Cadastro de Empresa

## Objetivo
Permitir que o usuario cadastre e mantenha os dados cadastrais da empresa (CNPJ, razao social, nome fantasia, endereco, contatos e redes sociais) na aba "Dados Cadastrais" da EmpresaPage.

## Escopo
Modulo Empresa — EmpresaPage.tsx > aba Dados Cadastrais / Backend CRUD empresas

## Atores
- Ator primario: Usuario autenticado (gestor da empresa)

## Gatilho
O usuario acessa o menu "Empresa" na navegacao lateral do sistema.

## Pre-condicoes
1. O usuario esta autenticado com JWT valido
2. O sistema (frontend e backend) esta operacional
3. O banco de dados esta acessivel

## Pos-condicoes
1. Os dados cadastrais da empresa estao persistidos na tabela "empresas"
2. Os dados exibidos na tela refletem o estado salvo no banco
3. A empresa fica vinculada ao usuario logado (user_scoped)

## Regras de negocio
- RN-EMP-001: Empresa scoped por usuario (user_scoped no CRUD) — cada usuario ve apenas suas empresas
- RN-EMP-002: CNPJ deve ser unico por empresa
- RN-EMP-003: Razao social e campo obrigatorio
- RN-EMP-004: Campos de redes sociais (website, instagram, linkedin, facebook) estao disponiveis no modelo
- RN-EMP-005: Campos de contato incluem emails e celulares (campos texto livre, multiplos valores separados por virgula)

## Componentes de UI envolvidos
- EmpresaPage (pagina com abas)
- Aba "Dados Cadastrais" (aba padrao ao abrir)
- Icone Building (lucide-react)
- FormField + TextInput para cada campo:
  - CNPJ
  - Razao Social
  - Nome Fantasia
  - Inscricao Estadual
  - Endereco
  - Cidade
  - UF
  - CEP
  - Emails
  - Celulares
  - Website
  - Instagram
  - LinkedIn
  - Facebook
- Botao Salvar (ActionButton com icone Save)
- Botao Novo (ActionButton com icone Plus)
- DataTable para listagem de empresas cadastradas
- Modal de edicao/criacao

## Endpoints ou servicos envolvidos
- GET /api/crud/empresas — lista empresas do usuario (com paginacao)
- POST /api/crud/empresas — cria nova empresa
- PUT /api/crud/empresas/{id} — atualiza dados da empresa
- DELETE /api/crud/empresas/{id} — remove empresa

## Dados manipulados
- Entrada: cnpj, razao_social, nome_fantasia, inscricao_estadual, endereco, cidade, uf, cep, emails, celulares, website, instagram, linkedin, facebook
- Saida: objeto Empresa completo com id gerado
- Armazenamento: tabela "empresas" no MySQL

## Fluxo principal

### Passo 1: Acesso a pagina de empresa
- **Acao:** O ator clica no menu "Empresa" na barra de navegacao lateral
- **Elemento UI:** Item de menu com icone Building e texto "Empresa"
- **Resposta visual:** O sistema renderiza a EmpresaPage com a aba "Dados Cadastrais" ativa por padrao; exibe a listagem de empresas ja cadastradas (DataTable) ou estado vazio
- **Efeito funcional:** GET /api/crud/empresas e invocado para carregar dados existentes
- **Requisicao:** GET /api/crud/empresas com header Authorization: Bearer {token}
- **Persistencia:** Nenhuma (apenas leitura)
- **Evidencia:** Screenshot da tela inicial de empresa (runtime/screenshots/UC-002/01_tela_empresa.png)

### Passo 2: Iniciar cadastro de nova empresa
- **Acao:** O ator clica no botao "Novo" ou "+" para cadastrar uma nova empresa
- **Elemento UI:** ActionButton com icone Plus
- **Resposta visual:** O sistema abre um modal ou formulario inline com todos os campos vazios prontos para preenchimento
- **Efeito funcional:** Estado de edicao e ativado; campos do formulario sao limpos
- **Requisicao:** Nenhuma
- **Persistencia:** Nenhuma
- **Evidencia:** Screenshot do formulario vazio (runtime/screenshots/UC-002/02_formulario_novo.png)

### Passo 3: Preenchimento do CNPJ
- **Acao:** O ator digita o CNPJ da empresa no campo correspondente
- **Elemento UI:** TextInput com label "CNPJ"
- **Resposta visual:** O campo exibe o texto digitado
- **Efeito funcional:** Estado local do formulario e atualizado
- **Requisicao:** Nenhuma
- **Persistencia:** Nenhuma
- **Evidencia:** Screenshot com CNPJ preenchido (runtime/screenshots/UC-002/03_cnpj.png)

### Passo 4: Preenchimento da razao social e demais campos
- **Acao:** O ator preenche razao social, nome fantasia, endereco, cidade, UF, CEP, emails, celulares e redes sociais
- **Elemento UI:** Multiplos TextInput com labels correspondentes
- **Resposta visual:** Cada campo exibe o valor digitado
- **Efeito funcional:** Estado local do formulario e atualizado para cada campo
- **Requisicao:** Nenhuma
- **Persistencia:** Nenhuma
- **Evidencia:** Screenshot do formulario preenchido (runtime/screenshots/UC-002/04_formulario_preenchido.png)

### Passo 5: Salvar empresa
- **Acao:** O ator clica no botao "Salvar"
- **Elemento UI:** ActionButton com icone Save
- **Resposta visual:** O sistema exibe estado de loading no botao; apos sucesso, exibe mensagem de confirmacao e atualiza a listagem de empresas
- **Efeito funcional:** Os dados do formulario sao enviados ao backend para persistencia
- **Requisicao:** POST /api/crud/empresas com body contendo todos os campos preenchidos
- **Persistencia:** Registro criado na tabela "empresas" com id gerado e vinculado ao user_id do token JWT
- **Evidencia:** Screenshot apos salvar com sucesso (runtime/screenshots/UC-002/05_salvo_sucesso.png)

### Passo 6: Verificacao na listagem
- **Acao:** O ator observa a listagem atualizada
- **Elemento UI:** DataTable com colunas de empresa
- **Resposta visual:** A empresa recem-cadastrada aparece na tabela com seus dados (razao social, CNPJ, etc.)
- **Efeito funcional:** GET /api/crud/empresas e re-executado para atualizar a listagem
- **Requisicao:** GET /api/crud/empresas
- **Persistencia:** Nenhuma (apenas leitura)
- **Evidencia:** Screenshot da listagem atualizada (runtime/screenshots/UC-002/06_listagem_atualizada.png)

## Fluxos alternativos

### FA-001: Edicao de empresa existente
1. O ator clica no botao de edicao (icone Pencil) de uma empresa na listagem
2. O sistema carrega os dados da empresa no formulario
3. O ator altera os campos desejados
4. O ator clica em "Salvar"
5. PUT /api/crud/empresas/{id} e invocado com os dados atualizados
6. Listagem e atualizada

### FA-002: Exclusao de empresa
1. O ator clica no botao de exclusao (icone Trash2) de uma empresa na listagem
2. O sistema exibe confirmacao de exclusao
3. O ator confirma
4. DELETE /api/crud/empresas/{id} e invocado
5. Empresa e removida da listagem

## Fluxos de excecao

### FE-001: Erro ao salvar (campos obrigatorios vazios)
1. O ator clica em "Salvar" sem preencher razao social
2. O sistema exibe mensagem de validacao indicando campo obrigatorio
3. O registro nao e salvo

### FE-002: Erro de rede / backend indisponivel
1. O ator clica em "Salvar" mas o backend nao responde
2. O sistema exibe mensagem de erro genérica
3. Os dados permanecem no formulario para nova tentativa

### FE-003: CNPJ duplicado
1. O ator tenta cadastrar empresa com CNPJ ja existente
2. O backend retorna erro de unicidade
3. O sistema exibe mensagem indicando CNPJ ja cadastrado

## Criterios de validacao
1. A aba "Dados Cadastrais" renderiza com todos os campos (CNPJ, razao social, nome fantasia, endereco, cidade, UF, CEP, emails, celulares, website, instagram, linkedin, facebook)
2. Botao "Novo" abre formulario limpo
3. Botao "Salvar" dispara POST /api/crud/empresas com dados corretos
4. Empresa salva aparece na listagem apos reload
5. Edicao carrega dados existentes e salva via PUT
6. Exclusao remove empresa da listagem e do banco
