# Caso de Uso: UC-041

## Nome
Submissao assistida no portal

## Objetivo
Auxiliar o usuario na submissao de recurso ou contra-razao no portal de licitacao, fornecendo checklist de validacao documental, processo guiado em 3 passos (exportar, acessar portal, registrar protocolo) e registro de protocolo para rastreabilidade.

## Escopo
Modulo Recursos — RecursosPage.tsx (aba Laudos — acao Submeter em laudo existente)

## Atores
- Usuario autenticado (Analista Comercial / Juridico)
- Sistema frontend (checklist e fluxo guiado)
- Portal externo de licitacao (ComprasNet, gov.br)
- Laudo de recurso ou contra-razao previamente gerado (UC-039/UC-040)

## Gatilho
Usuario clica no icone "Submeter" (Send) em um laudo existente na DataTable da aba Laudos, abrindo modal de submissao assistida.

## Pre-condicoes
1. Usuario esta autenticado com token JWT valido
2. Laudo de recurso ou contra-razao gerado e salvo no sistema (UC-039 ou UC-040)
3. Laudo com status "rascunho" ou "revisao" (nao protocolado ainda)
4. Conteudo do laudo revisado e aprovado pelo usuario
5. Documentos complementares anexados se necessarios
6. Acesso ao portal externo de licitacao disponivel (ComprasNet, gov.br)
7. RecursosPage com aba Laudos carregada com laudos na DataTable

## Pos-condicoes
1. Checklist de validacao documental verificado e aprovado
2. Laudo exportado em formato adequado para protocolo (DOCX/PDF)
3. Link para portal externo fornecido para acesso direto
4. Numero de protocolo registrado no sistema (submissaoProtocolo)
5. Status do laudo atualizado para "protocolado"
6. Data e hora de protocolo registrados para rastreabilidade
7. Laudo nao mais editavel apos status "protocolado"

## Regras de negocio
1. **RN-SUB-001:** Checklist valida completude documental antes da submissao (RN-SUB-001 da especificacao)
2. **RN-SUB-002:** Processo guiado em 3 passos: (1) Exportar documento, (2) Acessar portal, (3) Registrar protocolo
3. **RN-SUB-003:** Submissao e assistida (nao automatica) — portal nao tem API de submissao
4. **RN-SUB-004:** Numero de protocolo registrado via campo submissaoProtocolo no modal
5. **RN-SUB-005:** Status do laudo atualiza para "protocolado" apos registro de protocolo
6. **RN-SUB-006:** Submissao vinculada ao laudo especifico (submissaoLaudo)
7. **RN-SUB-007:** Validacao impede submissao de laudo sem conteudo ou com campos obrigatorios vazios

## Componentes de UI envolvidos
1. Icone Send na DataTable de laudos — abre modal showSubmissaoModal
2. Modal de submissao assistida com 3 passos sequenciais visuais
3. Checklist de validacao com Checkbox para cada item documental obrigatorio
4. ActionButton "Exportar" (icone Download) — exporta laudo em DOCX/PDF
5. ActionButton/Link "Acessar Portal" (icone ExternalLink) — abre portal externo em nova aba
6. TextInput para numero de protocolo (submissaoProtocolo)
7. ActionButton "Confirmar Protocolo" (icone CheckCircle) para finalizar submissao

## Endpoints ou servicos envolvidos
1. **crudUpdate("recursos-detalhados", id)** — atualiza status para "protocolado" e registra protocolo
2. **setShowSubmissaoModal(true/false)** — controla visibilidade do modal de submissao
3. **setSubmissaoLaudo(laudo)** — define laudo selecionado para submissao
4. **setSubmissaoProtocolo(string)** — gerencia numero de protocolo digitado
5. **setSubmissaoStatus("submetido")** — atualiza status local apos confirmacao
6. **loadLaudos()** — recarrega lista de laudos apos atualizacao de status
7. **window.open(url)** — abre portal externo de licitacao em nova aba

## Dados manipulados
1. **submissaoLaudo:** Laudo | null — laudo selecionado para submissao assistida
2. **submissaoProtocolo:** string — numero de protocolo registrado pelo usuario
3. **submissaoStatus:** "pendente" | "submetido" — estado da submissao no modal
4. **showSubmissaoModal:** boolean — controle de visibilidade do modal
5. **Checklist items:** array de itens de validacao documental com status checked/unchecked
6. **Laudo.status:** atualizado de "rascunho"/"revisao" para "protocolado"
7. **URL do portal:** link para ComprasNet ou gov.br conforme edital

## Fluxo principal
1. Usuario visualiza laudo na DataTable da aba Laudos e clica no icone "Submeter" (Send) para iniciar fluxo de submissao
2. Modal de submissao assistida abre exibindo 3 passos: Exportar, Acessar Portal, Registrar Protocolo
3. Passo 1 — Checklist de validacao: usuario verifica cada item documental (conteudo completo, fundamentacao, assinatura, anexos) marcando Checkboxes
4. Passo 2 — Exportar: usuario clica "Exportar" para baixar laudo em DOCX/PDF; em seguida clica "Acessar Portal" que abre ComprasNet/gov.br em nova aba
5. Passo 3 — Registrar protocolo: usuario submete manualmente no portal externo e retorna ao modal para digitar numero de protocolo
6. Usuario preenche campo submissaoProtocolo com numero obtido no portal e clica "Confirmar Protocolo"
7. Sistema atualiza status do laudo para "protocolado" via crudUpdate, fecha modal, recarrega lista e exibe laudo com badge "protocolado"

## Fluxos alternativos
1. **FA-01 — Checklist incompleto:** Usuario nao marca todos os itens; sistema alerta que validacao esta incompleta mas permite continuar
2. **FA-02 — Exportar somente:** Usuario exporta documento mas nao registra protocolo; status permanece inalterado
3. **FA-03 — Voltar ao passo anterior:** Usuario navega entre passos do modal para revisar itens anteriores
4. **FA-04 — Cancelar submissao:** Usuario fecha modal sem confirmar; nenhuma alteracao de status e feita
5. **FA-05 — Protocolo ja registrado:** Laudo com status "protocolado"; icone Send desabilitado ou abre modal em modo visualizacao
6. **FA-06 — Atualizar protocolo:** Usuario corrige numero de protocolo de laudo ja protocolado
7. **FA-07 — Submissao de contra-razao:** Mesmo fluxo aplica-se a contra-razao (tipo "contra_razao") sem distincao de passos

## Fluxos de excecao
1. **FE-01 — Laudo sem conteudo:** Validacao impede submissao; alerta "Laudo sem conteudo — gere ou edite antes de submeter"
2. **FE-02 — Protocolo vazio:** Botao "Confirmar Protocolo" desabilitado se submissaoProtocolo estiver vazio
3. **FE-03 — Erro ao atualizar status:** crudUpdate falha; modal mantem estado e exibe mensagem de erro
4. **FE-04 — Portal indisponivel:** Link externo nao abre; sistema exibe URL para acesso manual
5. **FE-05 — Token expirado:** Requisicao retorna 401; frontend redireciona para login
6. **FE-06 — Erro de exportacao:** Geracao do DOCX/PDF falha; sistema exibe erro e sugere copiar conteudo markdown
7. **FE-07 — Laudo ja protocolado:** Tentativa de re-submissao; sistema avisa que laudo ja possui protocolo registrado
