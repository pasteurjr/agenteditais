# Caso de Uso: UC-016

## Nome
Calcular score de aderencia

## Objetivo
Calcular automaticamente o score de aderencia de um edital ao portfolio da empresa, avaliando 6 dimensoes (tecnico, documental, complexidade, juridico, logistico, comercial) com pesos ponderados, gerando score geral 0-100 e recomendacao automatica.

## Escopo
Modulo Validacao — ValidacaoPage.tsx

## Atores
- Usuario autenticado (Analista Comercial)
- Sistema backend (Flask)
- Motor de IA (DeepSeek)
- PNCP API (historico de precos e vencedores)

## Gatilho
Usuario seleciona um edital salvo na lista da ValidacaoPage e clica em "Calcular Score" ou "Validar".

## Pre-condicoes
1. Usuario esta autenticado no sistema
2. Edital foi salvo previamente (UC-015) e aparece na ValidacaoPage
3. Portfolio de produtos da empresa esta cadastrado com especificacoes tecnicas
4. Empresa possui documentacao cadastrada (certidoes, atestados)
5. Parametros de score configurados em Parametrizacoes (pesos das 6 dimensoes)
6. Backend Flask ativo com acesso ao DeepSeek para analise IA
7. Dados do edital incluem objeto, itens e requisitos para comparacao

## Pos-condicoes
1. Score geral ponderado (0-100) calculado e exibido no ScoreCircle
2. Scores individuais das 6 dimensoes exibidos em ScoreBar
3. Recomendacao automatica gerada: PARTICIPAR (>=70), AVALIAR (40-69), NAO PARTICIPAR (<40)
4. Pontos positivos e pontos de atencao listados
5. Justificativa textual da recomendacao disponivel
6. Scores persistidos no banco para consulta futura
7. Status do edital atualizado para "avaliando"

## Regras de negocio
1. **RN-VAL-001:** 6 dimensoes de score — tecnico (30%), documental (25%), complexidade (15%), juridico (10%), logistico (10%), comercial (10%)
2. **RN-VAL-002:** Score geral = media ponderada das 6 dimensoes, escala 0-100
3. **RN-VAL-003:** Recomendacao automatica: >=70 PARTICIPAR, 40-69 AVALIAR, <40 NAO PARTICIPAR
4. **RN-VAL-004:** Score logistico possui 4 sub-dimensoes: distancia (30%), historico entregas (25%), custo frete (25%), prazo (20%)
5. **RN-VAL-005:** Score tecnico compara especificacoes do edital com ficha tecnica dos produtos do portfolio
6. **RN-VAL-006:** Score documental verifica checklist de documentos obrigatorios (certidoes, atestados, registros)
7. **RN-VAL-007:** Fatal flaws (impedimentos graves) zeram o score geral independente das dimensoes

## Componentes de UI envolvidos
1. ScoreCircle — score geral ponderado no centro da tela com cor (verde/amarelo/vermelho)
2. ScoreBar — 6 barras horizontais com score de cada dimensao e label
3. StatusBadge — recomendacao (PARTICIPAR/AVALIAR/NAO PARTICIPAR) com cor correspondente
4. Card de pontos positivos — lista de vantagens competitivas identificadas
5. Card de pontos de atencao — lista de riscos e gaps identificados
6. Card de justificativa — texto explicativo da recomendacao
7. Botao "Recalcular" — permite reprocessar score com parametros atualizados

## Endpoints ou servicos envolvidos
1. **POST /api/editais/{id}/validar** — executa validacao completa com IA (6 dimensoes)
2. **GET /api/editais/{id}/scores** — retorna scores por dimensao ja calculados
3. **tool_validar_edital** — tool de IA que analisa edital e gera scores 6D
4. **crudList('produtos')** — portfolio de produtos para comparacao tecnica
5. **crudList('documentos_empresa')** — checklist documental para score documental
6. **GET /api/editais/{id}/itens** — itens do edital para analise de aderencia
7. **DeepSeek API** — motor de IA para analise semantica de aderencia

## Dados manipulados
1. **EditalScores:** tecnico, documental, complexidade, juridico, logistico, comercial (0-100 cada)
2. **Score geral:** media ponderada 0-100 (scoreProfundo.score_geral)
3. **Decisao:** string "PARTICIPAR" | "AVALIAR" | "NAO PARTICIPAR"
4. **Justificativa:** texto explicativo da recomendacao
5. **Pontos positivos:** string[] — lista de vantagens identificadas
6. **Pontos de atencao:** string[] — lista de riscos e gaps
7. **Fatal flaws:** string[] — impedimentos graves que inviabilizam participacao

## Fluxo principal
1. Usuario acessa ValidacaoPage e visualiza lista de editais salvos com status "novo" ou "avaliando"
2. Usuario clica em um edital da lista para abrir o painel de detalhes e analise
3. Usuario clica no botao "Calcular Score" que dispara POST /api/editais/{id}/validar
4. Backend executa tool_validar_edital via DeepSeek, comparando edital com portfolio e documentacao da empresa
5. IA analisa cada uma das 6 dimensoes, gerando score individual (0-100) e pontos relevantes
6. Backend calcula media ponderada (tecnico 30%, documental 25%, complexidade 15%, juridico 10%, logistico 10%, comercial 10%) e gera recomendacao
7. Frontend renderiza ScoreCircle com score geral, 6 ScoreBars, StatusBadge com recomendacao e cards de pontos positivos/atencao

## Fluxos alternativos
1. **FA-01 — Score ja calculado:** Se edital ja possui scores, sistema exibe resultados anteriores com opcao de recalcular
2. **FA-02 — Parametros personalizados:** Usuario altera pesos das dimensoes em Parametrizacoes antes de calcular
3. **FA-03 — Calculo sem portfolio:** Se portfolio esta vazio, score tecnico e calculado como 0 com aviso
4. **FA-04 — Calculo parcial:** Se faltam dados para alguma dimensao, score dessa dimensao e 0 com flag "dados insuficientes"
5. **FA-05 — Fatal flaw detectado:** IA identifica impedimento grave (ex: exige ANVISA e empresa nao tem); score geral zerado
6. **FA-06 — Recalculo apos atualizacao:** Usuario atualiza portfolio e recalcula score para ver impacto
7. **FA-07 — Score via Chat:** Usuario solicita calculo de score pelo FloatingChat; mesma tool e acionada

## Fluxos de excecao
1. **FE-01 — DeepSeek indisponivel:** Backend retorna erro de conexao com IA; frontend exibe alerta e sugere tentar novamente
2. **FE-02 — Edital sem itens:** Calculo de score tecnico prejudicado; sistema avisa que resultado pode ser impreciso
3. **FE-03 — Timeout da IA:** Analise excede 60s; backend cancela e retorna erro de timeout
4. **FE-04 — Dados do portfolio incompletos:** Score tecnico recebe penalidade por falta de especificacoes detalhadas
5. **FE-05 — Erro de parsing da resposta IA:** Resposta do DeepSeek fora do formato esperado; backend aplica fallback com scores padrao
6. **FE-06 — Edital nao encontrado:** ID invalido; backend retorna 404 e frontend redireciona para lista
7. **FE-07 — Pesos invalidos:** Soma dos pesos diferente de 100%; backend normaliza automaticamente e registra log

## Criterios de validacao
1. Score geral e calculado corretamente como media ponderada das 6 dimensoes
2. Cada dimensao possui score individual entre 0 e 100
3. Recomendacao segue regra: >=70 PARTICIPAR, 40-69 AVALIAR, <40 NAO PARTICIPAR
4. Pontos positivos e pontos de atencao sao listados de forma relevante ao edital
5. Fatal flaws detectados zeram o score geral com justificativa
6. Scores sao persistidos no banco e podem ser recuperados posteriormente
7. Recalculo gera novos scores refletindo alteracoes no portfolio ou parametros
