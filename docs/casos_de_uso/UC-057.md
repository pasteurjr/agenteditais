# Caso de Uso: UC-057

## Nome
Chat com IA (sessoes e ferramentas)

## Objetivo
Oferecer interface de chat flutuante acessivel em todas as paginas do sistema, permitindo ao usuario interagir com IA via linguagem natural para executar qualquer das 76 tools registradas (buscar editais, gerar propostas, analisar documentos, etc.), com sessoes persistentes e historico de mensagens.

## Escopo
Modulo Transversal — FloatingChat.tsx, ChatArea.tsx, ChatInput.tsx

## Atores
- Usuario autenticado (qualquer perfil)
- Sistema backend (Flask)
- Motor de IA (DeepSeek) para classificacao de intencao e execucao de tools
- TOOLS_MAP (76 tools registradas em tools.py)

## Gatilho
Usuario clica no balao flutuante do FloatingChat em qualquer pagina do sistema para abrir a interface de chat.

## Pre-condicoes
1. Usuario esta autenticado no sistema (editais_ia_access_token)
2. Backend Flask ativo com endpoints de chat (/api/chat/*)
3. DeepSeek API acessivel para classificacao de intencao e execucao
4. TOOLS_MAP com 76 tools registradas no backend (tools.py)
5. Sessao de chat ativa ou possibilidade de criar nova
6. FloatingChat renderizado em todas as paginas (via App.tsx)
7. Conexao com internet para comunicacao com IA

## Pos-condicoes
1. Mensagem do usuario processada e resposta da IA exibida
2. Intencao classificada e tool adequada executada automaticamente
3. Resposta formatada em markdown com fontes e referencias
4. Mensagem e resposta persistidas na sessao ativa
5. Historico de sessoes disponivel para consulta
6. Upload de arquivos processado quando enviado
7. Interface de chat fechavel sem perder sessao

## Regras de negocio
1. **RN-CHAT-001:** Cada sessao tem historico persistente no banco de dados
2. **RN-CHAT-002:** Upload de arquivos suportado no chat via ChatInput
3. **RN-CHAT-003:** 76 tools registradas em TOOLS_MAP acessiveis via linguagem natural
4. **RN-CHAT-004:** DeepSeek classifica intencao do usuario e roteia para tool adequada
5. **RN-CHAT-005:** Respostas renderizadas com markdown (titulos, listas, codigo, tabelas)
6. **RN-CHAT-006:** Sessoes podem ser criadas, listadas e acessadas individualmente
7. **RN-CHAT-007:** Chat flutuante acessivel em qualquer pagina sem sair do contexto

## Componentes de UI envolvidos
1. FloatingChat — balao flutuante com toggle abrir/fechar em todas as paginas
2. ChatArea — area de mensagens com scroll automatico e renderizacao markdown
3. ChatInput — campo de texto com botao enviar e botao de upload de arquivo
4. Lista de sessoes — sidebar com sessoes anteriores para navegacao
5. Botao "Nova Sessao" — cria sessao de chat nova
6. Indicador de loading — spinner durante processamento da IA
7. Renderizador markdown — formatacao de titulos, listas, codigo, tabelas na resposta

## Endpoints ou servicos envolvidos
1. **POST /api/chat/send** — envia mensagem do usuario e retorna resposta da IA
2. **GET /api/chat/sessions** — lista todas as sessoes de chat do usuario
3. **POST /api/chat/sessions** — cria nova sessao de chat
4. **GET /api/chat/sessions/{id}/messages** — historico de mensagens de uma sessao
5. **DeepSeek API** — classificacao de intencao e execucao de tools
6. **TOOLS_MAP (tools.py)** — mapeamento de 76 tools disponiveis
7. **Auth middleware** — validacao de token JWT em todos os endpoints

## Dados manipulados
1. **Mensagem usuario:** texto livre em linguagem natural + arquivo opcional
2. **Resposta IA:** texto formatado em markdown com resultado da tool executada
3. **Sessao:** id, titulo, created_at, updated_at (persistida no banco)
4. **Historico:** array de mensagens (role: user/assistant, content, timestamp)
5. **Intencao classificada:** mapeamento para nome da tool (ex: tool_buscar_editais, tool_gerar_proposta)
6. **TOOLS_MAP:** dicionario com 76 funcoes registradas e descricoes
7. **Upload:** arquivo enviado pelo usuario (PDF, imagem, etc.) para processamento

## Fluxo principal
1. Usuario clica no balao flutuante do FloatingChat em qualquer pagina do sistema
2. Interface de chat abre mostrando sessao ativa com historico de mensagens anteriores (se existir)
3. Usuario digita mensagem em linguagem natural no ChatInput (ex: "busque editais de seringa descartavel")
4. Frontend envia POST /api/chat/send com conteudo da mensagem e session_id
5. Backend envia mensagem ao DeepSeek que classifica intencao e seleciona tool adequada do TOOLS_MAP
6. Tool selecionada e executada (ex: tool_buscar_editais_fonte) e resultado formatado em markdown
7. Frontend recebe resposta e renderiza na ChatArea com formatacao markdown (titulos, listas, tabelas) e scroll automatico

## Fluxos alternativos
1. **FA-01 — Nova sessao:** Usuario clica "Nova Sessao"; POST /api/chat/sessions cria sessao; historico limpo
2. **FA-02 — Navegar entre sessoes:** Usuario seleciona sessao anterior na sidebar; historico daquela sessao carregado
3. **FA-03 — Upload de arquivo:** Usuario anexa PDF no ChatInput; arquivo processado junto com mensagem
4. **FA-04 — Pergunta sem tool:** Pergunta geral sem tool especifica; DeepSeek responde diretamente sem executar tool
5. **FA-05 — Tool de outra pagina:** Usuario solicita acao de outro modulo (ex: gerar proposta); tool executa independente da pagina atual
6. **FA-06 — Fechar e reabrir:** Usuario fecha chat; sessao mantida; reabrir mostra mesmo historico
7. **FA-07 — Resposta longa:** Resposta da IA com muitas linhas; ChatArea faz scroll automatico para ultimo item

## Fluxos de excecao
1. **FE-01 — DeepSeek indisponivel:** Backend retorna erro de conexao; frontend exibe mensagem de erro na ChatArea
2. **FE-02 — Token expirado:** Requisicao retorna 401; sistema redireciona para LoginPage
3. **FE-03 — Tool nao encontrada:** DeepSeek classifica intencao para tool inexistente; backend retorna erro generico
4. **FE-04 — Timeout de resposta:** Processamento excede timeout; frontend exibe mensagem de timeout
5. **FE-05 — Erro de rede:** fetch falha; frontend exibe mensagem de erro de conexao
6. **FE-06 — Upload muito grande:** Arquivo excede limite; backend rejeita com erro de tamanho
7. **FE-07 — Sessao corrompida:** ID de sessao invalido; backend retorna 404; frontend cria nova sessao
