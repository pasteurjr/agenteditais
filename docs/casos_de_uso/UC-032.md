# Caso de Uso: UC-032

## Nome
Validacao legal do edital com IA

## Objetivo
Analisar o PDF do edital com inteligencia artificial para identificar inconsistencias legais, clausulas restritivas e violacoes da Lei 14.133/2021, exibindo lista de irregularidades com gravidade, lei violada e sugestao de acao (esclarecimento ou impugnacao).

## Escopo
Modulo Impugnacao — ImpugnacaoPage.tsx (aba Validacao Legal)

## Atores
- Usuario autenticado (Analista Comercial / Juridico)
- Sistema backend (Flask)
- Motor de IA (DeepSeek)
- PDF do edital (armazenado no backend)

## Gatilho
Usuario seleciona um edital no SelectInput e clica no botao "Analisar Edital" na aba Validacao Legal da ImpugnacaoPage.

## Pre-condicoes
1. Usuario esta autenticado (token JWT valido no localStorage)
2. Backend Flask esta ativo na porta 5007
3. Edital salvo no sistema com PDF baixado e texto extraido
4. Motor de IA (DeepSeek) acessivel e configurado no .env
5. Edital possui data de abertura futura (prazo de impugnacao nao expirado)
6. Portfolio de produtos cadastrado (para avaliar relevancia das clausulas)
7. Empresa do usuario cadastrada com dados juridicos (CNPJ, razao social)

## Pos-condicoes
1. Lista de inconsistencias exibida na DataTable com trecho, lei violada, gravidade e sugestao
2. Cada inconsistencia classificada como gravidade ALTA, MEDIA ou BAIXA
3. Sugestao de acao indicada para cada item: Esclarecimento ou Impugnacao
4. Fundamentacao juridica apresentada para cada inconsistencia identificada
5. Sugestao de correcao disponivel para cada item irregular
6. Peticao pode ser gerada a partir das inconsistencias encontradas (link com UC-033)
7. Status da validacao registrado para rastreabilidade do edital

## Regras de negocio
1. **RN-VAL-001:** Analise lê texto completo do PDF do edital via tool_validacao_legal_edital no DeepSeek
2. **RN-VAL-002:** Inconsistencias classificadas em gravidade: ALTA (ilegalidade clara), MEDIA (restricao questionavel), BAIXA (melhoria possivel)
3. **RN-VAL-003:** Cada inconsistencia mapeia para lei violada especifica (Lei 14.133/2021, LC 123/2006, etc.)
4. **RN-VAL-004:** Sugestao de acao automatica: gravidade ALTA -> Impugnacao, MEDIA/BAIXA -> Esclarecimento
5. **RN-VAL-005:** Se edital nao possui PDF com texto extraido, sistema exibe aviso sem_pdf e orienta download
6. **RN-VAL-006:** Fundamentacao inclui artigos e incisos da legislacao aplicavel
7. **RN-VAL-007:** Resultado da analise permite geracao automatica de peticao via botao "Gerar Peticao"

## Componentes de UI envolvidos
1. SelectInput com lista de editais salvos (editalId) com label "numero - orgao"
2. ActionButton "Analisar Edital" com icone Scale e estado loading (Loader2)
3. Status text com mensagem de progresso ("Analisando conformidade legal do edital...")
4. DataTable de inconsistencias com colunas: trecho, lei_violada, gravidade (badge colorido), sugestao
5. Badges de gravidade: ALTA (status-badge-error vermelho), MEDIA (status-badge-warning amarelo), BAIXA (status-badge-info azul)
6. Botao "Gerar Peticao" que aparece apos identificar inconsistencias
7. Area de texto com peticao sugerida renderizada em Markdown (ReactMarkdown com remarkGfm)

## Endpoints ou servicos envolvidos
1. **POST /api/editais/{id}/validacao-legal** — executa analise legal do PDF com IA
2. **tool_validacao_legal_edital** — tool DeepSeek que le PDF e identifica inconsistencias
3. **GET /api/editais** — lista editais salvos para SelectInput (getEditais)
4. **createSession("gerar-peticao")** — cria sessao de chat para geracao de peticao
5. **sendMessage(sid, prompt)** — envia prompt para gerar peticao a partir das inconsistencias
6. **localStorage.getItem("editais_ia_access_token")** — token JWT para autenticacao
7. **crudList("recursos")** — lista peticoes ja criadas para o edital

## Dados manipulados
1. **Inconsistencia:** id (number), trecho (string), lei_violada (string), gravidade ("ALTA"|"MEDIA"|"BAIXA"), sugestao ("Esclarecimento"|"Impugnacao")
2. **fundamentacao:** string — artigos e incisos da legislacao violada
3. **sugestao_correcao:** string — texto sugerido para correcao da clausula
4. **validacaoStatus:** string — mensagem de progresso da analise
5. **peticaoSugerida:** string — texto markdown da peticao gerada pela IA
6. **editalId:** string — UUID do edital selecionado para analise
7. **sem_pdf:** boolean — flag indicando ausencia de PDF com texto extraido

## Fluxo principal
1. Usuario acessa ImpugnacaoPage e visualiza aba "Validacao Legal" com SelectInput de editais e botao "Analisar Edital"
2. Usuario seleciona edital no dropdown que exibe "numero - orgao" de cada edital salvo no sistema
3. Usuario clica em "Analisar Edital"; frontend envia POST /api/editais/{id}/validacao-legal com token JWT no header Authorization
4. Backend executa tool_validacao_legal_edital que le texto extraido do PDF do edital e analisa conformidade com Lei 14.133/2021
5. IA identifica clausulas irregulares, mapeia para artigos violados e classifica gravidade (ALTA/MEDIA/BAIXA) com sugestao de acao
6. Backend retorna JSON com lista de inconsistencias (trecho, lei_violada, gravidade, fundamentacao, sugestao_correcao) e flag success
7. Frontend renderiza DataTable com inconsistencias, badges de gravidade coloridos e habilita botao "Gerar Peticao" para criacao de peticao formal

## Fluxos alternativos
1. **FA-01 — Edital sem PDF:** Backend retorna sem_pdf=true; frontend exibe aviso orientando usuario a baixar PDF do edital primeiro
2. **FA-02 — Nenhuma inconsistencia:** Backend retorna lista vazia; frontend exibe mensagem "Nenhuma inconsistencia legal encontrada neste edital"
3. **FA-03 — Gerar peticao apos analise:** Usuario clica em "Gerar Peticao"; sistema cria sessao de chat e gera peticao formal em markdown
4. **FA-04 — Exportar analise:** Usuario exporta lista de inconsistencias em formato PDF para uso juridico externo
5. **FA-05 — Refinar analise:** Usuario adiciona contexto adicional (ex: clausulas especificas) e solicita re-analise focada
6. **FA-06 — Enviar ao chat:** Usuario clica em "Enviar ao Chat" para explorar inconsistencia especifica via FloatingChat
7. **FA-07 — Validacao parcial:** IA identifica trechos com baixa confianca e marca como "requer revisao humana"

## Fluxos de excecao
1. **FE-01 — Erro na IA:** DeepSeek retorna erro ou timeout; frontend exibe "Erro ao analisar edital" no validacaoStatus
2. **FE-02 — Token expirado:** Requisicao retorna 401; frontend redireciona para LoginPage
3. **FE-03 — Edital nao encontrado:** Backend retorna 404; frontend exibe mensagem de erro
4. **FE-04 — PDF corrompido:** Texto extraido ilegivel; IA retorna resultado vazio com aviso de qualidade
5. **FE-05 — Nenhum edital selecionado:** Botao "Analisar Edital" inativo (editalId vazio); handler retorna sem acao
6. **FE-06 — Erro de rede:** fetch lanca excecao; bloco catch exibe erro no console e atualiza status
7. **FE-07 — Resposta malformada:** JSON sem campo inconsistencias; frontend trata graciosamente exibindo mensagem generica

## Criterios de validacao
1. Endpoint POST /api/editais/{id}/validacao-legal retorna JSON com campo inconsistencias (array)
2. Cada inconsistencia possui campos: trecho, lei_violada, gravidade, fundamentacao, sugestao_correcao
3. Badges de gravidade exibem cores corretas: ALTA=vermelho, MEDIA=amarelo, BAIXA=azul
4. Edital sem PDF exibe mensagem de aviso adequada (sem_pdf=true)
5. Botao "Gerar Peticao" aparece somente quando ha inconsistencias identificadas
6. Peticao gerada contém formato formal: Cabecalho, Dos Fatos, Do Direito, Do Pedido
7. Analise completa executa em menos de 60 segundos para edital de tamanho medio
