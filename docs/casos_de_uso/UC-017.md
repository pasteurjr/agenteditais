# Caso de Uso: UC-017

## Nome
Importar lotes, confrontar documentacao e analisar riscos do edital

## Objetivo
Cobrir tres abas da ValidacaoPage que aprofundam a analise do edital: (1) aba "Lotes" — importar itens do PNCP e extrair lotes via IA; (2) aba "Documentos" — confrontar documentacao necessaria com a empresa e reidentificar requisitos; (3) aba "Riscos" — analisar riscos, recorrencia, atas, vencedores e concorrentes do edital.

## Escopo
Modulo Validacao — ValidacaoPage.tsx

## Atores
- Usuario autenticado (Analista Comercial)
- Sistema backend (Flask)
- Motor de IA (DeepSeek)
- PNCP API (dados de referencia)

## Gatilho
- Aba "Lotes": usuario clica em "Buscar Itens no PNCP" ou "Extrair Lotes via IA"
- Aba "Documentos": usuario clica em "Identificar Documentos Exigidos pelo Edital", "Buscar Documentos Exigidos" ou "Verificar Certidoes"
- Aba "Riscos": usuario clica em "Analisar Riscos do Edital", "Rebuscar Atas", "Buscar Vencedores e Precos" ou "Atualizar" concorrentes

## Pre-condicoes
1. Edital salvo e com score calculado (UC-016 executado ou em execucao conjunta)
2. Portfolio de produtos com especificacoes tecnicas detalhadas
3. Documentacao da empresa cadastrada (certidoes, atestados, registros ANVISA)
4. Endereco da empresa cadastrado (para calculo de distancia no score logistico)
5. Historico de participacoes anteriores da empresa (para referencia)
6. Parametros de pesos das dimensoes configurados em Parametrizacoes
7. Backend com acesso ao DeepSeek e PNCP API

## Pos-condicoes
1. Analise detalhada das 6 dimensoes exibida com sub-criterios e justificativas
2. Checklist documental completo com status de cada documento (ok/vencido/faltando/ajustavel)
3. Gaps tecnicos identificados entre edital e portfolio
4. Flags juridicos listados (clausulas restritivas, exigencias atipicas)
5. Score logistico calculado com 4 sub-dimensoes (distancia, historico, frete, prazo)
6. Sinais de mercado e analise competitiva disponibilizados
7. Dados de validacao persistidos para rastreabilidade

## Regras de negocio
1. **RN-DIM-001:** Dimensao tecnica (30%) — compara especificacoes do edital vs ficha tecnica dos produtos; identifica gaps e aderencia trecho-a-trecho
2. **RN-DIM-002:** Dimensao documental (25%) — verifica checklist de documentos obrigatorios: certidoes (CND, FGTS, CNDT), atestados de capacidade, registros (ANVISA, INMETRO)
3. **RN-DIM-003:** Dimensao complexidade (15%) — avalia numero de lotes, itens por lote, exigencias tecnicas atipicas, prazo de entrega vs capacidade
4. **RN-DIM-004:** Dimensao juridica (10%) — identifica clausulas restritivas, exigencias desproporcionais, riscos de impugnacao, conformidade com Lei 14.133/2021
5. **RN-DIM-005:** Dimensao logistica (10%) — sub-dimensoes: distancia (30%), historico entregas (25%), custo frete (25%), prazo (20%)
6. **RN-DIM-006:** Dimensao comercial (10%) — analisa valor estimado vs custos da empresa, margem de lucro potencial, historico de precos no PNCP
7. **RN-DIM-007:** Cada dimensao gera lista de AderenciaTrecho com trechoEdital, aderencia (0-100) e trechoPortfolio correspondente

## Endpoints ou servicos envolvidos

Aba "Lotes":
1. **POST /api/editais/{id}/buscar-itens-pncp** — importa itens para `editais_itens`
2. **POST /api/editais/{id}/lotes/extrair** — extrai e processa lotes via IA
3. **DELETE lote** — remove um lote (endpoint de exclusao de lote)
4. **POST/PUT lote-itens** — move item entre lotes

Aba "Documentos":
5. **GET /api/editais/{id}/documentacao-necessaria** — carrega documentos necessarios
6. **POST /api/editais/{id}/extrair-requisitos** — extrai requisitos do PDF e recarrega documentacao
7. **POST /api/empresa-certidoes/buscar-stream** — reexecuta busca de certidoes (DIVERGENCIA: envia empresa_id: edital.id)
8. **POST /api/chat** — "Buscar Documentos Exigidos" pergunta via chat

Aba "Riscos":
9. **POST /api/editais/{id}/analisar-riscos** — pipeline unificado de riscos
10. **POST /api/editais/{id}/historico-vencedores** — busca atas (Rebuscar Atas)
11. **POST /api/editais/{id}/vencedores-atas** — busca vencedores e precos
12. **GET /api/concorrentes/listar** — lista concorrentes conhecidos

## Dados manipulados
1. **ChecklistItem:** documento (string), status ("ok"|"vencido"|"faltando"|"ajustavel"), validade (date)
2. **AderenciaTrecho:** trechoEdital (string), aderencia (number 0-100), trechoPortfolio (string)
3. **LoteItem:** item (string), tipo ("aderente"|"intruso") — itens do lote vs portfolio
4. **Certificacao:** nome (string), status ("ok"|"pendente"|"vencida")
5. **ReputacaoOrgao:** pregoeiro (string), pagador (string), historico (string)
6. **FlagsJuridicos:** string[] — clausulas restritivas identificadas
7. **SinaisMercado:** string[] — indicadores de competitividade e tendencias

## Fluxo principal — Aba Lotes (UC-CV09)
1. Usuario abre a aba "Lotes"
2. Se nao houver itens, usuario clica em "Buscar Itens no PNCP"
3. Sistema chama POST /api/editais/{id}/buscar-itens-pncp e importa itens para `editais_itens`
4. Com itens carregados, usuario clica em "Extrair Lotes via IA"
5. Sistema chama POST /api/editais/{id}/lotes/extrair
6. A interface exibe lotes, itens por lote, valor estimado e status
7. Usuario pode reprocessar, excluir lotes ou mover itens entre lotes via combo "Mover para"

## Fluxo alternativo — Aba Documentos (UC-CV10)
1. Usuario abre a aba "Documentos"
2. Sistema tenta carregar GET /api/editais/{id}/documentacao-necessaria
3. Interface agrupa os documentos por categoria e mostra status de disponibilidade
4. Usuario pode acionar "Identificar Documentos Exigidos pelo Edital" para chamar POST /api/editais/{id}/extrair-requisitos
5. Ao concluir, a tela recarrega documentacao-necessaria
6. Usuario pode acionar "Buscar Documentos Exigidos" via chat (POST /api/chat)
7. Usuario pode acionar "Verificar Certidoes" que reexecuta a busca de certidoes (DIVERGENCIA: envia empresa_id: edital.id)

## Fluxo alternativo — Aba Riscos (UC-CV11)
1. Usuario abre a aba "Riscos"
2. Usuario clica em "Analisar Riscos do Edital"
3. Sistema chama POST /api/editais/{id}/analisar-riscos
4. A interface exibe riscos por categoria, fatal flaws, flags juridicos e trechos relevantes
5. Usuario pode acionar "Rebuscar Atas" para chamar POST /api/editais/{id}/historico-vencedores
6. Usuario pode acionar "Buscar Vencedores e Precos" para chamar POST /api/editais/{id}/vencedores-atas
7. Usuario pode atualizar a lista de concorrentes conhecidos via GET /api/concorrentes/listar

## Fluxos de excecao
1. **FE-01 — Portfolio vazio:** Score tecnico retorna 0; sistema exibe alerta "Cadastre produtos no portfolio para analise tecnica"
2. **FE-02 — Documentos nao cadastrados:** Score documental retorna 0; sistema lista documentos necessarios para cadastro
3. **FE-03 — Endereco nao cadastrado:** Score logistico nao calcula distancia; sub-dimensao distancia recebe score 50 (neutro)
4. **FE-04 — Erro na analise IA:** DeepSeek retorna resposta incompleta; backend aplica fallback com analise parcial
5. **FE-05 — Edital com formato nao reconhecido:** Itens do edital em formato atipico; IA tenta interpretar com aviso de baixa confianca
6. **FE-06 — Timeout por edital complexo:** Editais com mais de 100 itens podem exceder timeout; backend processa em chunks
7. **FE-07 — Certidao ANVISA nao encontrada:** Score documental penalizado; flag de atencao gerado com orientacao de providencia

## Divergencias encontradas
- ValidacaoPage e a maior pagina do sistema (~2.300 linhas), indicando alta complexidade de UI
- **CRITICO (aba Documentos):** O botao "Verificar Certidoes" envia `POST /api/empresa-certidoes/buscar-stream` com `empresa_id: edital.id`. Isso mistura a identidade do edital com a identidade da empresa, aparentando ser um acoplamento incorreto. A revalidacao documental deve ser tratada como PARCIAL / COM DIVERGENCIA FUNCIONAL ate confirmacao do comportamento correto no backend
- O UC original descrevia validacao em 6 dimensoes genericas; o comportamento real e organizado em abas especificas (Lotes, Documentos, Riscos) com fluxos e endpoints distintos por aba
