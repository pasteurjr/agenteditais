# Caso de Uso: UC-030

## Nome
Exportar dossie PDF/DOCX/ZIP

## Objetivo
Permitir ao usuario exportar a proposta tecnica e comercial em formatos PDF, DOCX ou ZIP consolidado, gerando o dossie completo para submissao ao orgao licitante, incluindo texto da proposta, documentos anexos e comprovantes.

## Escopo
Modulo Proposta — PropostaPage.tsx

## Atores
- Usuario autenticado (Analista Comercial / Redator de Proposta)
- Sistema backend (Flask)
- gerador_documentos.py (motor de exportacao)
- Banco de dados

## Gatilho
Usuario clica no botao "Exportar" (icone Download) na PropostaPage e seleciona o formato desejado (PDF, DOCX ou ZIP) no menu de opcoes.

## Pre-condicoes
1. Proposta com texto tecnico preenchido (UC-026 e UC-027 executados)
2. Validacao ANVISA realizada sem registros vencidos (UC-028, recomendado)
3. Auditoria documental com completude >= 100% (UC-029, recomendado)
4. Proposta em status "revisao" ou "aprovada" (nao rascunho)
5. Backend Flask ativo com gerador_documentos.py funcional
6. Documentos anexos disponiveis no acervo da empresa
7. PropostaPage carregada com proposta selecionada

## Pos-condicoes
1. Arquivo PDF gerado com texto da proposta formatado profissionalmente
2. Arquivo DOCX gerado editavel com estilos e formatacao preservados
3. Arquivo ZIP gerado contendo proposta + documentos anexos consolidados
4. Download iniciado automaticamente no navegador do usuario
5. Log de exportacao registrado com formato, timestamp e usuario
6. Proposta pronta para submissao ao portal (UC-031)
7. Arquivo disponivel para re-download por periodo configurado

## Regras de negocio
1. **RN-EXP-001:** PDF gerado via gerador_documentos.py com cabecalho, rodape e paginacao
2. **RN-EXP-002:** DOCX preserva formatacao Markdown convertida para estilos Word
3. **RN-EXP-003:** ZIP consolida: proposta (PDF/DOCX) + certidoes + ANVISA + atestados + contrato social
4. **RN-EXP-004:** Tamanho maximo do arquivo ZIP definido por configuracao (padrao 50MB)
5. **RN-EXP-005:** Proposta em status "rascunho" nao pode ser exportada (precisa revisao)
6. **RN-EXP-006:** Nome do arquivo segue padrao: Proposta_{numero_edital}_{data}.{extensao}
7. **RN-EXP-007:** Exportacao registra entrada em PropostaLog para auditoria

## Componentes de UI envolvidos
1. Botao "Exportar" (ActionButton com icone Download) que abre menu de formatos
2. Menu dropdown com opcoes: "PDF", "DOCX", "ZIP (Dossie Completo)"
3. Indicador de loading (Loader2) durante geracao do arquivo
4. Card de preview do documento antes da exportacao (opcional)
5. Modal de confirmacao com resumo do conteudo a ser exportado
6. Botao "Fracionar PDF" (icone Scissors) para dividir PDF grande (Smart Split)
7. Botao "Compactar" (icone Archive) para gerar ZIP com compressao otimizada

## Endpoints ou servicos envolvidos
1. **POST /api/propostas/{id}/exportar** — gera arquivo no formato solicitado (PDF/DOCX/ZIP)
2. **gerador_documentos.py** — modulo Python que converte Markdown para PDF e DOCX
3. **crudGet('propostas', id)** — busca dados da proposta para exportacao
4. **crudList('documentos_empresa')** — lista documentos para inclusao no ZIP
5. **crudList('certidoes')** — lista certidoes para inclusao no ZIP
6. **crudCreate('proposta_logs')** — registra log de exportacao
7. **crudList('validacoes_anvisa', { proposta_id })** — inclui resultado ANVISA no dossie

## Dados manipulados
1. **Texto tecnico:** string (markdown) — conteudo da proposta a ser convertido
2. **Formato:** "pdf"|"docx"|"zip" — formato de exportacao selecionado
3. **Arquivo gerado:** blob/binary — arquivo resultante para download
4. **Documentos anexos:** lista de arquivos para inclusao no ZIP
5. **Nome arquivo:** string — nome padronizado do arquivo exportado
6. **Tamanho arquivo:** number (bytes) — tamanho do arquivo gerado
7. **PropostaLog:** proposta_id, acao ("exportacao"), formato, usuario, timestamp

## Fluxo principal
1. Usuario acessa PropostaPage com proposta em status "revisao" ou "aprovada" e clica no botao "Exportar"
2. Sistema exibe menu dropdown com opcoes de formato: PDF, DOCX e ZIP (Dossie Completo)
3. Usuario seleciona formato desejado e sistema exibe Modal de confirmacao com resumo do conteudo
4. Usuario confirma exportacao e sistema dispara POST /api/propostas/{id}/exportar com formato selecionado
5. Backend aciona gerador_documentos.py que converte Markdown para PDF/DOCX com cabecalho, estilos e paginacao
6. Para formato ZIP, sistema consolida proposta + documentos + certidoes + ANVISA em arquivo compactado
7. Frontend recebe arquivo, inicia download automatico no navegador e registra log de exportacao

## Fluxos alternativos
1. **FA-01 — Apenas PDF:** Usuario exporta somente o texto da proposta em PDF sem documentos anexos
2. **FA-02 — Smart Split:** PDF excede limite do portal; usuario usa "Fracionar PDF" para dividir em partes menores
3. **FA-03 — Re-exportar:** Usuario exporta novamente apos edicoes; sistema gera nova versao
4. **FA-04 — Preview antes:** Usuario visualiza preview do documento antes de confirmar exportacao
5. **FA-05 — Exportar com marca d'agua:** Versao de rascunho com marca d'agua "DRAFT" para revisao interna
6. **FA-06 — Selecionar documentos:** Usuario escolhe quais documentos incluir no ZIP manualmente
7. **FA-07 — Exportacao programada:** Sistema agenda exportacao para horario especifico

## Fluxos de excecao
1. **FE-01 — Proposta em rascunho:** Status "rascunho" impede exportacao; sistema exibe "Mude para revisao antes"
2. **FE-02 — Texto vazio:** Proposta sem texto tecnico; sistema exibe "Gere ou edite o texto antes de exportar"
3. **FE-03 — Erro no gerador:** gerador_documentos.py falha; sistema exibe toast de erro com detalhes
4. **FE-04 — ZIP muito grande:** Arquivo excede 50MB; sistema sugere remover documentos ou fracionar
5. **FE-05 — Documentos ausentes:** ZIP sem documentos obrigatorios; sistema alerta mas permite exportacao
6. **FE-06 — Erro de download:** Navegador bloqueia download; sistema fornece link alternativo
7. **FE-07 — Timeout na geracao:** Exportacao excede 60s; sistema retorna erro e sugere tentar novamente
