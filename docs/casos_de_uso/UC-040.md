# Caso de Uso: UC-040

## Nome
Gerar laudo de contra-razao com IA

## Objetivo
Gerar laudo formal de contra-razao com inteligencia artificial quando recurso for interposto contra a empresa do usuario, contendo secoes de defesa (rebater alegacoes) e ataque (apontar falhas do recorrente), para protocolo no orgao licitante.

## Escopo
Modulo Recursos — RecursosPage.tsx (aba Laudos — botao Novo Laudo, tipo "contra_razao")

## Atores
- Usuario autenticado (Analista Juridico / Comercial)
- Sistema backend (Flask)
- Motor de IA (DeepSeek) via tool_gerar_contra_razao
- Templates de contra-razao (CRUD recurso-templates)

## Gatilho
Usuario clica no botao "Novo Laudo" (icone Plus) na aba Laudos da RecursosPage, seleciona tipo "contra_razao" e confirma criacao.

## Pre-condicoes
1. Usuario esta autenticado com token JWT valido
2. Backend Flask ativo com DeepSeek configurado
3. Edital salvo no sistema onde empresa do usuario foi a vencedora
4. Recurso interposto por concorrente contra a empresa (contexto de contra-razao)
5. Templates de contra-razao cadastrados via crudList("recurso-templates") com tipo "contra_razao"
6. Empresa cadastrada com dados juridicos completos (razao social, CNPJ)
7. RecursosPage com aba Laudos acessivel e carregada

## Pos-condicoes
1. Laudo de contra-razao criado e salvo via crudCreate("recursos-detalhados") com status "rascunho"
2. Conteudo inclui secao de defesa: rebate cada alegacao do recorrente com fundamentacao
3. Conteudo inclui secao de ataque: aponta falhas na proposta do recorrente
4. Laudo associado ao edital e empresa recorrente (empresa_alvo)
5. Tipo "contra_razao" registrado no campo tipo do laudo
6. Laudo visivel na DataTable com badge diferenciado de contra-razao
7. Conteudo em markdown pronto para edicao e exportacao

## Regras de negocio
1. **RN-CRA-001:** Laudo possui tipo "contra_razao" diferenciado do tipo "recurso" (RN-REC-004)
2. **RN-CRA-002:** Secao de defesa rebate alegacoes do recorrente com fundamentacao legal e tecnica
3. **RN-CRA-003:** Secao de ataque aponta falhas e inconsistencias na proposta do recorrente
4. **RN-CRA-004:** Geracao via POST /api/recursos com tool_gerar_contra_razao no DeepSeek
5. **RN-CRA-005:** Templates filtrados por tipo "contra_razao" via crudList("recurso-templates")
6. **RN-CRA-006:** Status segue mesmo fluxo: "rascunho" -> "revisao" -> "protocolado" -> "deferido"/"indeferido"
7. **RN-CRA-007:** Campo empresa_alvo identifica empresa recorrente (que interpôs o recurso)

## Componentes de UI envolvidos
1. ActionButton "Novo Laudo" (icone Plus) na ActionBar da aba Laudos — mesmo botao do UC-039
2. Modal showNovoLaudoModal com SelectInput tipo onde usuario seleciona "contra_razao"
3. SelectInput subtipo (administrativo/tecnico), TextInput empresa_alvo (empresa recorrente), SelectInput template
4. TextArea para conteudo do laudo (laudoConteudo) com pre-preenchimento do template de contra-razao
5. ActionButton "Salvar" (icone Save) com estado loading (laudosSaving)
6. DataTable na aba Laudos com badge diferenciado para tipo "contra_razao"
7. Icones de acao: Eye (visualizar), Edit3 (editar), Trash2 (excluir), Download (exportar), Send (submeter)

## Endpoints ou servicos envolvidos
1. **POST /api/recursos** — cria recurso tipo contra_razao com tool_gerar_contra_razao
2. **tool_gerar_contra_razao** — tool DeepSeek que gera secoes defesa e ataque (RN-REC-004)
3. **crudCreate("recursos-detalhados")** — persiste contra-razao no banco de dados
4. **crudList("recursos-detalhados")** — lista laudos incluindo contra-razoes para DataTable
5. **crudList("recurso-templates")** — carrega templates filtrados por tipo
6. **crudUpdate("recursos-detalhados", id)** — atualiza contra-razao (status, conteudo)
7. **crudDelete("recursos-detalhados", id)** — exclui contra-razao

## Dados manipulados
1. **Laudo:** id, edital_id, edital_numero, tipo ("contra_razao"), subtipo, empresa_alvo, template_id, status, conteudo, data_criacao
2. **laudoTipo:** "contra_razao" — tipo selecionado no modal
3. **laudoSubtipo:** "administrativo" | "tecnico" — subtipo da contra-razao
4. **laudoEmpresaAlvo:** string — nome da empresa recorrente (que interpôs o recurso)
5. **laudoTemplateId:** string — ID do template de contra-razao
6. **laudoConteudo:** string — texto editavel com secoes defesa e ataque
7. **laudosSaving:** boolean — estado de loading durante salvamento

## Fluxo principal
1. Usuario acessa aba Laudos na RecursosPage e clica no botao "Novo Laudo" (icone Plus) na ActionBar
2. Modal abre; usuario seleciona tipo "contra_razao" no SelectInput, distinguindo de recurso convencional
3. Usuario preenche: edital vinculado, subtipo (administrativo/tecnico), nome da empresa recorrente e template de contra-razao
4. Sistema pre-preenche TextArea com template selecionado; conteudo inclui estrutura com secoes defesa e ataque
5. Usuario edita conteudo ajustando argumentos de defesa e ataque ao caso concreto; clica "Salvar"
6. Backend executa tool_gerar_contra_razao que gera secao de defesa (rebate alegacoes) e secao de ataque (aponta falhas do recorrente) via DeepSeek
7. Contra-razao salva com status "rascunho"; modal fecha, lista recarrega e novo laudo aparece na DataTable com badge "contra_razao"

## Fluxos alternativos
1. **FA-01 — Editar contra-razao existente:** Usuario clica Edit3; modal reabre com dados pre-preenchidos para ajuste
2. **FA-02 — Alterar status:** Usuario evolui status de "rascunho" para "revisao" e depois "protocolado"
3. **FA-03 — Exportar para protocolo:** Usuario exporta contra-razao em DOCX para protocolar no orgao
4. **FA-04 — Sem template:** TextArea vazio; usuario redige ou solicita geracao completa pela IA
5. **FA-05 — Submissao assistida:** Usuario inicia fluxo de submissao (UC-041) a partir da contra-razao
6. **FA-06 — Comparar com recurso:** Usuario compara contra-razao com recurso do concorrente para validar cobertura
7. **FA-07 — Adicionar anexos:** Usuario anexa documentos complementares (atestados, certidoes) a contra-razao

## Fluxos de excecao
1. **FE-01 — Nenhum edital selecionado:** Botao Salvar desabilitado se laudoEditalId vazio
2. **FE-02 — Empresa recorrente nao informada:** Validacao alerta que empresa_alvo e obrigatoria
3. **FE-03 — Erro ao salvar:** crudCreate retorna erro; frontend exibe mensagem e mantem modal
4. **FE-04 — Template nao carregado:** crudList falha; templates vazio; usuario digita manualmente
5. **FE-05 — Erro na IA:** tool_gerar_contra_razao falha; laudo salvo sem conteudo gerado automaticamente
6. **FE-06 — Token expirado:** Requisicao retorna 401; redirecionamento para login
7. **FE-07 — Conflito tipo/template:** Template de recurso selecionado para contra-razao; sistema aceita mas conteudo pode ser inadequado
