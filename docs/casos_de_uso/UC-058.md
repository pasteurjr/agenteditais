# Caso de Uso: UC-058

## Nome
CRUD generico de tabelas

## Objetivo
Oferecer componente CRUD generico reutilizavel (CrudPage) para gestao de todas as tabelas do sistema, suportando listagem paginada, busca, criacao, edicao e exclusao de registros, com configuracao declarativa de campos, hierarquia pai-filho e formatacao customizada.

## Escopo
Modulo Transversal — CrudPage.tsx + crudTables.tsx (45+ configuracoes de tabela)

## Atores
- Usuario autenticado (qualquer perfil)
- Sistema backend (Flask)
- API CRUD generica (/api/crud/{tabela})
- Banco de dados MySQL

## Gatilho
Usuario acessa qualquer pagina de cadastro no menu que utiliza o componente CrudPage, ou modulos internos chamam crudList/crudCreate/crudUpdate/crudDelete programaticamente.

## Pre-condicoes
1. Usuario esta autenticado no sistema (editais_ia_access_token via useAuth)
2. Backend Flask ativo com endpoints CRUD genericos
3. Tabela desejada registrada em ALL_CRUD_CONFIGS (crudTables.tsx)
4. Banco de dados MySQL com tabela correspondente criada
5. Configuracao da tabela com fields, title, icon e searchPlaceholder definidos
6. API client CRUD (crud.ts) importado no componente
7. Endpoint /api/crud/{tabela} acessivel e funcional

## Pos-condicoes
1. Registros listados em DataTable paginada com busca
2. Novo registro criado via Modal de formulario e persistido no banco
3. Registro existente editado via Modal e atualizado no banco
4. Registro excluido com confirmacao e removido do banco
5. DataTable atualizada automaticamente apos cada operacao CRUD
6. Contagem total de registros exibida
7. Relacoes pai-filho gerenciadas via parentFk e parentTable

## Regras de negocio
1. **RN-CRUD-001:** 45+ tabelas configuradas em ALL_CRUD_CONFIGS com definicao declarativa
2. **RN-CRUD-002:** Cada configuracao define: table, title, icon, fields[], searchPlaceholder
3. **RN-CRUD-003:** Fields suportam tipos: text, number, select, textarea, date, boolean
4. **RN-CRUD-004:** Hierarquia suportada via parentFk, parentTable, parentLabelField (ex: produto > especificacao)
5. **RN-CRUD-005:** Busca filtra registros por termo em campos configurados
6. **RN-CRUD-006:** Paginacao com limit e offset para tabelas grandes
7. **RN-CRUD-007:** Campos com render customizado para formatacao especial (moeda, data, badge)

## Componentes de UI envolvidos
1. CrudPage — componente wrapper que recebe config e renderiza CRUD completo
2. DataTable — tabela com colunas configuradas, paginacao e ordenacao
3. TextInput de busca — filtro por termo com searchPlaceholder customizado
4. Modal de criacao — formulario gerado dinamicamente a partir de fields[]
5. Modal de edicao — formulario pre-preenchido com dados do registro selecionado
6. ActionButton "Novo" — abre modal de criacao com icone Plus
7. Botoes de acao por linha: Editar (icone Pencil) e Excluir (icone Trash) com confirmacao

## Endpoints ou servicos envolvidos
1. **GET /api/crud/{tabela}?limit=&offset=&search=&filters=** — lista registros paginados
2. **POST /api/crud/{tabela}** — cria novo registro
3. **PUT /api/crud/{tabela}/{id}** — atualiza registro existente
4. **DELETE /api/crud/{tabela}/{id}** — exclui registro
5. **crudList(tabela, options)** — funcao wrapper do API client (crud.ts)
6. **crudCreate(tabela, data)** — funcao wrapper de criacao
7. **crudUpdate(tabela, id, data) / crudDelete(tabela, id)** — funcoes de atualizacao e exclusao

## Dados manipulados
1. **Config:** table (string), title (string), icon (React element), fields (FieldConfig[]), searchPlaceholder
2. **FieldConfig:** name, label, type (text/number/select/textarea/date/boolean), options?, required?, render?
3. **Registro:** objeto generico com campos definidos pela tabela (ex: { id, nome, descricao, ... })
4. **Paginacao:** limit (itens por pagina), offset (ponto de inicio), total (contagem)
5. **Busca:** search (string) aplicado como filtro server-side
6. **Hierarquia:** parentFk (chave estrangeira), parentTable (tabela pai), parentLabelField (campo de exibicao)
7. **Grandparent:** grandparentTable, grandparentFk, grandparentLabelField (3 niveis de hierarquia)

## Fluxo principal
1. Usuario acessa pagina de cadastro no menu; App.tsx renderiza CrudPage com config da tabela correspondente
2. CrudPage inicializa e chama crudList(table, { limit, offset, search }) para carregar registros
3. DataTable renderiza registros com colunas definidas em fields[] e paginacao
4. Usuario clica em "Novo" (icone Plus); Modal de criacao abre com formulario gerado a partir de fields[]
5. Usuario preenche campos e clica "Salvar"; CrudPage chama crudCreate(table, data) e POST persiste registro
6. CrudPage recarrega lista; DataTable exibe novo registro
7. Para edicao/exclusao: usuario clica Editar/Excluir na linha; Modal pre-preenchido ou confirmacao de exclusao

## Fluxos alternativos
1. **FA-01 — Busca:** Usuario digita no campo de busca; lista filtrada server-side; DataTable atualiza
2. **FA-02 — Edicao:** Usuario clica Editar; Modal abre pre-preenchido; crudUpdate persiste alteracoes
3. **FA-03 — Exclusao:** Usuario clica Excluir; confirmacao exibida; crudDelete remove registro
4. **FA-04 — Tabela hierarquica:** CrudPage com parentFk carrega registros filtrados pelo pai selecionado
5. **FA-05 — Paginacao:** Muitos registros; usuario navega entre paginas via controles de paginacao
6. **FA-06 — Campo select:** Field tipo "select" renderiza dropdown com options predefinidas
7. **FA-07 — Uso programatico:** Outros modulos chamam crudCreate/crudList diretamente (ex: ProducaoPage chama crudCreate("contratos"))

## Fluxos de excecao
1. **FE-01 — Tabela inexistente:** Backend retorna 404 para tabela nao encontrada; frontend exibe erro
2. **FE-02 — Token expirado:** Requisicao retorna 401; sistema redireciona para LoginPage
3. **FE-03 — Erro de validacao:** Backend rejeita dados invalidos; frontend exibe mensagem de erro
4. **FE-04 — Registro nao encontrado:** PUT/DELETE com ID inexistente; backend retorna 404
5. **FE-05 — Erro de rede:** fetch falha; try/catch captura e loga
6. **FE-06 — Conflito de exclusao:** Registro referenciado por outra tabela; backend retorna erro de foreign key
7. **FE-07 — Config incompleta:** fields[] vazio ou malformado; CrudPage renderiza sem colunas
