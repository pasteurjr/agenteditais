# Caso de Uso: UC-037

## Nome
Analisar proposta vencedora com IA

## Objetivo
Analisar a proposta vencedora de uma licitacao utilizando inteligencia artificial para identificar inconsistencias tecnicas e juridicas, desvios em relacao ao edital e motivacoes para recurso, fornecendo base fundamentada para elaboracao de laudo.

## Escopo
Modulo Recursos — RecursosPage.tsx (aba Analise)

## Atores
- Usuario autenticado (Analista Comercial / Juridico)
- Sistema backend (Flask)
- Motor de IA (DeepSeek) via tool_analisar_proposta_vencedora
- PDF do edital e proposta vencedora

## Gatilho
Usuario seleciona edital na aba Analise, opcionalmente cola texto da proposta vencedora, e clica no botao "Analisar Vencedora".

## Pre-condicoes
1. Usuario esta autenticado com token JWT valido
2. Backend Flask ativo na porta 5007 com DeepSeek configurado
3. Edital salvo no sistema com resultado de licitacao registrado (ha vencedor)
4. PDF do edital baixado e texto extraido no backend
5. Texto da proposta vencedora disponivel (colado pelo usuario ou extraido do sistema)
6. RecursosPage com aba Analise acessivel e carregada
7. Portfolio da empresa cadastrado para comparacao tecnica

## Pos-condicoes
1. Analise completa exibida em markdown (analiseResultadoMd) com resumo de desvios
2. Lista de inconsistencias da proposta vencedora exibida na DataTable
3. Cada inconsistencia classificada por gravidade (ALTA/MEDIA/BAIXA) com motivacao de recurso
4. Sessao de chat criada para perguntas complementares (UC-038)
5. Chatbox interativo habilitado para explorar desvios identificados
6. Dados disponiveis para geracao de laudo de recurso (UC-039)
7. Analise persistida para rastreabilidade

## Regras de negocio
1. **RN-ANA-001:** Analise usa endpoint POST /api/editais/{id}/analisar-vencedora com tool_analisar_proposta_vencedora
2. **RN-ANA-002:** Texto da proposta vencedora pode ser fornecido pelo usuario (propostaVencedoraTexto) ou nulo
3. **RN-ANA-003:** IA compara proposta vencedora contra requisitos do edital e identifica desvios
4. **RN-ANA-004:** Inconsistencias mapeadas com: item, inconsistencia (descricao), motivacao_recurso, gravidade
5. **RN-ANA-005:** Apos analise, sessao de chat criada automaticamente para chatbox interativo (chatSessionId)
6. **RN-ANA-006:** Gravidade classificada: ALTA (desvio critico), MEDIA (desvio relevante), BAIXA (desvio menor)
7. **RN-ANA-007:** Resultado inclui analise_ia (markdown), resumo e lista de inconsistencias estruturada

## Componentes de UI envolvidos
1. SelectInput para selecao de edital (analiseEditalId) com opcoes "numero - orgao"
2. TextArea para colar texto da proposta vencedora (propostaVencedoraTexto) — opcional
3. ActionButton "Analisar Vencedora" com icone Search e estado loading (analiseLoading/Loader2)
4. Area de resultado em markdown renderizado (ReactMarkdown com remarkGfm) para analiseResultadoMd
5. DataTable de inconsistencias com colunas: item, inconsistencia, motivacao_recurso, gravidade (badge)
6. Badges de gravidade: ALTA (error vermelho), MEDIA (warning amarelo), BAIXA (info azul)
7. Chatbox interativo que aparece apos analise (UC-038) com campo de input e historico de mensagens

## Endpoints ou servicos envolvidos
1. **POST /api/editais/{id}/analisar-vencedora** — executa analise da proposta vencedora com IA
2. **tool_analisar_proposta_vencedora** — tool DeepSeek que analisa desvios e inconsistencias
3. **GET /api/editais** — lista editais para SelectInput (getEditais)
4. **createSession("analise-recurso")** — cria sessao de chat para chatbox interativo pos-analise
5. **localStorage.getItem("editais_ia_access_token")** — token JWT para autenticacao
6. **fetch()** — chamada direta ao endpoint de analise com body JSON
7. **sendMessage(sid, prompt)** — envia perguntas complementares no chatbox (UC-038)

## Dados manipulados
1. **analiseEditalId:** string — UUID do edital selecionado para analise
2. **propostaVencedoraTexto:** string — texto colado da proposta vencedora (opcional)
3. **analiseResultadoMd:** string — resultado da analise em markdown retornado pela IA
4. **InconsistenciaVencedora:** id, item, inconsistencia, motivacao, gravidade ("ALTA"|"MEDIA"|"BAIXA")
5. **inconsistenciasVencedora:** InconsistenciaVencedora[] — lista de desvios identificados
6. **chatSessionId:** string — ID da sessao de chat criada para chatbox interativo
7. **analiseLoading:** boolean — estado de loading durante analise

## Fluxo principal
1. Usuario acessa RecursosPage e navega para aba "Analise" que exibe seletor de edital e campo de texto opcional
2. Usuario seleciona edital no dropdown e opcionalmente cola texto da proposta vencedora no TextArea
3. Usuario clica em "Analisar Vencedora"; frontend envia POST /api/editais/{id}/analisar-vencedora com token e proposta_vencedora_texto
4. Backend executa tool_analisar_proposta_vencedora que compara proposta vencedora contra requisitos do edital usando DeepSeek
5. IA identifica inconsistencias tecnicas e juridicas, classifica gravidade e gera motivacoes de recurso para cada desvio
6. Backend retorna JSON com success, analise_ia (markdown), resumo e lista de inconsistencias; frontend renderiza resultado
7. Frontend exibe analise em markdown, DataTable de inconsistencias com badges de gravidade e habilita chatbox interativo para perguntas

## Fluxos alternativos
1. **FA-01 — Sem texto da proposta:** Usuario nao cola texto; IA analisa apenas com base no edital e dados do sistema
2. **FA-02 — Nenhuma inconsistencia:** IA nao identifica desvios; analise exibe "proposta conforme" com recomendacao de nao recorrer
3. **FA-03 — Explorar via chatbox:** Apos analise, usuario faz perguntas no chatbox interativo (UC-038) para aprofundar
4. **FA-04 — Gerar laudo:** Usuario clica para gerar laudo de recurso (UC-039) a partir das inconsistencias encontradas
5. **FA-05 — Re-analisar:** Usuario ajusta texto da proposta e solicita nova analise; resultados anteriores substituidos
6. **FA-06 — Exportar analise:** Usuario exporta resultado markdown em PDF para uso externo
7. **FA-07 — Enviar ao chat:** Usuario envia inconsistencia especifica ao FloatingChat para analise mais profunda

## Fluxos de excecao
1. **FE-01 — Nenhum edital selecionado:** Handler retorna sem acao se analiseEditalId estiver vazio
2. **FE-02 — Erro na IA:** Backend retorna success=false com mensagem de erro; frontend exibe erro no analiseResultadoMd
3. **FE-03 — Token expirado:** Requisicao retorna 401; frontend redireciona para login
4. **FE-04 — Edital sem PDF:** IA nao consegue analisar sem texto do edital; retorna mensagem de erro orientando download
5. **FE-05 — Timeout de analise:** Analise excede tempo limite; backend retorna resultados parciais
6. **FE-06 — Erro de rede:** fetch lanca excecao; bloco catch registra erro no console
7. **FE-07 — Resposta malformada:** JSON sem campo inconsistencias; frontend trata exibindo apenas analise_ia markdown
