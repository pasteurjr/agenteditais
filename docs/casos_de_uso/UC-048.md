# Caso de Uso: UC-048

## Nome
Cadastrar contrato

## Objetivo
Permitir ao usuario cadastrar contratos de licitacao na ProducaoPage, registrando dados essenciais (numero, orgao, objeto, valor, periodo de vigencia), com criacao automatica em caso de vitoria (UC-042) ou manual via formulario.

## Escopo
Modulo Contratos — ProducaoPage.tsx > Aba Contratos

## Atores
- Usuario autenticado (Gestor de Contratos)
- Sistema backend (Flask)
- CRUD generico (crudCreate)
- Modulo de Follow-up (criacao automatica)

## Gatilho
Usuario clica no botao "Novo Contrato" na aba Contratos da ProducaoPage, ou contrato e criado automaticamente apos registro de vitoria no Follow-up (UC-042).

## Pre-condicoes
1. Usuario esta autenticado no sistema (editais_ia_access_token)
2. Backend Flask ativo com CRUD /api/crud/contratos
3. ProducaoPage acessivel no menu do sistema
4. API crudCreate importada e disponivel no frontend
5. Dados do contrato disponíveis (numero, orgao, objeto, valor)
6. Tabela de contratos existente no banco de dados
7. Em criacao automatica: edital com resultado "vitoria" registrado

## Pos-condicoes
1. Contrato persistido no banco com status "vigente"
2. DataTable de contratos atualizada com novo registro
3. StatCards atualizados: Total, Vigentes, A Vencer (30d), Alertas
4. Contrato disponivel para registro de entregas (UC-049), aditivos (UC-051) e designacoes (UC-052)
5. Contrato aparece no cronograma de entregas (UC-050)
6. Contrato incluido no dashboard Contratado x Realizado (UC-054)
7. Modal de cadastro fechado e formulario resetado

## Regras de negocio
1. **RN-CON-CRIA-001:** Status inicial do contrato = "vigente" (setado automaticamente)
2. **RN-CON-CRIA-002:** 4 status possiveis: vigente, encerrado, rescindido, suspenso
3. **RN-CON-CRIA-003:** valor_total convertido para float via parseFloat (default 0)
4. **RN-CON-CRIA-004:** Campos obrigatorios: numero_contrato, orgao, objeto, valor_total
5. **RN-CON-CRIA-005:** Contrato criado via Follow-up herda dados do edital (numero, orgao, valor)
6. **RN-CON-CRIA-006:** StatCard "A Vencer" calcula contratos com data_fim dentro de 30 dias
7. **RN-CON-CRIA-007:** crudCreate("contratos") utiliza CRUD generico reutilizavel (UC-058)

## Componentes de UI envolvidos
1. TabPanel aba "Contratos" com icone Package na ProducaoPage
2. StatCards com 4 metricas: Total, Vigentes, A Vencer (30d), Alertas (icones Package, CheckCircle, Clock, AlertTriangle)
3. ActionButton "Novo Contrato" com icone Plus para abrir modal
4. Modal de cadastro com FormFields: numero_contrato, orgao, objeto, valor_total, data_inicio, data_fim
5. DataTable de contratos com colunas: Numero, Orgao, Objeto, Valor, Termino, Status, Acao
6. Badge de status colorido: vigente (verde), encerrado (cinza), rescindido (vermelho), suspenso (amarelo)
7. ActionButton "Selecionar" por linha para navegar entre abas do contrato

## Endpoints ou servicos envolvidos
1. **CRUD /api/crud/contratos** — CRUD completo (list, create, update, delete)
2. **crudList("contratos")** — lista contratos com limit e filtros
3. **crudCreate("contratos")** — cria novo contrato via API generica
4. **GET /api/contratos/{id}/cronograma** — cronograma de entregas (carregado ao selecionar)
5. **GET /api/contratos/{id}/aditivos** — aditivos do contrato (carregado ao selecionar)
6. **GET /api/contratos/{id}/designacoes** — designacoes do contrato (carregado ao selecionar)
7. **crudList("contrato-entregas")** — entregas do contrato (carregado ao selecionar)

## Dados manipulados
1. **ContratoAPI:** id, numero_contrato, orgao, objeto, valor_total, data_assinatura, data_inicio, data_fim, status, edital_id
2. **Form state:** numero_contrato, orgao, objeto, valor_total (string), data_inicio, data_fim
3. **Payload crudCreate:** { ...formContrato, valor_total: parseFloat(), status: "vigente" }
4. **StatCards:** total (contratos.length), vigentes (filter status=vigente), a_vencer (filter 0<dias<=30)
5. **Badge cores:** vigente=#dcfce7/#166534, encerrado=#f3f4f6/#374151, rescindido=#fee2e2/#991b1b, suspenso=#fef3c7/#92400e
6. **Formatacao valor:** R$ com toLocaleString("pt-BR", { minimumFractionDigits: 2 })
7. **Data formatada:** toLocaleDateString("pt-BR") para exibicao

## Fluxo principal
1. Usuario acessa ProducaoPage e visualiza aba "Contratos" com StatCards (Total, Vigentes, A Vencer, Alertas) e DataTable
2. Usuario clica no botao "Novo Contrato" (icone Plus) que abre Modal de cadastro
3. Usuario preenche campos: numero_contrato, orgao, objeto, valor_total, data_inicio, data_fim
4. Usuario clica em "Salvar" e frontend chama crudCreate("contratos") com payload incluindo status="vigente"
5. Backend persiste contrato via CRUD generico e retorna id gerado
6. Frontend fecha Modal, reseta formulario e chama fetchContratos() para atualizar lista
7. DataTable exibe novo contrato com badge "vigente" verde e StatCards atualizam totais

## Fluxos alternativos
1. **FA-01 — Criacao automatica por vitoria:** Follow-up (UC-042) registra vitoria; backend cria contrato automaticamente herdando dados do edital
2. **FA-02 — Contrato sem datas:** Usuario nao preenche data_inicio/data_fim; contrato criado com datas nulas
3. **FA-03 — Editar contrato:** Usuario usa CRUD update para alterar dados; DataTable atualiza
4. **FA-04 — Selecionar contrato:** Usuario clica "Selecionar" na linha; sistema carrega entregas, cronograma, aditivos e designacoes
5. **FA-05 — Filtrar por status:** DataTable permite filtragem por status (vigente/encerrado/rescindido/suspenso)
6. **FA-06 — Contrato proximo do vencimento:** StatCard "A Vencer" destaca contratos com data_fim em menos de 30 dias
7. **FA-07 — Cancelar cadastro:** Usuario fecha Modal sem salvar; formulario resetado sem efeito

## Fluxos de excecao
1. **FE-01 — Erro no crudCreate:** API retorna erro; try/catch loga no console sem feedback visual
2. **FE-02 — Token expirado:** Requisicao retorna 401; sistema redireciona para LoginPage
3. **FE-03 — Valor invalido:** parseFloat retorna NaN; valor_total salvo como 0 (default)
4. **FE-04 — Numero duplicado:** Backend rejeita numero_contrato ja existente; erro capturado no catch
5. **FE-05 — Campos obrigatorios vazios:** CRUD aceita mas contrato fica incompleto; usuario deve corrigir
6. **FE-06 — Erro de rede:** fetch falha; try/catch captura e exibe no console
7. **FE-07 — Data inconsistente:** data_inicio > data_fim; backend aceita mas cronograma fica incorreto
