# Caso de Uso: UC-010

## Nome
Configuracao de Fontes de Editais

## Objetivo
Permitir que o administrador configure as fontes de editais do sistema, incluindo listar fontes disponiveis, ativar/desativar fontes e configurar parametros de conexao (tipo e URL).

## Escopo
Modulo Parametrizacoes — aba/secao Fontes de Editais

## Atores
- Administrador (operador do sistema com permissao de configuracao)

## Gatilho
O administrador acessa a ParametrizacoesPage e navega para a secao de Fontes de Editais.

## Pre-condicoes
1. Usuario autenticado no sistema
2. Backend ativo na porta 5007
3. Tabela fontes-editais existente no banco de dados
4. Ao menos uma fonte pre-cadastrada (PNCP, ComprasNet, BEC, etc.)

## Pos-condicoes
1. Fontes de editais configuradas e persistidas no banco
2. Fontes ativadas serao utilizadas na busca de editais (CaptacaoPage)
3. Fontes desativadas serao ignoradas na busca
4. UUID de cada fonte registrado para uso pelo frontend e backend

## Regras de negocio
- RN-PAR-002: Fontes de edital possuem UUID unico. Frontend envia UUID e backend resolve nome via DB
- RN-UC010-001: Cada fonte tem tipo: "api" (conexao direta com API oficial) ou "scraper" (busca via Brave Search/Serper)
- RN-UC010-002: Fontes com tipo "api" possuem URL da API direta (ex: PNCP = pncp.gov.br/api)
- RN-UC010-003: Fontes com tipo "scraper" usam Brave Search API para extracoes web
- RN-UC010-004: A flag "ativa" controla se a fonte sera considerada nas buscas de captacao
- RN-UC010-005: PNCP e a fonte principal com tipo "api" e estrategia hibrida (Search API + detalhes)
- RN-UC010-006: Chave de API do Brave Search e configurada via .env (SCRAPE_API=brave), nao na interface de fontes

## Componentes de UI envolvidos
- ParametrizacoesPage.tsx — secao Fontes de Editais
- Lista/tabela de fontes com colunas: nome, tipo (api/scraper), URL, status (ativa/inativa)
- StatusBadge para indicar se fonte esta ativa (verde) ou inativa (vermelho/cinza)
- Toggle/Checkbox para ativar/desativar fonte
- Icones: Globe para fontes, Play/Pause para ativar/desativar, Lock para fontes obrigatorias
- Card com loadingFontes (Loader2) e errorFontes
- ActionButton para salvar alteracoes

## Endpoints ou servicos envolvidos
- GET /api/crud/fontes-editais — lista todas as fontes (loadFontes)
- PUT /api/crud/fontes-editais/{id} — atualiza fonte (ativar/desativar, editar URL)
- POST /api/crud/fontes-editais — criar nova fonte
- DELETE /api/crud/fontes-editais/{id} — excluir fonte

## Dados manipulados
- Fonte: id (UUID), nome (string), tipo ("api" | "scraper"), url (string), ativa (boolean)
- Exemplos de fontes:
  - PNCP (api, https://pncp.gov.br/api, ativa)
  - ComprasNet (scraper, URL scraper, ativa/inativa)
  - BEC-SP (scraper, URL scraper, ativa/inativa)
  - Licitacoes-e (scraper, URL scraper, ativa/inativa)

## Fluxo principal

### Passo 1: Acessar Parametrizacoes
**Acao do ator:** O administrador clica no menu "Parametrizacoes" na barra lateral.
**Elemento UI:** Item de menu "Parametrizacoes" com icone Settings.
**Resposta visual:** ParametrizacoesPage e carregada. Secoes de configuracao sao exibidas organizadas por abas ou cards.
**Efeito funcional:** Pagina inicializa e dispara carregamento de dados (loadFontes, loadParametros, loadModalidades, etc.).
**Requisicao de rede:** GET /api/crud/fontes-editais (loadFontes), GET /api/crud/parametros-score (loadParametros), entre outros.
**Persistencia:** Nenhuma.
**Evidencia esperada:** Screenshot da ParametrizacoesPage com secoes visiveis.

### Passo 2: Visualizar lista de fontes
**Acao do ator:** O administrador localiza a secao "Fontes de Editais" na pagina.
**Elemento UI:** Card com titulo "Fontes de Editais" com icone Globe. Lista de fontes com nome, tipo, URL e status.
**Resposta visual:** Se loadingFontes === true, Loader2 animado. Apos carregamento, lista de fontes exibida com StatusBadge (verde = ativa, cinza = inativa). Tipo exibido como badge ("api" ou "scraper").
**Efeito funcional:** Estado fontes populado com dados do backend.
**Requisicao de rede:** Ja disparada no Passo 1 (loadFontes).
**Persistencia:** Nenhuma.
**Evidencia esperada:** Screenshot da lista de fontes com status de cada uma.

### Passo 3: Ativar/desativar fonte
**Acao do ator:** O administrador clica no toggle/checkbox de uma fonte para alterar seu status ativo/inativo.
**Elemento UI:** Checkbox ou toggle ao lado do nome da fonte. Icones Play (ativar) / Pause (desativar).
**Resposta visual:** StatusBadge muda de cor (verde <-> cinza). Toggle reflete novo estado.
**Efeito funcional:** Estado local da fonte atualizado (ativa = !ativa).
**Requisicao de rede:** PUT /api/crud/fontes-editais/{id} com { ativa: true/false } via crudUpdate.
**Persistencia:** Flag ativa atualizada no banco de dados.
**Evidencia esperada:** Screenshot mostrando fonte com status alterado.

### Passo 4: Verificar tipo da fonte
**Acao do ator:** O administrador observa o tipo de cada fonte (api ou scraper).
**Elemento UI:** Badge ou label indicando "api" ou "scraper" ao lado do nome.
**Resposta visual:** Fontes tipo "api" indicam conexao direta (ex: PNCP). Fontes tipo "scraper" indicam que a busca sera feita via Brave Search.
**Efeito funcional:** Nenhum — apenas visualizacao.
**Requisicao de rede:** Nenhuma.
**Persistencia:** Nenhuma.
**Evidencia esperada:** Screenshot com badges de tipo visiveis.

### Passo 5: Confirmar configuracao
**Acao do ator:** O administrador verifica que as fontes desejadas estao ativas e confirma a configuracao.
**Elemento UI:** Lista de fontes com estados finais.
**Resposta visual:** Fontes ativas marcadas em verde. Fontes inativas em cinza. Mensagem de sucesso se houve alteracao.
**Efeito funcional:** Configuracao aplicada. Proxima busca de editais (CaptacaoPage) usara apenas fontes ativas.
**Requisicao de rede:** Nenhuma adicional (alteracoes ja persistidas no Passo 3).
**Persistencia:** Nenhuma adicional.
**Evidencia esperada:** Screenshot final da configuracao de fontes.

## Fluxos alternativos

### FA-001: Criar nova fonte
**Condicao:** Administrador precisa adicionar uma fonte nao pre-cadastrada.
**Variacao:** Formulario de criacao com campos: nome, tipo (api/scraper), URL, ativa (boolean). Chamada crudCreate("fontes-editais", { nome, tipo, url, ativa }).

### FA-002: Editar URL de fonte existente
**Condicao:** URL da API de uma fonte mudou.
**Variacao:** Administrador edita campo URL da fonte. crudUpdate chamado com nova URL.

### FA-003: Erro ao carregar fontes
**Condicao:** Backend retorna erro no GET de fontes.
**Variacao:** errorFontes exibido no card. Lista vazia. Administrador pode tentar recarregar.

## Fluxos de excecao

### FE-001: Backend indisponivel
**Condicao:** Backend na porta 5007 esta fora do ar.
**Resposta:** loadingFontes permanece true ou errorFontes exibe mensagem de erro de rede.

### FE-002: Erro ao atualizar fonte
**Condicao:** PUT /api/crud/fontes-editais/{id} retorna erro.
**Resposta:** Estado local revertido. Mensagem de erro exibida.

### FE-003: Tentativa de desativar todas as fontes
**Condicao:** Administrador desativa todas as fontes.
**Resposta:** Sistema permite (nao ha validacao frontend), mas busca de editais retornara resultados vazios.

1. Lista de fontes deve carregar via GET /api/crud/fontes-editais
2. Cada fonte deve exibir nome, tipo, URL e status ativa
3. Toggle de ativacao deve chamar PUT /api/crud/fontes-editais/{id}
4. Fontes ativas devem ser usadas na CaptacaoPage para busca de editais
5. Fontes tipo "api" e "scraper" devem ser diferenciadas visualmente
6. Erro de carregamento deve exibir mensagem (errorFontes)
