# Caso de Uso: UC-010

## Nome
Configuracao de Fontes de Editais

## Objetivo
Permitir que o administrador configure as fontes de editais do sistema, incluindo listar fontes disponiveis, ativar/desativar fontes e configurar parametros de conexao (tipo e URL).

## Escopo
Modulo Parametrizacoes — ParametrizacoesPage.tsx > aba `Fontes de Busca`

## Atores
- Administrador (operador do sistema com permissao de configuracao)

## Gatilho
O administrador acessa a ParametrizacoesPage e navega para a secao de Fontes de Editais.

## Pre-condicoes
1. Usuario autenticado no sistema
2. Backend ativo na porta 5007
3. Tabela `fontes_editais` existente no banco de dados (disponivel)
4. Registro de `parametros_score` disponivel para palavras-chave e NCMs

## Pos-condicoes
1. Fontes ficam ativadas ou desativadas no banco
2. Palavras-chave e NCMs alimentam rotinas de monitoramento e captacao
3. Fontes ativadas serao utilizadas na busca de editais
4. Fontes desativadas serao ignoradas na busca

## Regras de negocio
- RN-PAR-002: Fontes de edital possuem UUID unico. Frontend envia UUID e backend resolve nome via DB
- RN-UC010-001: Cada fonte tem tipo: "api" (conexao direta com API oficial) ou "scraper" (busca via Brave Search/Serper)
- RN-UC010-002: Fontes com tipo "api" possuem URL da API direta (ex: PNCP = pncp.gov.br/api)
- RN-UC010-003: Fontes com tipo "scraper" usam Brave Search API para extracoes web
- RN-UC010-004: A flag "ativa" controla se a fonte sera considerada nas buscas de captacao
- RN-UC010-005: PNCP e a fonte principal com tipo "api" e estrategia hibrida (Search API + detalhes)
- RN-UC010-006: Chave de API do Brave Search e configurada via .env (SCRAPE_API=brave), nao na interface de fontes

## Componentes de UI envolvidos
- ParametrizacoesPage.tsx — aba `Fontes de Busca`
- Lista de fontes disponiveis com controle de ativacao
- Botao `Ativar` ou `Desativar` por fonte — alterna o status operacional de `fontes_editais`
- Campo de texto editavel para palavras-chave livres
- Botao `Salvar Palavras-chave` — atualiza lista de palavras-chave
- Campo de texto editavel para NCMs
- Botao `Salvar NCMs` — atualiza lista de NCMs de busca
- Botao `Gerar do portfolio (Onda 4)` — **desabilitado** (nao compoe escopo implementado)
- Botao `Sincronizar NCMs (Onda 4)` — **desabilitado** (nao compoe escopo implementado)

## Endpoints ou servicos envolvidos
- GET /api/crud/fontes-editais — lista todas as fontes
- PUT /api/crud/fontes-editais/{id} — ativa ou desativa fonte (altera campo `ativa`)
- GET /api/crud/parametros-score — carrega palavras-chave e NCMs existentes
- PUT /api/crud/parametros-score/{id} — persiste palavras-chave e NCMs atualizados

## Dados manipulados
- Fonte: id (UUID), nome (string), tipo ("api" | "scraper"), url (string), ativa (boolean)
- Palavras-chave: lista de termos livres persistidos em `parametros_score`
- NCMs: lista de codigos NCM persistidos em `parametros_score`

### Persistencia observada (UC-F16)
Tabelas utilizadas: `fontes_editais`, `parametros_score`

## Fluxo principal

### Passo 1: Acessar aba Fontes de Busca
**Acao do ator:** O administrador acessa a ParametrizacoesPage e clica na aba `Fontes de Busca`.
**Elemento UI:** Aba `Fontes de Busca` na ParametrizacoesPage.
**Resposta visual:** Lista de fontes disponiveis e carregada. Palavras-chave e NCMs atuais sao exibidos.
**Efeito funcional:** Pagina carrega dados de `fontes_editais` e `parametros_score`.
**Requisicao de rede:** GET /api/crud/fontes-editais + GET /api/crud/parametros-score.
**Persistencia:** Nenhuma.
**Evidencia esperada:** Screenshot da aba Fontes de Busca com lista de fontes e campos de palavras-chave/NCMs.

### Passo 2: Visualizar lista de fontes
**Acao do ator:** O administrador revisa a lista de fontes disponiveis.
**Elemento UI:** Lista de fontes na aba `Fontes de Busca`.
**Resposta visual:** Lista exibida com nome, tipo e status atual de cada fonte (ativa/inativa).
**Efeito funcional:** Estado de fontes populado com dados do backend.
**Requisicao de rede:** Ja carregado no Passo 1.
**Persistencia:** Nenhuma.
**Evidencia esperada:** Screenshot da lista de fontes com status de cada uma.

### Passo 3: Ativar ou desativar fonte
**Acao do ator:** O administrador clica em `Ativar` ou `Desativar` em uma fonte.
**Elemento UI:** Botao `Ativar` ou `Desativar` ao lado da fonte.
**Resposta visual:** Status da fonte muda imediatamente na interface.
**Efeito funcional:** `crudUpdate("fontes-editais", id, { ativa: true/false })` chamado.
**Requisicao de rede:** PUT /api/crud/fontes-editais/{id} com { ativa: true/false }.
**Persistencia:** Flag `ativa` atualizada no banco em `fontes_editais`.
**Evidencia esperada:** Screenshot mostrando fonte com status alterado.

### Passo 4: Editar palavras-chave
**Acao do ator:** O administrador ajusta as palavras-chave livres no campo correspondente.
**Elemento UI:** Campo de texto editavel para palavras-chave.
**Resposta visual:** Campo editavel com lista de termos.
**Efeito funcional:** Estado de palavras-chave atualizado localmente.
**Requisicao de rede:** Nenhuma (ainda nao salvo).
**Persistencia:** Nenhuma.
**Evidencia esperada:** Screenshot com palavras-chave editadas.

### Passo 5: Salvar palavras-chave
**Acao do ator:** O administrador clica em `Salvar Palavras-chave`.
**Elemento UI:** Botao `Salvar Palavras-chave`.
**Resposta visual:** Mensagem de confirmacao apos persistencia.
**Efeito funcional:** `crudUpdate("parametros-score", id, { palavras_chave: [...] })` chamado.
**Requisicao de rede:** PUT /api/crud/parametros-score/{id}.
**Persistencia:** Palavras-chave atualizadas em `parametros_score`.
**Evidencia esperada:** Screenshot apos salvar palavras-chave.

### Passo 6: Editar e salvar NCMs
**Acao do ator:** O administrador ajusta a lista de NCMs e clica em `Salvar NCMs`.
**Elemento UI:** Campo de texto editavel para NCMs + botao `Salvar NCMs`.
**Resposta visual:** Mensagem de confirmacao apos persistencia.
**Efeito funcional:** `crudUpdate("parametros-score", id, { ncms_busca: [...] })` chamado.
**Requisicao de rede:** PUT /api/crud/parametros-score/{id}.
**Persistencia:** NCMs atualizados em `parametros_score`.
**Evidencia esperada:** Screenshot apos salvar NCMs.

## Fluxos alternativos

### FA-001: Botoes de Onda 4 desabilitados
**Condicao:** Administrador visualiza os botoes `Gerar do portfolio (Onda 4)` e `Sincronizar NCMs (Onda 4)`.
**Variacao:** Esses botoes aparecem desabilitados na interface e nao executam nenhuma acao. Nao compoem o escopo implementado nesta fase.

## Fluxos de excecao

### FE-001: Backend indisponivel
**Condicao:** Backend na porta 5007 esta fora do ar.
**Resposta:** loadingFontes permanece true ou errorFontes exibe mensagem de erro de rede.

### FE-002: Erro ao atualizar fonte
**Condicao:** PUT /api/crud/fontes-editais/{id} retorna erro.
**Resposta:** Estado local revertido. Mensagem de erro exibida.

### FE-003: Tentativa de desativar todas as fontes
**Condicao:** Administrador desativa todas as fontes.
**Resposta:** Sistema permite (nao ha validacao frontend), mas busca de editais retornara resultados vazios.
