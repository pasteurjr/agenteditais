# Caso de Uso: UC-013

## Nome
Buscar editais por termo

## Objetivo
Permitir ao usuario buscar editais de licitacao em multiplas fontes (PNCP Search API + Brave/Serper scraper) a partir de palavras-chave, exibindo resultados consolidados com score de aderencia ao portfolio.

## Escopo
Modulo Captacao — CaptacaoPage.tsx

## Atores
- Usuario autenticado (Analista Comercial)
- Sistema backend (Flask)
- PNCP Search API
- Brave Search API / Serper (scraper)

## Gatilho
Usuario digita um termo de busca no campo de pesquisa da CaptacaoPage e clica no botao "Buscar" ou pressiona Enter.

## Pre-condicoes
1. Usuario esta autenticado (useAuth do AuthContext)
2. Backend Flask esta ativo na porta 5007
3. Chave de API do Brave Search (SCRAPE_API=brave) configurada no .env
4. Ao menos uma fonte de editais cadastrada em Parametrizacoes
5. Portfolio de produtos cadastrado (para calculo de score de aderencia)
6. Conexao com internet disponivel para acesso a APIs externas
7. Empresa do usuario cadastrada com CNPJ valido

## Pos-condicoes
1. Lista de editais exibida na tela com dados consolidados
2. Cada edital exibe score de aderencia ao portfolio
3. StatCards atualizados com metricas de busca (total encontrado, media de score, etc.)
4. Valores estimados enriquecidos via endpoint de detalhes PNCP
5. Resultados incluem dados de ambas as fontes (PNCP + scraper)
6. Editais ficam disponiveis para selecao e salvamento (UC-015)
7. Historico de busca registrado para monitoramento

## Regras de negocio
1. **RN-CAP-001:** Estrategia hibrida — Search API para busca rapida (~15s) + detalhes para valores
2. **RN-CAP-002:** Search API (pncp.gov.br/api/search/?q=) filtra por texto mas NAO retorna valorTotalEstimado
3. **RN-CAP-003:** Detalhes via /orgaos/{cnpj}/compras/{ano}/{seq} retorna valorTotalEstimado
4. **RN-CAP-004:** Chat e UI usam mesmo caminho: _buscar_editais_multifonte() -> tool_buscar_editais_fonte()
5. **RN-CAP-005:** CaptacaoPage usa crudList/crudCreate para persistencia
6. **RN-CAP-006:** Endpoint antigo /contratacoes/publicacao com sub-janelas foi abandonado por ser lento (~200s+)
7. **RN-CAP-007:** Score de aderencia calculado automaticamente comparando objeto do edital com portfolio cadastrado

## Componentes de UI envolvidos
1. Campo de texto para termo de busca (TextInput com icone Search)
2. Botao "Buscar" (ActionButton com icone Sparkles)
3. StatCards com metricas da busca (total, media score, valor total)
4. DataTable com colunas: numero, orgao, UF, objeto, valor, data abertura, score, status
5. Indicador de loading/progresso durante busca
6. Badges de fonte (PNCP, Brave, etc.) em cada resultado
7. Icones de acao por linha: Eye (ver detalhes), Save (salvar), ExternalLink (abrir no portal)

## Endpoints ou servicos envolvidos
1. **POST /api/editais/buscar** — busca multifontes (PNCP + scraper)
2. **GET https://pncp.gov.br/api/search/?q={termo}** — PNCP Search API
3. **GET https://pncp.gov.br/api/consulta/v1/orgaos/{cnpj}/compras/{ano}/{seq}** — detalhes com valor
4. **Brave Search API** — scraping de fontes alternativas (ComprasNet, Licitacoes-e)
5. **tool_buscar_editais_fonte** — tool de IA para busca em fonte especifica
6. **tool_score_aderencia** — calcula score de aderencia ao portfolio
7. **GET /api/editais/{id}/itens** — itens do edital (enriquecimento)

## Dados manipulados
1. **Termo de busca:** string com palavras-chave do usuario
2. **Filtros:** UF, categoria, modalidade, periodo, faixa de valor
3. **EditalBusca:** interface com id, numero, orgao, uf, objeto, valor, dataAbertura, score, status
4. **Scores:** tecnico, comercial, recomendacao (EditalScores)
5. **Itens do edital:** numero_item, descricao, quantidade, unidade_medida, valor_unitario_estimado
6. **Dados PNCP:** cnpj_orgao, ano_compra, seq_compra (para download PDF e URL portal)
7. **Classificacao:** tipoProdutoInferido, origemInferida, potencialGanho, intencaoEstrategica

## Fluxo principal
1. Usuario acessa a CaptacaoPage e visualiza o formulario de busca com campo de texto e filtros
2. Usuario digita o termo de busca (ex: "seringa descartavel") e clica em "Buscar"
3. Frontend envia POST /api/editais/buscar com termo e filtros ao backend Flask
4. Backend executa _buscar_editais_multifonte() que dispara em paralelo: PNCP Search API + Brave scraper
5. Para cada resultado PNCP, backend busca valores via endpoint de detalhes (/orgaos/{cnpj}/compras/{ano}/{seq}) em paralelo
6. Backend consolida resultados de todas as fontes, calcula score de aderencia e retorna lista ordenada por score
7. Frontend renderiza DataTable com resultados, StatCards com metricas e badges de fonte em cada linha

## Fluxos alternativos
1. **FA-01 — Busca sem resultados:** Sistema exibe mensagem "Nenhum edital encontrado para o termo informado" e sugere termos alternativos
2. **FA-02 — Apenas uma fonte responde:** Sistema exibe resultados parciais com aviso de que nem todas as fontes responderam
3. **FA-03 — Busca via Chat:** Usuario usa o FloatingChat com intencao de busca; DeepSeek classifica intencao e chama mesma tool_buscar_editais_fonte
4. **FA-04 — Paginacao:** Se mais de 50 resultados, DataTable exibe paginacao com 20 itens por pagina
5. **FA-05 — Busca por NCM:** Usuario informa codigo NCM em vez de texto livre; sistema adapta consulta
6. **FA-06 — Monitoramento ativo:** Usuario ativa monitoramento para receber alertas de novos editais com mesmo termo
7. **FA-07 — Exportar resultados:** Usuario clica em Download para exportar lista de resultados em CSV

## Fluxos de excecao
1. **FE-01 — PNCP indisponivel:** Backend retorna erro 503; frontend exibe alerta "PNCP temporariamente indisponivel" e tenta apenas fontes alternativas
2. **FE-02 — Brave API key invalida:** Backend retorna erro de autenticacao; frontend exibe alerta e busca apenas no PNCP
3. **FE-03 — Timeout de busca:** Se busca exceder 30s, backend cancela e retorna resultados parciais obtidos ate o momento
4. **FE-04 — Termo muito curto:** Frontend valida minimo de 3 caracteres antes de enviar requisicao
5. **FE-05 — Usuario nao autenticado:** Middleware de auth redireciona para LoginPage
6. **FE-06 — Erro de rede:** Frontend exibe toast de erro e botao "Tentar novamente"
7. **FE-07 — Rate limit PNCP:** Backend detecta HTTP 429 e aplica backoff exponencial antes de retentar

## Criterios de validacao
1. Busca retorna resultados em menos de 20 segundos para termo comum
2. Resultados incluem editais de pelo menos uma fonte (PNCP ou scraper)
3. Cada edital exibe score de aderencia numerico (0-100)
4. Valores estimados sao preenchidos via endpoint de detalhes PNCP
5. DataTable permite ordenacao por score, valor e data de abertura
6. StatCards refletem corretamente totais e medias dos resultados
7. Busca pelo Chat retorna mesmos resultados que busca pela UI
